# 🔍 비지도 학습 (Unsupervised Learning)

---

## 1. 비지도 학습이란?

> **Label(정답) 없이, 데이터 자체의 구조/패턴/집단을 스스로 찾아내는 학습**

---

### 지도 학습 vs 비지도 학습 비교

| 구분 | 지도 학습 (Supervised) | 비지도 학습 (Unsupervised) |
|------|----------------------|--------------------------|
| **입력** | Feature + Label (정답) | Feature만 |
| **목표** | Label을 예측하는 규칙 학습 | 데이터의 구조/패턴 발견 |
| **결과물** | 예측 모델 | 군집, 축소된 표현, 이상치 탐지 |
| **예시** | 스팸 분류, 집값 예측 | 고객 세분화, 이상 거래 탐지 |

---

### 비지도 학습의 목표

예측(Prediction)이 아닌 **구조 발견, 요약, 표현(Embedding)**

```
지도 학습: "이 이메일이 스팸인가 아닌가?" (정답 있음)
비지도 학습: "이 데이터들을 어떻게 자연스럽게 묶을 수 있을까?" (정답 없음)
```

---

### 비지도 학습의 주요 응용 분야

| 분야 | 설명 | 예시 |
|------|------|------|
| **군집화 (Clustering)** | 유사한 데이터끼리 그룹화 | 고객 세분화, 문서 분류 |
| **차원 축소 (PCA 등)** | 복잡한 데이터를 간단하게 압축 | 이미지 압축, 시각화 |
| **밀도 추정** | 데이터 분포 학습 | 생성 모델 |
| **이상치 탐지** | 정상과 다른 데이터 찾기 | 사기 거래 탐지, 불량 제품 검출 |

---

### 비지도 학습의 핵심 질문

1. **"무엇을 비슷하고, 다른 것으로 볼 것인가?"**
   - 어떤 기준으로 유사도를 측정할 것인가?
   - 유클리드 거리? 코사인 유사도? 맨해튼 거리?

2. **"전처리를 어떻게 할 것인가?"**
   - 데이터 스케일이 다르면 → 표준화 필요
   - 잡음이 많으면 → 전처리 후 학습

---

## 2. 군집화 (Clustering)

> **데이터 안에서 하위 집단(Cluster)을 찾는 기법들의 총칭**

---

### 좋은 군집화의 조건

```
같은 군집 안: 데이터들이 서로 비슷해야 함 (within-cluster similarity ↑)
다른 군집 간: 데이터들이 서로 달라야 함 (between-cluster difference ↑)
```

---

### 주의사항

- 모든 데이터가 반드시 명확한 군집을 형성하지는 않습니다
- Feature 선택에 따라 군집이 잘 나뉘기도, 안 나뉘기도 합니다
- **"이게 맞는 군집인가?"를 검증하기 어렵습니다 (정답이 없으니까!)**

---

### 대표적인 군집화 기법

| 방법 | 특징 | 군집 수 |
|------|------|---------|
| **K-means** | 군집 수 K를 미리 정하고 분할 | 미리 지정 필요 |
| **계층적 군집 (Hierarchical)** | 트리 구조로 점진적 분할/합병 | 나중에 선택 가능 |

---

## 3. K-means 클러스터링

> **모든 데이터를 K개의 군집에 빠짐없이, 겹치지 않게 하나씩 배정하는 알고리즘**

---

### 기본 규칙

1. 모든 데이터는 반드시 하나의 군집에 속해야 함 (누락 없음)
2. 하나의 데이터가 두 개 이상의 군집에 속할 수 없음 (겹침 없음)

---

### 목표: 클러스터 내부 변동 최소화

> **중심점(Centroid)에서 데이터들이 얼마나 가까이 모여 있는가?**

```
좋은 군집: 군집 내부 데이터들이 중심점에 가까이 모여 있음 (변동 작음)
나쁜 군집: 군집 내부 데이터들이 중심점에서 멀리 흩어져 있음 (변동 큼)
```

---

### K-means 동작 순서 (단계별 설명)

#### 🔢 Step 1: 무작위 초기화

```
데이터가 100개이고, K = 3이라면:
→ 100개 데이터에 무작위로 1, 2, 3 중 하나의 군집 번호를 임시 부여
```

#### 📍 Step 2: 중심점(Centroid) 계산

```
군집 1에 속한 데이터들의 평균 위치 → 중심점 1
군집 2에 속한 데이터들의 평균 위치 → 중심점 2
군집 3에 속한 데이터들의 평균 위치 → 중심점 3
```

#### 🔄 Step 3: 재할당

```
각 데이터를 가장 가까운 중심점의 군집으로 재배정
(유클리드 거리 등으로 거리 계산)
```

#### ✅ Step 4: 반복 또는 종료

```
할당이 더 이상 바뀌지 않으면 → 종료 (수렴)
바뀌었다면 → Step 2로 돌아가 반복
```

---

### 시각적 이해

```
초기 상태:              Step 2/3 후:            수렴 후:
● ○ ▲ ● ○            ● ● ○ ▲ ▲             [● ●] [○ ○] [▲ ▲]
▲ ○ ● ▲ ○    →    ● ○ ▲ ○ ▲     →    잘 모인 3개 군집
(무작위 배정)        (중심점 계산)           (최종 군집)
```

---

### K-means의 특징

#### ✅ 장점
- 단순하고 직관적
- 계산이 빠름 (대용량 데이터에도 적용 가능)
- 구현이 쉬움

#### ⚠️ 핵심 단점: 지역 최솟값 문제 (Local Minimum)

```
K-means는 전역 최솟값(Global Minimum)을 보장하지 않습니다!

왜냐하면: 초기 무작위 배정에 따라
→ 다른 결과에 수렴할 수 있기 때문

예시:
시도 1 (초기값 A): 중심점이 왼쪽에 모임 → 나쁜 군집화
시도 2 (초기값 B): 중심점이 고르게 분포 → 좋은 군집화

같은 알고리즘, 같은 데이터인데 다른 결과!
```

#### 대응 방법

- **여러 번 실행** (다른 초기값으로 반복)
- **K-means++** 알고리즘 사용 (초기값을 더 지능적으로 선택)

---

### K는 어떻게 정할까?

K-means의 가장 큰 고민: **군집 수를 미리 알 수가 없다!**

**엘보우 방법 (Elbow Method):**
```
K를 1부터 늘려가면서 군집 내 변동(Inertia)을 기록

K=1: 변동 매우 큼
K=2: 변동 많이 줄어듦
K=3: 변동 조금 줄어듦 ← "꺾이는 지점" = 최적 K
K=4: 변동 거의 안 줄어듦
...

팔꿈치처럼 꺾이는 지점이 적절한 K
```

## 4. 계층적 군집화 (Hierarchical Clustering)

> **군집 수 K를 미리 정하지 않고, 전체 군집 구조를 트리(덴드로그램) 형태로 만드는 방법**

---

### 덴드로그램 (Dendrogram)

```
           ┌─────────────┐
           │             │
        ┌──┘    ┌────────┘
        │       │
     ┌──┘    ┌──┘
     │       │
  [A, B]  [C, D, E]
     
  (서로 유사한 것들이 먼저 합쳐지고, 점차 더 큰 군집으로 합쳐짐)
```

**수평선을 어디에 자르냐에 따라 원하는 K개의 군집을 얻을 수 있습니다!**

```
높은 곳에서 자르기 → 적은 수의 큰 군집 (K = 2)
낮은 곳에서 자르기 → 많은 수의 작은 군집 (K = 5)
```

---

### 계층적 군집화 동작 순서

#### 🔢 Step 1: 비유사도(거리) 계산

```
모든 데이터 쌍 사이의 거리를 계산
(n개의 데이터 → n(n-1)/2 쌍)

사용 가능한 거리 측도:
- 유클리드 거리: √((x₁-x₂)² + (y₁-y₂)²) (직선 거리)
- 맨해튼 거리: |x₁-x₂| + |y₁-y₂| (격자 거리)
```

#### 🔗 Step 2: 가장 유사한 두 군집 병합

처음에는 모든 데이터가 각자 하나의 군집입니다.
→ 가장 가까운 두 군집을 하나로 합칩니다.

#### 🔄 Step 3: 반복

하나의 군집이 남을 때까지 Step 2를 반복합니다.

---

### 연결법 (Linkage): 군집 간 거리를 어떻게 측정할까?

두 군집 사이의 거리를 계산하는 방법에 따라 결과가 달라집니다:

| 연결법 | 거리 기준 | 특징 |
|--------|---------|------|
| **단일 연결 (Single Linkage)** | 두 군집에서 가장 가까운 점 쌍의 거리 (최솟값) | 길쭉한 군집 형성 경향 |
| **완전 연결 (Complete Linkage)** | 두 군집에서 가장 먼 점 쌍의 거리 (최댓값) | 둥근 군집 형성 경향 |
| **평균 연결 (Average Linkage)** | 두 군집의 모든 점 쌍 거리의 평균 | 균형 잡힌 결과 |

---

### 시각적 이해

```
데이터: A B C D E

초기:  [A] [B] [C] [D] [E]  ← 각자 독립 군집

Step1: [A,B] [C] [D] [E]    ← A와 B가 가장 가까워 합병

Step2: [A,B] [C,D] [E]      ← C와 D 합병

Step3: [A,B,C,D] [E]        ← 두 그룹 합병

Step4: [A,B,C,D,E]          ← 최종 하나로 합병

덴드로그램은 이 과정을 트리로 시각화한 것
```

---

### 계층적 군집화의 특징

#### ✅ 장점
- **K를 미리 정하지 않아도 됨** → 덴드로그램을 보고 나중에 결정
- **전체 군집 구조를 한눈에 볼 수 있음**
- 다양한 K 값의 결과를 한 번에 확인 가능

#### ⚠️ 단점
- **K-means보다 계산 시간이 많이 걸림** → 데이터가 많으면 느림
  - 이유: 모든 단계에서 군집 간 거리를 다시 계산해야 하기 때문
- 연결법 선택에 따라 결과가 크게 달라질 수 있음

---

### K-means vs 계층적 군집화 비교

| 항목 | K-means | 계층적 군집화 |
|------|---------|-------------|
| **K 사전 지정** | 필요 | 불필요 (나중에 선택) |
| **계산 속도** | 빠름 | 느림 |
| **결과 시각화** | 어려움 | 덴드로그램으로 직관적 |
| **데이터 크기** | 대용량 적합 | 소~중간 규모 적합 |
| **군집 모양** | 구형 군집에 적합 | 다양한 형태 가능 |
| **반복성** | 실행마다 결과 다를 수 있음 | 항상 동일한 결과 |

---

## 5. 클러스터링 시 주의사항

### ⚖️ 스케일링 (Scaling) - 매우 중요!

> **데이터의 단위가 서로 다르면 반드시 표준화가 필요합니다!**

```
예시: 고객 데이터
- 나이: 20~60 (범위 40)
- 연봉: 2000만~1억 (범위 8000만)

표준화 없이 클러스터링하면?
→ 연봉의 숫자가 훨씬 크므로, 나이는 거의 무시됨
→ "연봉만으로" 군집화되는 결과 발생!

해결: 모두 0~1 사이로 표준화 or 평균 0, 표준편차 1로 변환
```

---

### 🔢 최적 K 결정의 어려움

- 몇 개의 군집이 적합한지 **합의된 정답이 없음**
- 같은 데이터라도 관점에 따라 적절한 K가 다를 수 있음

**권장 접근법:**
- 단일 시도보다 **여러 K 값으로 반복 시도**
- 다양한 평가 지표 활용 (실루엣 스코어, 엘보우 방법 등)
- 도메인 지식을 함께 활용

---

## 6. 전체 정리

```
비지도 학습
├── 군집화 (Clustering)
│   ├── K-means: 빠르지만 K를 미리 정해야 함, 지역 최솟값 문제
│   └── 계층적 군집화: 느리지만 K 불필요, 덴드로그램으로 시각화
│
├── 차원 축소 (PCA 등)
│   └── 복잡한 데이터를 간단하게 표현
│
└── 이상치 탐지
    └── 정상 패턴에서 벗어난 데이터 찾기
```

> **핵심 메시지:**
> 비지도 학습은 정답이 없는 환경에서 데이터 스스로 이야기하는 **구조와 패턴을 찾아내는 방법**입니다.
> 전처리(스케일링)와 여러 번의 시도가 중요하며,
> 결과 해석에는 항상 도메인 지식이 함께 필요합니다.
