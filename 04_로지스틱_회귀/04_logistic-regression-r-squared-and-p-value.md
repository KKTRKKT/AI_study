## 1. 왜 R²가 여러 종류인가요?

### 🤔 로지스틱 회귀에서 R²가 어려운 이유

선형 회귀에서는 R²가 딱 하나예요.

```
R² = (SS평균 - SS적합) / SS평균
```

잔차 제곱합(SS)으로 계산하는 깔끔한 공식이에요.

그런데 로지스틱 회귀에서는 잔차가 ±무한대여서 SS를 계산할 수 없어요. 그래서 R²를 계산하는 방법이 10가지 이상 제안됐어요.

> ⚠️ **실용적 조언:** 자신의 분야에서 사람들이 어떤 방법을 쓰는지 먼저 확인하세요. 이 문서에서는 계산이 쉽고 자주 쓰이는 **맥파든 유사 R²** 를 다뤄요.

---

## 2. 선형 회귀 R²를 다시 생각해보기

### 선형 회귀에서의 비교 구조

```
(나쁜 모델) 기준선 = 전체 평균 직선 → SS평균
(좋은 모델) 최적 직선          → SS적합
R² = (SS평균 - SS적합) / SS평균
```

R²의 핵심 아이디어:
- **나쁜 모델의 오차 (SS평균):** 기준선 역할
- **좋은 모델의 오차 (SS적합):** 비교 대상
- "좋은 모델이 나쁜 모델보다 얼마나 더 나은가?" = R²

### 두 가지 한계점 확인

선형 회귀 R²에서:
- SS적합 = SS평균이면 → R² = 0 (직선이 평균선과 다를 바 없음)
- SS적합 = 0이면 → R² = 1 (직선이 모든 점을 완벽히 통과)

로지스틱 회귀에서도 이 두 한계점 사이를 오가는 지표를 만들어야 해요.

## 3. 맥파든 유사 R² — 로그 우도로 R² 만들기

### 아이디어

SS 대신 **로그 우도(Log-Likelihood)** 를 써요.

| 선형 회귀 | 로지스틱 회귀 대용물 |
|---|---|
| SS적합 (좋은 모델 오차) | LL적합 (최적 곡선의 로그 우도) |
| SS평균 (나쁜 모델 오차) | LL전체확률 (수평선의 로그 우도) |

### LL전체확률 계산

"전체 비만 확률"(몸무게를 전혀 고려하지 않는 모델)의 로그 우도예요.

```
전체 비만 확률 = 비만 생쥐 수 / 전체 생쥐 수
```

예를 들어 9마리 중 5마리가 비만이라면 전체 비만 확률 = 5/9 ≈ 0.556이에요. 이 확률로 수평선을 그리고, 모든 데이터 포인트의 로그 우도를 계산해요.

```
LL전체확률 = -6.18 (이 예시에서)
```

### LL적합 계산

최대 우도법으로 찾은 최적 곡선의 로그 우도예요.

```
LL적합 = -3.77 (이 예시에서)
```

### 맥파든 유사 R² 공식

```
R² = 1 - (LL적합 / LL전체확률)
   = 1 - (-3.77 / -6.18)
   = 1 - 0.610
   = 0.39
```

**이 수식이 말하는 것:**
몸무게를 고려한 최적 곡선이 "몸무게 무시" 수평선보다 39% 더 잘 설명해요.

## 4. R²가 0과 1 사이에 있는지 확인하기

### 케이스 1: 몸무게가 비만과 전혀 관계없을 때

```
데이터 패턴: 가벼운 생쥐도 비만, 무거운 생쥐도 비만/정상 (뒤섞임)
최적 곡선의 기울기 ≈ 0 (수평선과 거의 같음)
```

이 경우:
- LL적합 ≈ LL전체확률 (두 모델이 거의 같아요)
- R² = 1 - (LL적합 / LL전체확률) ≈ 1 - (같은 값 / 같은 값) = 1 - 1 = **0**

몸무게가 비만 예측에 도움이 안 되니 R² = 0이에요. ✅

### 케이스 2: 몸무게가 비만을 완벽하게 예측할 때

```
데이터 패턴: 가벼운 생쥐는 모두 정상, 무거운 생쥐는 모두 비만
최적 곡선이 데이터를 완벽히 통과
```

이 경우:
- 최적 곡선 예측 확률: 비만 생쥐는 ≈1, 비만 아닌 생쥐는 ≈0
- 모든 생쥐의 우도 ≈ 1 → 로그 우도 = log(1) = 0
- **LL적합 ≈ 0**

```
R² = 1 - (0 / LL전체확률) = 1 - 0 = **1**
```

완벽한 예측이니 R² = 1이에요. ✅

> 💡 **왜 LL적합이 0이 되냐고요?** 로그 우도는 항상 0 이하예요. 모델이 완벽하면 모든 우도가 1이고 log(1) = 0이에요.

## 5. p-값 계산 — 카이제곱 검정

### 아이디어

"이 R²가 우연히 나온 건 아닐까?"를 검정해야 해요.

선형 회귀에서는 F-통계량과 F-분포를 썼어요.
로지스틱 회귀에서는 **카이제곱 통계량(Chi-squared)** 과 **카이제곱 분포**를 써요.

### 카이제곱 통계량 계산

```
χ² = 2 × (LL적합 - LL전체확률)
   = 2 × (-3.77 - (-6.18))
   = 2 × 2.41
   = 4.82
```

**이 수식이 말하는 것:**
두 로그 우도의 차이에 2를 곱하면 카이제곱 분포를 따르는 값이 나와요.

### 자유도 계산

```
자유도 = p_적합 - p_전체확률
       = 2 - 1
       = 1
```

- **p_적합 = 2:** 최적 직선은 절편과 기울기 두 파라미터가 필요해요
- **p_전체확률 = 1:** 수평선은 전체 확률 하나만 필요해요

### p-값 계산

자유도 1인 카이제곱 분포에서 χ² = 4.82의 p-값을 구해요.

```
p-값 = 0.03
```

p-값 < 0.05이므로, 몸무게와 비만 사이의 관계는 통계적으로 유의미해요.

## 6. 전체 결과 정리

이 예시에서:
- **R² = 0.39:** 몸무게가 비만 변동의 39%를 설명해요 (효과 크기)
- **p-값 = 0.03:** 이 관계는 통계적으로 유의미해요 (우연이 아니에요)

## 7. 더 복잡한 공식에 대해

실제로 논문이나 소프트웨어에서 보이는 공식은 더 복잡해 보일 수 있어요.

```
R² = 1 - (LL적합 - LL포화모델) / (LL전체확률 - LL포화모델)
```

여기서 **포화 모델(Saturated Model)** 이 추가돼요. 포화 모델은 데이터 포인트마다 하나의 파라미터를 가진 최대한 복잡한 모델이에요.

> **로지스틱 회귀에서는 이 항을 무시해도 돼요!**
>
> 로지스틱 회귀의 포화 모델 로그 우도는 항상 **0**이에요. (모든 우도가 1 → log(1) = 0)
>
> 그래서 공식이 단순해져요:
> ```
> R² = 1 - (LL적합 - 0) / (LL전체확률 - 0) = 1 - LL적합/LL전체확률
> ```
>
> 포화 모델과 이탈도(Deviance)에 대한 자세한 내용은 다음 문서에서 다뤄요.

## ⚠️ 주의할 점 / 자주 하는 실수

- **맥파든 유사 R²는 선형 회귀 R²와 같은 기준으로 해석하면 안 돼요.** 선형 회귀에서 R² = 0.4는 "중간 정도"이지만, 로지스틱 회귀에서 맥파든 R² = 0.4는 꽤 좋은 편이에요. 값의 의미가 달라요.
- **R²가 크다고 항상 좋은 모델이 아니에요.** 표본이 작으면 R²가 크게 나와도 우연일 수 있어요. 반드시 p-값과 함께 봐야 해요.
- **카이제곱 검정의 자유도를 잘 계산해야 해요.** p_적합 - p_전체확률이에요. 변수가 많아지면 자유도도 그만큼 커져요.
- **로그 우도는 항상 음수예요.** 0에 가까울수록 좋아요. -3.77과 -6.18 중 -3.77이 더 좋아요.

**Q1.** 로지스틱 회귀에서 R²가 여러 종류인 이유는 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** 로지스틱 회귀에서는 잔차가 무한대여서 선형 회귀의 잔차 제곱합(SS)을 계산할 수 없기 때문이에요. 대안적인 방법들이 여럿 제안됐고, 어떤 방법이 최선인지에 대한 합의가 없어요.

**설명:** 선형 회귀는 잔차를 기반으로 R²를 계산해요. 로지스틱 회귀는 잔차가 ±∞라서 이 방법이 불가능해요. 그래서 로그 우도, 예측 확률 기반 방법 등 10여 가지 대안이 제안됐어요. 맥파든 유사 R²가 가장 많이 쓰이지만, 분야마다 관행이 달라요.

</details>

---

**Q2.** 맥파든 유사 R²에서 "LL전체확률"은 무엇이고 왜 "나쁜 모델"의 기준으로 쓰이나요?

<details>
<summary>정답 보기</summary>

**정답:** LL전체확률은 예측 변수를 전혀 사용하지 않고 전체 확률(수평선)만으로 예측할 때의 로그 우도예요. 변수가 없는 가장 단순한 모델이라서 "나쁜 모델"의 기준이 돼요.

**설명:** 선형 회귀에서 "전체 평균 수평선"이 나쁜 모델의 기준이었던 것처럼, 로지스틱 회귀에서는 "전체 비만 확률 수평선"이 기준이에요. 이 모델은 몸무게 정보를 전혀 활용하지 않아요. 최적 곡선의 로그 우도(LL적합)가 이것보다 얼마나 더 좋은지를 R²로 표현해요.

</details>

---

**Q3.** R² = 0은 어떤 상황에서 나오나요? R² = 1은요?

<details>
<summary>정답 보기</summary>

**정답:**
- R² = 0: 최적 곡선과 수평선(전체 평균)이 동일하게 나쁠 때예요. 즉, 예측 변수가 결과와 전혀 관계 없을 때예요.
- R² = 1: 최적 곡선이 모든 데이터를 완벽하게 설명할 때예요. LL적합이 0이 돼요.

**설명:**
- R² = 0 → LL적합 ≈ LL전체확률 → 1 - (같은값/같은값) = 0
- R² = 1 → LL적합 = 0 (완벽한 예측) → 1 - 0/LL전체확률 = 1 - 0 = 1

</details>

---

**Q4.** 카이제곱 검정에서 자유도는 어떻게 계산하나요?

<details>
<summary>정답 보기</summary>

**정답:** 최적 모델의 파라미터 수(p_적합) - 기준 모델의 파라미터 수(p_전체확률)예요.

**설명:** 몸무게 하나로 비만을 예측할 때:
- 최적 직선: 절편 + 기울기 = 2개 파라미터
- 수평선(전체 확률): 전체 확률 하나 = 1개 파라미터
- 자유도 = 2 - 1 = 1

변수가 2개라면 파라미터가 3개(절편 + 기울기1 + 기울기2)가 되고, 자유도 = 3 - 1 = 2가 돼요.

</details>

---

**Q5.** χ² = 4.82, 자유도 = 1일 때 p-값이 0.03이에요. 이 결과를 해석해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 몸무게와 비만 사이에 통계적으로 유의미한 관계가 있어요 (p < 0.05). 이 관계가 우연히 나왔을 확률은 3%예요.

**설명:** p-값 0.03은 "몸무게와 비만 사이에 실제로 관계가 없는데도 이런 결과가 우연히 나올 확률이 3%"라는 의미예요. p < 0.05이므로 통계적으로 유의미해요. R² = 0.39도 함께 보면, 몸무게가 비만 변동의 39%를 설명하고 이 효과가 통계적으로 유의미하다고 결론 내릴 수 있어요.

</details>

### 실전 적용 문제

**문제.** 당뇨병 예측 로지스틱 회귀를 돌렸더니:
- LL전체확률 = -120.5
- LL적합 = -85.3
- 파라미터 수: 수평선 = 1개, 최적 직선 = 4개 (절편 + 혈당 + 비만지수 + 나이)

1. 맥파든 유사 R²를 계산해보세요
2. 카이제곱 통계량과 자유도를 계산해보세요
3. 이 R² 값이 얼마나 의미 있는지 해석해보세요 (단, χ² = 70.4, 자유도 = 3일 때 p < 0.0001이에요)

**힌트:**
- R² = 1 - (LL적합 / LL전체확률)
- χ² = 2 × (LL적합 - LL전체확률)
- 자유도 = 파라미터 차이

<details>
<summary>예시 답안 보기</summary>

**1. 맥파든 유사 R² 계산:**
```
R² = 1 - (LL적합 / LL전체확률)
   = 1 - (-85.3 / -120.5)
   = 1 - 0.708
   = 0.292
```
R² ≈ 0.29

**2. 카이제곱 통계량과 자유도:**
```
χ² = 2 × (LL적합 - LL전체확률)
   = 2 × (-85.3 - (-120.5))
   = 2 × 35.2
   = 70.4

자유도 = 4 - 1 = 3
```

**3. 해석:**
R² = 0.29는 혈당, 비만지수, 나이가 당뇨병 변동의 약 29%를 설명해요. 맥파든 R²에서 0.29는 괜찮은 수준이에요 (일반적으로 0.2~0.4는 좋음으로 봐요). p < 0.0001이므로 이 관계는 매우 통계적으로 유의미해요. 세 변수가 모두 함께 당뇨병 예측에 의미 있게 기여하고 있어요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>

| 🎭 R² 10+ 종류 | 로지스틱 회귀는 잔차 기반 R²를 못 써서 여러 대안이 있어요 |
| 📊 맥파든 유사 R² | R² = 1 - (LL적합 / LL전체확률)이에요. 가장 많이 쓰여요 |
| 📉 LL적합 | 최적 곡선의 로그 우도예요. 음수이고 0에 가까울수록 좋아요 |
| 📈 LL전체확률 | 수평선(변수 미사용 모델)의 로그 우도예요. 나쁜 모델 기준이에요 |
| 📐 카이제곱 검정 | χ² = 2×(LL적합 - LL전체확률). 자유도 = 파라미터 수 차이예요 |
| ✅ 두 조건 | R²(효과 크기)가 크고 p-값(유의성)이 작아야 좋은 결과예요 |
