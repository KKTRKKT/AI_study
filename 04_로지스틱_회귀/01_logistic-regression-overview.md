## 1. 선형 회귀로는 뭘 할 수 있었나요?

### 복습

선형 회귀로 할 수 있는 것들을 정리해볼게요.

- **R²:** 두 변수가 얼마나 상관되어 있는지
- **p-값:** 그 상관이 통계적으로 유의미한지
- **예측:** 새로운 값을 예측하기 (예: 몸무게로 키 예측)

다중 회귀로는 연속형 변수(몸무게, 혈액량)와 이산형 변수(유전자형)를 모두 사용할 수 있고, 여러 모델을 비교할 수도 있어요.

이 모든 개념이 로지스틱 회귀에도 비슷하게 적용돼요. 단, 예측 대상이 달라요.

## 2. 로지스틱 회귀가 필요한 이유

### 🤔 선형 회귀로 "예/아니오"를 예측하면 안 되나요?

생쥐가 비만인지 아닌지를 예측하고 싶어요. 결과는 "비만이다(1)" 또는 "비만이 아니다(0)" 두 가지예요.

선형 회귀로 직선을 맞춰보면 문제가 생겨요.

- 아주 무거운 생쥐는 예측값이 1을 훨씬 초과해요 (예: 2.3)
- 아주 가벼운 생쥐는 예측값이 0 미만으로 떨어져요 (예: -0.5)
- 확률은 반드시 0~1 사이여야 하는데, 이 조건을 보장할 수 없어요

### 한 줄 정의

> **로지스틱 회귀란:** "예/아니오"처럼 두 가지 결과 중 하나를 예측하는 방법이에요. 직선 대신 **S자 곡선(로지스틱 함수)**을 사용해서 항상 0~1 사이의 확률을 출력해요.

### 일상 비유

> 💡 **비유:** 체온계로 발열 여부를 판단하는 것과 같아요. 체온이 올라갈수록 발열일 확률이 높아지지만, 그 확률은 0%에서 100% 사이에서만 움직여야 해요. 로지스틱 함수가 이 "0~100% 제한"을 자동으로 지켜줘요.

## 3. S자 곡선 (로지스틱 함수)

### 모양과 의미

```
확률
  1 ──────────────────────╮
  │                      ╱ │
  │                    ╱   │
 0.5 ─────────────── ╱    │
  │               ╱       │
  │             ╱         │
  0 ────────────          │
      가벼움            무거움 → 몸무게
```

- **곡선이 0에 가까울 때:** 비만일 확률이 낮아요 (가벼운 생쥐)
- **곡선이 0.5일 때:** 비만일 확률이 50%예요
- **곡선이 1에 가까울 때:** 비만일 확률이 높아요 (무거운 생쥐)

### 분류에 적용하기

로지스틱 회귀는 확률을 계산하지만, 실제로는 **분류(Classification)** 에 주로 사용해요.

```
임계값(Threshold) = 0.5로 설정:
- 확률 > 0.5 → "비만"으로 분류
- 확률 ≤ 0.5 → "비만 아님"으로 분류
```

> 💡 **보충 설명:** 임계값은 0.5가 기본이지만 상황에 따라 바꿀 수 있어요. 에볼라 감염 여부처럼 "놓치면 안 되는" 경우라면 임계값을 0.1처럼 낮게 설정해서 더 민감하게 잡아내요. 이 내용은 ROC/AUC 문서에서 자세히 다뤄요.

## 4. 로지스틱 회귀로 할 수 있는 것들

선형 회귀와 마찬가지로 여러 모델을 만들 수 있어요.

### 단순 모델

```
비만 여부 = f(몸무게)
```

### 복잡한 모델

```
비만 여부 = f(몸무게 + 유전자형)
비만 여부 = f(몸무게 + 유전자형 + 나이)
비만 여부 = f(몸무게 + 유전자형 + 나이 + 별자리)
```

연속형 변수(몸무게, 나이)와 이산형 변수(유전자형, 별자리) 모두 사용할 수 있어요.

### 변수 유용성 검정

각 변수가 예측에 도움이 되는지 검정할 수 있어요.

```
결과 예시:
- 몸무게:    유의미함 ✅
- 유전자형:  유의미함 ✅
- 나이:      유의미함 ✅
- 별자리:    쓸모없음 ❌ (totes useless)
```

쓸모없는 변수는 제거해서 더 효율적인 모델을 만들어요. 이 검정에는 **왈드 검정(Wald Test)** 을 사용해요.

> ⚠️ **선형 회귀와의 차이:** 선형 회귀에서는 단순 모델과 복잡한 모델을 F-통계량으로 직접 비교할 수 있었어요. 로지스틱 회귀에서는 이 방식이 어렵기 때문에, 각 변수의 효과가 0과 유의미하게 다른지를 개별적으로 검정해요.

## 5. 선형 회귀 vs 로지스틱 회귀 — 직선을 맞추는 방식의 차이

### 선형 회귀: 최소제곱법

- 잔차(데이터와 직선의 거리)를 제곱해서 더한 값을 최소화해요
- 잔차 제곱합을 R²와 모델 비교에도 사용해요

### 로지스틱 회귀: 최대 우도법

- 로지스틱 회귀에서는 잔차가 무한대로 가버려서 잔차 제곱합을 쓸 수 없어요
- 대신 **최대 우도법(Maximum Likelihood)** 을 사용해요

**최대 우도법의 핵심 아이디어:**

1. 후보 곡선을 하나 그어요
2. 이 곡선이 주어졌을 때, 실제 데이터(비만인 생쥐들과 비만이 아닌 생쥐들)가 관측될 **우도(Likelihood)** 를 계산해요
3. 모든 데이터 포인트의 우도를 곱해요 → 전체 데이터의 우도
4. 곡선을 조금씩 움직이면서 우도를 다시 계산해요
5. **우도가 가장 높은 곡선**을 최적의 로지스틱 곡선으로 선택해요

> 💡 **비유:** 어두운 방에서 손전등으로 가장 많은 물건이 잘 보이는 각도를 찾는 것처럼, 최대 우도법은 데이터가 가장 잘 "보이는(설명되는)" 곡선을 찾아요.

이 과정의 자세한 내용은 다음 문서(`2026-03-22_logistic-regression-maximum-likelihood.md`)에서 다뤄요.

## ⚠️ 주의할 점 / 자주 하는 실수

- **"로지스틱 회귀는 회귀인가 분류인가?"** → 둘 다예요! 확률(연속값)을 출력하는 회귀이지만, 임계값을 통해 분류에 사용해요.
- **결과가 반드시 0 또는 1이 아니에요.** 로지스틱 회귀의 출력은 "0~1 사이의 확률"이에요. 0.7이 나왔다면 "70% 확률로 비만"이라는 의미예요.
- **임계값 0.5가 항상 최선이 아니에요.** 상황에 따라 다른 임계값이 더 적절할 수 있어요. ROC 곡선을 이용해서 최적 임계값을 찾을 수 있어요.
- **R²가 없어요.** 로지스틱 회귀에는 선형 회귀의 R²에 해당하는 "맥파든 유사 R²"가 있지만, 해석 방식이 달라요. 자세한 내용은 이후 문서에서 다뤄요.

**Q3.** 로지스틱 회귀가 분류에 사용될 때 "임계값"이 무슨 역할을 하나요?

<details>
<summary>정답 보기</summary>

**정답:** 연속적인 확률값을 "양성/음성" 같은 이진 결정으로 변환하는 기준점이에요.

**설명:** 로지스틱 회귀는 확률(예: 0.73)을 출력해요. 이 숫자만으로는 "비만이다/아니다"라고 결정할 수 없어요. 임계값(예: 0.5)을 설정하면 "0.5 이상이면 비만, 미만이면 비만 아님"으로 분류할 수 있어요. 임계값은 목적에 따라 조정 가능해요. 에볼라 감염 감지처럼 놓치면 안 되는 경우에는 임계값을 낮게(0.1 등) 설정해요.

</details>

**Q4.** 로지스틱 회귀가 직선 맞추기에 최소제곱법을 쓰지 않는 이유는 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** 로지스틱 회귀에서 데이터 포인트는 로그 오즈 축에서 ±무한대로 밀려나기 때문에 잔차가 무한대가 돼서 잔차 제곱합을 계산할 수 없어요.

**설명:** 선형 회귀에서는 데이터 포인트가 유한한 범위에 있어서 잔차를 계산할 수 있어요. 하지만 로지스틱 회귀에서는 y축을 확률에서 로그 오즈로 변환하면, "비만=1"인 데이터 포인트는 +∞로, "비만=0"인 데이터 포인트는 -∞로 밀려요. 무한대에서의 잔차를 더할 수 없기 때문에 대신 최대 우도법을 사용해요.

</details>

**Q5.** 로지스틱 회귀에서 "왈드 검정(Wald Test)"은 무엇을 검정하나요?

<details>
<summary>정답 보기</summary>

**정답:** 각 변수의 계수(효과)가 통계적으로 0과 유의미하게 다른지를 검정해요.

**설명:** 선형 회귀에서는 단순 모델 vs 복잡한 모델을 F-통계량으로 비교할 수 있었어요. 로지스틱 회귀에서는 이 비교가 직접적으로 어렵기 때문에, 왈드 검정으로 각 변수의 계수가 0인지 아닌지를 개별적으로 검정해요. 계수가 0이면 그 변수는 예측에 도움이 안 된다는 의미예요.

</details>

### 실전 적용 문제

**문제.** 이메일 스팸 필터를 만들려고 해요. 각 이메일에 대해 다음 정보를 갖고 있어요.

- 이메일 길이 (연속형)
- 특정 키워드 포함 여부 (이산형: 포함/미포함)
- 발신자 도메인 (이산형: 알려진 도메인/모르는 도메인)
- 링크 개수 (연속형)

1. 이 문제에 선형 회귀를 써야 할까요, 로지스틱 회귀를 써야 할까요? 이유를 설명해보세요.
2. 로지스틱 회귀 모델을 만든다면 예측 결과는 무엇인가요?
3. 임계값을 0.5가 아닌 0.3으로 낮추면 어떤 효과가 있나요?

**힌트:**
- 예측하려는 결과(스팸/정상)의 성격을 생각해봐요
- 로지스틱 회귀의 출력(확률)이 무엇인지 생각해봐요
- 임계값 낮추기 → 민감도(Sensitivity) vs 특이도(Specificity)의 관계를 생각해봐요

<details>
<summary>예시 답안 보기</summary>

**1. 로지스틱 회귀를 써야 해요.**
예측 결과가 "스팸이다(1) / 정상이다(0)"으로 이진 분류이기 때문이에요. 선형 회귀를 쓰면 예측값이 0 미만이나 1 초과가 나올 수 있어서 확률로 해석할 수 없어요.

**2. 예측 결과:**
각 이메일이 스팸일 확률(0~1 사이)이에요. 예를 들어 특정 이메일에 대해 0.85가 나오면 "스팸일 확률 85%"라는 뜻이에요. 임계값을 기준으로 스팸/정상으로 분류해요.

**3. 임계값을 0.3으로 낮추면:**
스팸 탐지율(민감도)이 높아져요. 더 많은 이메일이 스팸으로 분류되기 때문에 실제 스팸을 더 잘 잡아낼 수 있어요. 하지만 정상 이메일이 스팸으로 잘못 분류되는 경우(위양성)도 늘어나요. 중요한 이메일을 스팸함에 놓칠 위험이 생겨요. 스팸 필터에서는 임계값 조정이 매우 중요한 설계 결정이에요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>
