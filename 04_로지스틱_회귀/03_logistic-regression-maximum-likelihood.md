## 1. 왜 최소제곱법을 못 쓰나요?

### 복습: 최소제곱법의 원리

선형 회귀에서는 잔차(데이터와 직선의 거리)를 제곱해서 더한 뒤, 이 합이 최소인 직선을 찾았어요.

```
잔차 = 실제값 - 예측값
잔차 제곱합 = Σ(잔차)² → 이걸 최소화
```

### 로지스틱 회귀에서 잔차가 ±무한대가 되는 이유

로지스틱 회귀에서는 y축을 확률에서 로그 오즈로 변환해요.

- **비만인 생쥐:** 확률 = 1 → 로그 오즈 = log(1/0) = **+∞**
- **비만이 아닌 생쥐:** 확률 = 0 → 로그 오즈 = log(0/1) = **-∞**

데이터 포인트가 ±무한대에 있으니, 직선까지의 잔차도 ±무한대예요. 무한대를 제곱하고 더하는 건 의미가 없어요.

> 💡 **비유:** 지평선 너머에 있는 물체까지의 거리를 재는 것처럼, 무한대에 있는 데이터와의 거리를 최소화하는 직선을 찾는 건 불가능해요.

### 해결책: 최대 우도법

잔차 대신 "이 곡선이 얼마나 데이터를 잘 설명하는가"를 **우도(Likelihood)** 로 측정하고, 이 우도를 최대화하는 곡선을 찾아요.

---

## 2. 우도(Likelihood)란 무엇인가요?

### 🤔 우도의 직관

> **우도:** 어떤 모델(곡선)이 주어졌을 때, 실제로 관측된 데이터가 나타날 "그럴듯함(plausibility)"이에요.

### 일상 비유

> 💡 **비유:** 날씨 예보 앱이 "오늘 비 올 확률 80%"라고 했는데 실제로 비가 왔어요. 이 앱의 예측이 실제 결과와 잘 맞는 거예요 → 우도가 높아요. 반대로 "비 올 확률 10%"라고 했는데 비가 왔다면 → 우도가 낮아요.

로지스틱 회귀에서도 마찬가지예요. 후보 곡선이 주어졌을 때, 실제로 "비만인 생쥐는 비만이라고, 비만이 아닌 생쥐는 비만이 아니라고" 잘 예측하는지를 우도로 측정해요.

---

## 3. 최대 우도법 단계별 과정

### 준비: 데이터와 후보 직선

비만 여부와 몸무게 데이터가 있어요.
- 비만인 생쥐들 (파란 점)
- 비만이 아닌 생쥐들 (빨간 점)

후보 직선(로그 오즈 스케일)을 하나 그어봐요.

### 1단계: 데이터 포인트를 후보 직선에 투영해요

각 생쥐의 몸무게를 직선에 대입해서 **후보 로그 오즈 값**을 구해요.

```
예시:
- 생쥐 A (몸무게 = 3) → 후보 로그 오즈 = 2.1
- 생쥐 B (몸무게 = 2) → 후보 로그 오즈 = 1.4
- 생쥐 C (몸무게 = 1) → 후보 로그 오즈 = -2.1
```

### 2단계: 로그 오즈를 확률로 변환해요

로지스틱 함수(시그모이드)로 변환해요.

```
확률 = e^(로그 오즈) / (1 + e^(로그 오즈))
```

**이 수식이 말하는 것:**
로그 오즈를 입력하면 확률(0~1)이 출력돼요. 로그 오즈가 클수록 확률이 1에 가깝고, 작을수록 0에 가까워요.

```
예시:
- 로그 오즈 2.1  → 확률 ≈ 0.89 (비만일 확률 89%)
- 로그 오즈 1.4  → 확률 ≈ 0.80 (비만일 확률 80%)
- 로그 오즈 -2.1 → 확률 ≈ 0.11 (비만일 확률 11%)
```

### 3단계: 각 데이터 포인트의 우도를 계산해요

- **비만인 생쥐:** 우도 = 그 생쥐가 비만일 확률 (곡선의 y값)
- **비만이 아닌 생쥐:** 우도 = 1 - (그 생쥐가 비만일 확률)

```
우도 계산 예시:
- 비만인 생쥐 (곡선 y값 = 0.89): 우도 = 0.89
- 비만인 생쥐 (곡선 y값 = 0.80): 우도 = 0.80
- 비만 아닌 생쥐 (곡선 y값 = 0.11): 우도 = 1 - 0.11 = 0.89
```

### 4단계: 전체 우도를 계산해요

모든 데이터 포인트의 우도를 **곱해요**.

```
전체 우도 = 0.89 × 0.90 × 0.91 × 0.92 × (1-0.90) × (1-0.30) × (1-0.01) × (1-0.01)
          = 아주 작은 수
```

> ⚠️ **문제:** 우도를 곱하면 숫자가 엄청나게 작아져요. 0과 1 사이의 수를 계속 곱하면 금방 0에 가까워져요. 컴퓨터가 다루기 어려운 숫자가 돼요.

### 5단계: 로그 우도(Log-Likelihood)로 대체해요

각 우도에 로그를 취한 뒤 **더해요**.

```
로그 우도 = log(0.89) + log(0.90) + log(0.91) + log(0.92) + log(0.10) + ...
          = -0.12 + (-0.11) + (-0.09) + (-0.08) + (-2.30) + ...
          = -3.77 (이 후보 직선의 로그 우도)
```

**왜 로그 우도를 쓰나요?**

로그를 취하면 곱셈이 덧셈으로 바뀌어요. 아주 작은 수를 다루는 대신, 음수 범위의 더 큰 수로 계산할 수 있어요.

> 💡 **중요:** 우도를 최대화하는 곡선 = 로그 우도를 최대화하는 곡선이에요. 어느 방법으로 해도 같은 최적 곡선이 나와요.

### 6단계: 직선을 회전하며 반복해요

```
후보 직선 1 → 로그 우도 = -3.77  ← 더 좋아요
후보 직선 2 → 로그 우도 = -4.15  ← 덜 좋아요
후보 직선 3 → ...
...
최적 직선   → 로그 우도 = 최댓값
```

> 💡 **알고리즘의 영리한 점:** 매번 무작위로 직선을 바꾸지 않아요. 직선을 어느 방향으로 바꿔야 로그 우도가 높아지는지를 계산해서 그 방향으로 이동해요. 이 덕분에 몇 번의 반복만으로도 최적 직선을 찾을 수 있어요.

---

## 4. 최대 우도법의 결과

로그 우도가 가장 높은(0에 가장 가까운) 직선이 선택돼요.

이 직선을 확률 스케일로 되돌리면 S자 모양의 로지스틱 곡선이 나와요. 이게 최적의 로지스틱 회귀 곡선이에요.

## ⚠️ 주의할 점 / 자주 하는 실수

- **"최대 우도법은 복잡한 수학이라 알 필요 없다"고 생각하지 마세요.** 원리를 이해하면 로지스틱 회귀의 출력(로그 우도, 이탈도 등)을 훨씬 잘 해석할 수 있어요.
- **로그 우도는 항상 0 이하예요.** 우도(확률)는 0~1이고, 0~1 사이 수의 로그는 항상 음수예요. 따라서 로그 우도는 음수값이에요. 0에 가까울수록(덜 음수일수록) 더 좋은 모델이에요.
- **우도와 확률을 혼동하지 마세요.** 확률은 모델이 고정됐을 때 데이터가 나타날 가능성이에요. 우도는 데이터가 고정됐을 때 모델 파라미터의 "그럴듯함"이에요. 비슷해 보이지만 관점이 다르게요.
- **최대 우도법은 항상 수렴하지 않을 수 있어요.** 데이터가 완벽히 분리되는 경우(Perfect Separation)에는 수렴하지 않아요. 이런 경우 알고리즘이 수렴하지 않는다는 경고가 나와요.

### 개념 확인 퀴즈

**Q1.** 로지스틱 회귀에서 최소제곱법을 쓸 수 없는 근본적인 이유는 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** 로그 오즈 스케일에서 실제 데이터 포인트(비만 = 1, 비만 아님 = 0)가 각각 +∞와 -∞에 위치하기 때문이에요. 무한대에 있는 데이터와 직선 사이의 잔차는 계산할 수 없어요.

**설명:** 비만인 생쥐의 실제 확률 = 1이고, log(1/(1-1)) = log(1/0) = +∞예요. 잔차 = +∞ - (직선값)도 무한대예요. 이걸 제곱하고 더할 수 없어서 최소제곱법이 작동하지 않아요. 대신 최대 우도법을 써요.

</details>

---

**Q2.** "비만인 생쥐의 우도"는 어떻게 계산하나요? "비만이 아닌 생쥐의 우도"는요?

<details>
<summary>정답 보기</summary>

**정답:**
- **비만인 생쥐의 우도:** 그 생쥐가 비만일 예측 확률이에요. 곡선의 y값이에요.
- **비만이 아닌 생쥐의 우도:** 1 - (그 생쥐가 비만일 예측 확률)이에요.

**설명:** 비만인 생쥐에 대해서는 "이 생쥐가 실제로 비만인 것과 곡선이 얼마나 일치하는가"를 봐요. 곡선이 높은 확률을 예측할수록 우도가 높아요. 비만이 아닌 생쥐는 반대로, 낮은 확률을 예측할수록(1 - 낮은 확률 = 높은 값) 우도가 높아요.

</details>

---

**Q3.** 전체 우도를 곱셈 대신 로그 우도를 덧셈으로 계산하는 이점이 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** 두 가지 장점이 있어요. 첫째, 아주 작은 수(0에 가까운 곱)를 다루는 대신 더 다루기 쉬운 음수 범위의 수를 더해요. 둘째, 곱셈 대신 덧셈이라 계산이 훨씬 빠르고 수치적으로 안정해요.

**설명:** 0.9 × 0.8 × 0.7 × ... 처럼 계속 곱하면 금방 컴퓨터 부동소수점 한계(0으로 수렴)에 부딪혀요. 로그를 취하면 log(0.9) + log(0.8) + ... = -0.11 + (-0.22) + ...처럼 덧셈이 돼서 훨씬 안정적이에요. 로그의 단조증가 특성 덕분에 로그 우도를 최대화해도 원래 우도를 최대화하는 것과 동일한 결과가 나와요.

</details>

---

**Q4.** 로그 우도는 항상 음수예요. 로그 우도가 -3.77과 -4.15 중 어느 것이 더 좋은 모델인가요?

<details>
<summary>정답 보기</summary>

**정답:** -3.77이 더 좋아요.

**설명:** 로그 우도는 음수이기 때문에 "최대화"한다는 것은 "0에 가장 가까운 값"을 찾는 거예요. -3.77은 -4.15보다 0에 더 가까워요. 즉, 우도가 더 높아요. 완벽한 모델이면 로그 우도 = 0이 나와요(우도 = 1일 때 log(1) = 0).

</details>

---

**Q5.** 최대 우도법 알고리즘이 "영리한 이유"는 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** 직선을 어느 방향으로 이동해야 로그 우도가 높아지는지를 계산해서 그 방향으로만 이동하기 때문이에요. 무작위로 탐색하지 않아요.

**설명:** 직선을 조금씩 기울여볼 때, 기울기(미분)를 계산해서 로그 우도가 올라가는 방향만 찾아요. 이 덕분에 수많은 후보를 모두 시험할 필요 없이 몇 번의 반복만으로 최적값에 수렴해요. 이 방법을 경사 상승법(Gradient Ascent)이라고 해요.

</details>

### 실전 적용 문제

**문제.** 아래 데이터에서 후보 직선의 로그 우도를 계산해보세요.

| 생쥐 | 비만 여부 | 후보 곡선 예측 확률 |
|---|---|---|
| A | 비만 (1) | 0.80 |
| B | 비만 (1) | 0.60 |
| C | 비만 아님 (0) | 0.30 |
| D | 비만 아님 (0) | 0.10 |

(계산에는 ln(0.80) ≈ -0.22, ln(0.60) ≈ -0.51, ln(0.70) ≈ -0.36, ln(0.90) ≈ -0.11를 사용해요)

**힌트:**
- 비만인 생쥐의 우도 = 예측 확률
- 비만이 아닌 생쥐의 우도 = 1 - 예측 확률
- 로그 우도 = 모든 개별 log(우도)의 합

<details>
<summary>예시 답안 보기</summary>

**각 생쥐의 우도 계산:**
- A(비만): 우도 = 0.80 → log(0.80) ≈ -0.22
- B(비만): 우도 = 0.60 → log(0.60) ≈ -0.51
- C(비만 아님): 우도 = 1 - 0.30 = 0.70 → log(0.70) ≈ -0.36
- D(비만 아님): 우도 = 1 - 0.10 = 0.90 → log(0.90) ≈ -0.11

**전체 로그 우도:**
```
로그 우도 = (-0.22) + (-0.51) + (-0.36) + (-0.11) = -1.20
```

이 후보 직선의 로그 우도는 -1.20이에요. 다른 후보 직선과 비교해서 가장 0에 가까운(높은) 로그 우도를 가진 직선을 최적 직선으로 선택해요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>

| 🚫 최소제곱법 불가 | 로그 오즈 스케일에서 데이터가 ±∞에 있어서 잔차 계산이 불가능해요 |
| 📊 우도(Likelihood) | 주어진 곡선이 실제 데이터를 얼마나 잘 설명하는지의 척도예요 |
| 🔢 우도 계산 | 비만=예측확률, 비만아님=1-예측확률. 전체 우도는 이들의 곱이에요 |
| 📉 로그 우도 | 우도의 곱 → 로그 우도의 합. 계산이 더 안정적이에요. 항상 음수예요 |
| 🎯 최적화 | 로그 우도가 최대(0에 가장 가까운)인 직선을 찾아요 |
| 🔄 알고리즘 | 직선을 영리하게 이동하며 최적값으로 수렴해요 (경사 상승법) |
