## 1부: 이탈도 잔차 (Deviance Residuals)

### 🤔 왜 이탈도 잔차가 필요한가요?

선형 회귀에서는 잔차가 있어요. 잔차를 시각화하면 어느 데이터 포인트가 이상치(Outlier)인지 확인할 수 있어요.

로지스틱 회귀에서는 진짜 잔차가 ±무한대라서 쓸 수 없어요. 대신 **이탈도 잔차(Deviance Residuals)** 가 비슷한 역할을 해요.

### 이탈도 잔차의 정의

**잔차 이탈도 = Σ (이탈도 잔차²)**

이탈도 잔차는 잔차 이탈도를 각 데이터 포인트로 "분해"한 것이에요.

```
잔차 이탈도 = 2 × Σ [log(LL_포화의 해당 항) - log(LL_제안의 해당 항)]
```

이 식에서 각 데이터 포인트에 해당하는 항을 따로 빼내면:

```
이탈도 잔차ᵢ = ±√(2 × |log(LL_포화ᵢ) - log(LL_제안ᵢ)|)
```

- 데이터 포인트가 곡선 **위**에 있으면 → **양수**
- 데이터 포인트가 곡선 **아래**에 있으면 → **음수**

> 💡 **비유:** 선형 회귀에서 잔차를 제곱해서 더하면 SS적합이 나오는 것처럼, 이탈도 잔차를 제곱해서 더하면 잔차 이탈도가 나와요. 그래서 "이탈도 잔차"라는 이름이에요.

### 이탈도 잔차의 활용 — 이상치 탐지

이탈도 잔차를 그래프로 그려서 이상치를 찾아요.

- 이탈도 잔차가 0 근처에 있으면 → 곡선이 이 점을 잘 설명해요
- 이탈도 잔차가 0에서 멀면 → 곡선이 이 점을 잘 설명하지 못해요 (이상치 후보)

```
그래프 해석:
이탈도 잔차
  +2 ─── 이상치 가능성 ─────── ●
  +1 ────────── ●
   0 ─── ● ─ ● ─── ●
  -1 ──────────── ●
  -2 ─── 이상치 가능성 ────── ●
```

R에서 출력하는 summary의 "Deviance Residuals" 섹션이 이걸 보여줘요. 이 값들이 0을 중심으로 균형 잡혀 있으면 좋은 신호예요.

## 2부: R에서 로지스틱 회귀 실행하기

### 전체 흐름

```
1. 데이터 불러오기 및 전처리
2. 결측값 처리
3. 데이터 불균형 확인
4. 로지스틱 회귀 실행
5. 결과 해석
6. 맥파든 R² 계산
7. 예측 확률 시각화
```

### 핵심 R 코드 설명

**1단계: 데이터 불러오기와 전처리**

UCI 심장 질환 데이터셋을 사용해요. 열 이름이 없어서 직접 지정해야 해요.

```r
# 데이터 읽기
url <- "https://..."
data <- read.csv(url, header = FALSE)

# 열 이름 지정
colnames(data) <- c("age", "sex", "cp", "trestbps", ..., "hd")

# 범주형 변수 변환
data$sex <- ifelse(data$sex == 0, "F", "M")
data$sex <- as.factor(data$sex)
data$hd <- ifelse(data$hd == 0, "healthy", "unhealthy")
data$hd <- as.factor(data$hd)
```

**이 코드가 하는 일:** 숫자로 저장된 성별(0/1)을 "F"/"M"으로, 심장 질환(0/1)을 "healthy"/"unhealthy"로 바꿔요. 범주형 변수임을 R에게 알려주는 `as.factor()`도 적용해요.

**2단계: 결측값 처리**

```r
# 결측값 있는 행 확인
nrow(data[!complete.cases(data), ])  # → 6행

# 결측값 있는 행 제거
data <- data[complete.cases(data), ]
# → 303 - 6 = 297행 남음
```

**이 코드가 하는 일:** 결측값(NA)이 있는 행을 찾고 제거해요. 더 많은 데이터가 있다면 결측값을 랜덤 포레스트 등으로 대체하는 것도 방법이에요.

**3단계: 데이터 불균형 확인**

```r
xtabs(~ hd + sex, data = data)
```

**이 코드가 하는 일:** 심장 질환 여부와 성별의 교차표를 보여줘요. 어느 한 그룹에만 양성이 몰려 있으면 모델이 편향될 수 있어요.

**4단계: 로지스틱 회귀 실행**

단순 모델 (변수 1개):
```r
logistic <- glm(hd ~ sex, data = data, family = "binomial")
summary(logistic)
```

복잡한 모델 (모든 변수):
```r
logistic_full <- glm(hd ~ ., data = data, family = "binomial")
summary(logistic_full)
```

**이 코드가 하는 일:** `glm()` 함수는 일반화 선형 모델을 실행해요. `family = "binomial"`이 로지스틱 회귀를 선택하는 옵션이에요. `hd ~ .`은 "hd를 나머지 모든 변수로 예측"이라는 뜻이에요.

**5단계: 결과 해석**

`summary()` 출력의 주요 부분:

```
Coefficients:
              Estimate Std. Error z value Pr(>|z|)
(Intercept)  -1.0438     0.3533  -2.955   0.003 **
sexM          1.2737     0.3561   3.577   0.0003 ***

Null deviance:     354.49  on 296  degrees of freedom
Residual deviance: 340.70  on 295  degrees of freedom
AIC: 344.70
```

**읽는 방법:**
- **계수(Estimate):** 로그 오즈 스케일의 계수예요
- **z value:** 왈드 검정 z-값이에요. 절댓값 > 2이면 유의미해요
- **Pr(>|z|):** p-값이에요. < 0.05이면 유의미해요
- **Null/Residual deviance:** 이탈도예요. 차이가 클수록 모델이 좋아요
- **AIC:** 모델 비교에 쓰는 지표예요. 낮을수록 좋아요

**6단계: 맥파든 R² 계산**

```r
# 로그 우도 추출
ll_null <- logistic$null.deviance / (-2)
ll_proposed <- logistic$deviance / (-2)

# R² 계산
mcfadden_r2 <- 1 - (ll_proposed / ll_null)
mcfadden_r2

# p-값 계산
chi_sq <- 2 * (ll_proposed - ll_null)
p_value <- pchisq(chi_sq, df = 1, lower.tail = FALSE)
p_value
```

**이 코드가 하는 일:** 이탈도를 -2로 나누면 로그 우도가 나와요. 이걸로 맥파든 R²와 p-값을 계산해요.

**7단계: 예측 확률 시각화**

```r
predicted_probs <- data.frame(
  probability = logistic$fitted.values,
  hd = data$hd
)

# 확률 낮은 순에서 높은 순으로 정렬
predicted_probs <- predicted_probs[order(predicted_probs$probability), ]
predicted_probs$rank <- 1:nrow(predicted_probs)

# ggplot으로 그래프 그리기
library(ggplot2)
ggplot(predicted_probs, aes(x = rank, y = probability, color = hd)) +
  geom_point(alpha = 0.7) +
  labs(x = "순위", y = "심장 질환 예측 확률")
```

**이 코드가 하는 일:** 각 환자의 예측 확률을 낮은 것부터 높은 것 순서로 정렬해서 점 그래프로 그려요. 건강한 환자는 낮은 확률에, 심장 질환 환자는 높은 확률에 주로 분포하면 좋은 모델이에요.

## 3부: ROC 곡선과 AUC

### 🤔 임계값을 어떻게 정해야 하나요?

로지스틱 회귀는 확률을 출력해요. 임계값을 0.5로 정하면 분류를 할 수 있어요. 그런데 0.5가 항상 최선일까요?

- 임계값을 낮추면 (예: 0.1) → 더 많은 것을 "양성"으로 분류 → 진양성 ↑, 위양성 ↑
- 임계값을 높이면 (예: 0.9) → 더 적은 것을 "양성"으로 분류 → 진양성 ↓, 위양성 ↓

임계값마다 혼동 행렬이 달라요. 이 모든 결과를 한눈에 볼 수 있는 도구가 **ROC 곡선**이에요.

### ROC 곡선의 구조

```
진양성률   1.0 ─┐
(민감도)       │  ╱─────
               │╱
          0.5 ─┤
               │╱
               │
          0.0 ─┴────────
               0.0  0.5  1.0
               위양성률 (1 - 특이도)
```

- **y축:** 진양성률(True Positive Rate) = 민감도(Sensitivity)
  - 실제 양성 중 올바르게 양성으로 분류된 비율
- **x축:** 위양성률(False Positive Rate) = 1 - 특이도
  - 실제 음성 중 잘못 양성으로 분류된 비율

### ROC 곡선 그리는 방법

임계값을 0에서 1로 올리면서 각 임계값에서의 (위양성률, 진양성률) 좌표를 찍어요.

```
임계값 = 모두 양성: (위양성률=1.0, 진양성률=1.0) → 우상단 점
임계값 높이기:      (0.75, 1.0)
임계값 높이기:      (0.50, 1.0)
임계값 높이기:      (0.50, 0.75)
임계값 높이기:      (0.25, 0.75)
임계값 높이기:      (0.00, 0.75)
임계값 = 모두 음성: (0.0, 0.0) → 원점
```

이 점들을 연결하면 ROC 곡선이 완성돼요.

### 대각선의 의미

y = x 대각선은 **무작위 분류기**의 결과예요. 진양성률 = 위양성률이면 동전 던지기와 같아요. ROC 곡선이 대각선보다 왼쪽 위에 있을수록 좋은 모델이에요.

### AUC (Area Under the Curve)

ROC 곡선 아래의 넓이예요.

| AUC 값 | 의미 |
|---|---|
| 0.5 | 무작위 분류 (동전 던지기) |
| 0.5 ~ 0.7 | 낮은 성능 |
| 0.7 ~ 0.9 | 괜찮은 성능 |
| 0.9 ~ 1.0 | 우수한 성능 |
| 1.0 | 완벽한 분류 |

**AUC로 모델 비교하기:**

```
모델 A (로지스틱 회귀): AUC = 0.90
모델 B (랜덤 포레스트): AUC = 0.85
→ 모델 A를 선택해요!
```

### R에서 ROC 그래프 그리기

```r
library(pROC)

# ROC 계산 및 그래프
roc_result <- roc(response = data$hd,       # 실제 라벨
                  predictor = logistic$fitted.values,  # 예측 확률
                  plot = TRUE,
                  legacy.axes = TRUE,       # x축: 위양성률 (0→1)
                  percent = TRUE,           # 백분율로 표시
                  xlab = "위양성 비율 (%)",
                  ylab = "진양성 비율 (%)",
                  col = "#377EB8",          # 색상
                  lwd = 4,                  # 선 두께
                  print.auc = TRUE)         # AUC 출력
```

**이 코드가 하는 일:** pROC 패키지의 roc() 함수로 ROC 곡선을 그려요. AUC를 자동으로 계산하고 그래프에 표시해요.

두 모델 비교하기:
```r
# 첫 번째 모델 ROC
roc(response = data$hd,
    predictor = logistic$fitted.values,
    plot = TRUE, percent = TRUE, col = "blue", print.auc = TRUE)

# 두 번째 모델 ROC (랜덤 포레스트 등)
plot.roc(data$hd,
         rf_predictions,
         percent = TRUE, col = "green",
         add = TRUE,                 # 기존 그래프에 추가
         print.auc = TRUE,
         print.auc.y = 40)          # AUC 텍스트 위치
```

## 4부: 정밀도-재현율 곡선 (Precision-Recall Curve)

### 🤔 ROC 대신 언제 쓰나요?

ROC의 위양성률(False Positive Rate) 계산에는 **진음성(True Negative)** 이 들어가요.

데이터가 심각하게 불균형한 경우(예: 희귀 질환 연구에서 환자가 100명, 건강한 사람이 10,000명), 진음성이 매우 많아서 위양성률이 항상 낮게 나와요. 이 경우 ROC 곡선이 실제보다 더 좋아 보일 수 있어요.

이럴 때 **정밀도-재현율 곡선(Precision-Recall Curve)** 이 더 적합해요.

### 정밀도(Precision)란?

```
정밀도 = 진양성 / (진양성 + 위양성)
       = 양성으로 분류된 것 중 실제 양성의 비율
```

- **높은 정밀도:** 양성이라고 했을 때 실제로 양성일 확률이 높아요
- 진음성을 포함하지 않아서 불균형 데이터에 적합해요

### ROC vs 정밀도-재현율

| 상황 | 추천 지표 |
|---|---|
| 데이터가 균형 잡힘 | ROC / AUC |
| 데이터가 심각하게 불균형 (희귀 질환 등) | 정밀도-재현율 곡선 |
| 위양성 최소화가 중요 | 두 지표 모두 확인 |

## ⚠️ 주의할 점 / 자주 하는 실수

- **이탈도 잔차가 너무 치우쳐 있으면 문제예요.** 중앙값이 0에 가깝고 최솟값/최댓값이 너무 크지 않아야 해요.
- **AUC가 높아도 임계값 선택이 중요해요.** AUC는 모든 임계값의 평균적 성능이에요. 실제 배포할 임계값은 목적에 따라 신중히 선택해야 해요.
- **로지스틱 회귀 출력의 "Deviance Residuals"는 이상치 확인에 써요.** 이 섹션이 0 중심으로 대칭이면 좋은 신호예요.
- **ROC 곡선은 임계값 중립적이에요.** 어떤 임계값이 좋은지는 알려주지 않고, 가능한 임계값 전체의 성능을 보여줘요. 최적 임계값은 목적에 따라 선택해요.
- **AUC = 0.9의 "랜덤 포레스트"가 AUC = 0.85의 "로지스틱 회귀"보다 무조건 좋은 건 아니에요.** 해석 가능성, 계산 비용, 데이터 크기 등 다른 요소도 고려해야 해요.

**Q1.** 이탈도 잔차(Deviance Residuals)가 선형 회귀의 잔차와 유사한 역할을 한다고 했어요. 어떤 역할인가요?

<details>
<summary>정답 보기</summary>

**정답:** 각 데이터 포인트가 모델 전체 오차에 얼마나 기여하는지를 나타내고, 이를 통해 이상치(Outlier)를 찾아요.

**설명:** 선형 회귀에서 잔차를 제곱해서 더하면 SS적합이 나오는 것처럼, 이탈도 잔차를 제곱해서 더하면 잔차 이탈도가 나와요. 그래서 이탈도 잔차가 큰 데이터 포인트는 모델이 잘 설명하지 못하는 이상치 후보예요. 이런 점을 발견하면 데이터 입력 오류가 있는지 확인해야 해요.

</details>

---

**Q2.** ROC 곡선에서 이상적인 곡선은 어느 방향으로 쏠려 있어야 하나요?

<details>
<summary>정답 보기</summary>

**정답:** 왼쪽 위 방향으로 쏠려 있어야 해요.

**설명:** 이상적인 모델은 위양성률은 낮고(x축 값 작음) 진양성률은 높아요(y축 값 큼). 그래프에서 왼쪽 위 모서리에 가까울수록 좋은 모델이에요. AUC는 이 곡선 아래 넓이인데, 왼쪽 위로 쏠릴수록 넓이가 1에 가까워져요. 대각선(무작위 분류)은 AUC = 0.5예요.

</details>

---

**Q3.** AUC = 0.5라는 것은 어떤 의미인가요?

<details>
<summary>정답 보기</summary>

**정답:** 무작위 분류기와 동일한 성능이에요. 동전 던지기 수준이에요.

**설명:** AUC = 0.5이면 ROC 곡선이 y = x 대각선과 일치해요. 위양성률과 진양성률이 항상 같은 거예요. 어떤 임계값을 써도 맞춘 것만큼 틀리는 거예요. 사실상 아무것도 예측하지 못하는 모델이에요.

</details>

---

**Q4.** 데이터가 심각하게 불균형(희귀 질환: 환자 50명 vs 건강인 5,000명)일 때 ROC 대신 정밀도-재현율 곡선을 써야 하는 이유는 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** ROC의 위양성률은 진음성 수에 크게 영향받는데, 불균형 데이터에서는 진음성이 매우 많아서 위양성률이 항상 낮게 나와요. 이 때문에 ROC가 실제보다 더 좋아 보여요.

**설명:** 건강인 5,000명 중 50명을 잘못 양성으로 분류해도 위양성률 = 50/5,000 = 1%로 아주 낮아요. ROC에서는 좋아 보이지만 실제로는 나쁜 분류예요. 정밀도는 "양성으로 분류된 것 중 진짜 양성"이라서 진음성에 영향을 받지 않아요. 희귀 질환 진단처럼 위양성의 비용이 큰 경우에는 정밀도가 더 중요한 지표예요.

</details>

---

**Q5.** R의 glm() 출력에서 Null deviance와 Residual deviance의 차이가 크면 어떤 의미인가요?

<details>
<summary>정답 보기</summary>

**정답:** 모델이 영 모델보다 유의미하게 더 잘 설명한다는 뜻이에요. 즉, 사용한 변수들이 결과를 예측하는 데 도움이 된다는 거예요.

**설명:** Null deviance는 영 모델(변수 없이 전체 확률만 사용)의 이탈도예요. Residual deviance는 제안 모델의 이탈도예요. 차이가 클수록 모델에 포함된 변수들이 예측력을 많이 높였다는 뜻이에요. 이 차이가 카이제곱 통계량이 되어서 p-값으로 이어져요.

</details>

### 실전 적용 문제

**문제.** 의료 AI를 개발하는 팀에서 코로나 바이러스 감염 여부를 예측하는 두 모델을 비교하고 있어요.

- **모델 A:** AUC = 0.88, 임계값 0.5에서 민감도 = 80%, 특이도 = 82%
- **모델 B:** AUC = 0.85, 임계값 0.5에서 민감도 = 92%, 특이도 = 70%

1. AUC만 보면 어떤 모델이 더 좋은가요?
2. 감염자를 절대 놓치면 안 되는 상황(예: 격리 판단)이라면 어떤 모델이 더 적합한가요?
3. 불필요한 격리를 최소화하는 것이 목표라면 어떤 모델이 더 적합한가요?

**힌트:**
- AUC는 전반적 성능, 민감도는 양성 탐지 능력, 특이도는 음성 판별 능력이에요
- 상황에 따라 어떤 지표가 더 중요한지 달라요

<details>
<summary>예시 답안 보기</summary>

**1. AUC만 보면 모델 A가 더 좋아요.** (0.88 > 0.85)

**2. 감염자를 절대 놓치면 안 되는 상황 → 모델 B**
민감도 92%가 더 높아요. 코로나 양성인 사람 중 92%를 올바르게 양성으로 분류해요. 모델 A는 80%만 잡아내서 20%는 놓쳐요. 감염자를 놓치면 격리 없이 돌아다니다 확산될 수 있어요. 이 상황에서는 위양성(건강한 사람이 격리되는 것)이 위음성(감염자를 놓치는 것)보다 덜 나빠요.

**3. 불필요한 격리 최소화 → 모델 A**
특이도 82%가 더 높아요. 음성인 사람 중 82%를 올바르게 음성으로 분류해요. 모델 B는 70%만 맞춰서 30%는 잘못 양성으로 분류돼요. 의료 시스템 과부하나 개인 불편을 줄이려면 위양성을 최소화하는 게 중요해요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>
