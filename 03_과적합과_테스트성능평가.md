# ⚖️ 과적합과 테스트 성능 평가

---

## 1. 과적합 (Overfitting)

> **훈련 데이터의 잡음까지 전부 외워버려서, 훈련에서는 성능이 좋지만 새로운 데이터에서는 성능이 나쁜 현상**

---

### 직관적 이해

학생이 시험 공부를 할 때를 생각해보세요:

```
❌ 과적합 학생:
- 작년 기출문제를 통째로 암기함
- 작년 문제는 100점
- 올해 새로운 문제는 50점
→ "공부"한 게 아니라 "암기"한 것

✅ 일반화가 잘 된 학생:
- 기출문제로 개념과 원리를 이해함
- 작년 문제도 90점
- 올해 새로운 문제도 88점
→ 진짜 학습이 된 것
```

---

### 과적합의 원인

모델이 너무 복잡할 때 발생합니다:

- **데이터의 진짜 패턴**뿐만 아니라
- **우연한 잡음(Noise)**까지 학습해버림

```
예시: 집값 데이터
- 진짜 패턴: 평수가 클수록 집값이 높다
- 잡음: 특정 날짜에 비가 왔을 때 집값이 낮았다 (우연)

과적합된 모델은 "비 오는 날 = 집값 낮음"이라는 규칙까지 학습해버림
→ 새로운 데이터에서 이 규칙이 맞지 않아 성능 저하
```

---

### 과적합의 특징

#### 1. 표본 의존성 & 불안정성

- 훈련 데이터의 일부 표본에만 최적화되어 있음
- **샘플 몇 개만 바뀌어도 예측이 크게 흔들림** (분산 상승)

#### 2. 일반화 실패

- 훈련 데이터 성능: 매우 높음 (거의 100%)
- 테스트 데이터 성능: 현저히 낮음

```
훈련 오류: 2%  (훈련 데이터에서의 오류)
테스트 오류: 30% (새로운 데이터에서의 오류)
                ↑
           이 차이가 크면 과적합!
```

---

### ⚠️ 주의: 과적합 ≠ 분포 변화 오류

과적합과 비슷해 보이지만 **전혀 다른 개념**이 있습니다:

| 구분 | 과적합 (Overfitting) | 분포 변화 (Distribution Shift) |
|------|---------------------|-------------------------------|
| **원인** | 모델이 훈련 데이터에 과도하게 맞춰짐 | 훈련/테스트 데이터의 분포가 다름 |
| **예시** | 훈련 사진만 외워버림 | 한국인 사진으로 학습 → 유럽인 사진 테스트 |
| **해결책** | 단순한 모델, 더 많은 데이터 | 새로운 분포의 데이터 수집 |

> 분포 변화로 인한 오류는 **모델이 과적합하지 않아도** 얼마든지 발생할 수 있습니다.

---

## 2. 과적합 vs 과소적합 (Underfitting)

```
너무 단순한 모델                    너무 복잡한 모델
(과소적합, Underfitting)          (과적합, Overfitting)
        
훈련 오류: 높음                    훈련 오류: 매우 낮음
테스트 오류: 높음                  테스트 오류: 매우 높음
        
문제: 패턴 자체를 못 잡음          문제: 잡음까지 다 외워버림
```

### 비유로 이해하기

| 상황 | 과소적합 | 적절한 모델 | 과적합 |
|------|---------|------------|--------|
| **복잡도** | 너무 단순 | 적절 | 너무 복잡 |
| **훈련 성능** | 낮음 | 높음 | 매우 높음 |
| **테스트 성능** | 낮음 | 높음 | 낮음 |
| **예시** | 직선 하나로 모든 데이터 설명 | 적절한 곡선 | 모든 점을 통과하는 복잡한 곡선 |

---

### 해결책

1. **더 많은 데이터 확보** → 잡음의 영향력 감소
2. **모델 복잡도 조절** → 과적합이면 단순하게, 과소적합이면 복잡하게
3. **교차 검증(Cross Validation)** → 테스트 데이터를 활용한 모델 평가

---

## 3. 훈련 오류 vs 테스트 오류

### 훈련 오류 (Training Error)

> **모델을 학습시킨 데이터에 다시 적용해 계산한 오류**

- 문제: 이미 본 데이터에 대한 성능이라 실제 성능보다 **낙관적**
- 과적합된 모델은 훈련 오류가 거의 0에 가깝지만, 실제로는 쓸모없을 수 있음

### 테스트 오류 (Test Error)

> **학습에 쓰지 않은 새로운 데이터에 대한 평균 예측 오류**

- 모델의 **실제 성능**을 나타내는 진짜 지표
- 훈련 오류가 낮다고 테스트 오류가 낮을 것이라 보장할 수 없음!

```
훈련 오류 낮음 ≠ 테스트 오류 낮음
               ↓
   훈련 오류만 보면 속는다!
```

---

## 4. 재표본화 (Resampling)를 통한 오류 추정

### 이상적인 상황

별도의 테스트 데이터셋이 충분히 크다면 가장 좋겠지만... **현실에서는 데이터가 부족합니다.**

### 해결책: 재표본화 (Resampling)

> **가지고 있는 데이터를 나누어, 여러 번 훈련→평가를 반복해 오류를 추정하는 방법**

장점: **별도의 테스트 데이터 없이도 효율적으로 오차 추정 가능!**

---

## 5. 검증셋 (Validation Set / Hold-out)

> **가용 샘플을 무작위로 훈련셋과 검증셋으로 나누는 가장 단순한 방법**

```
전체 데이터
├── 훈련셋 (예: 70%) → 모델 학습에 사용
└── 검증셋 (예: 30%) → 학습된 모델의 성능 평가

훈련셋으로 모델 학습 → 검증셋으로 검증 오류 측정
```

### 검증 오류 측정 방법

- **회귀 문제:** MSE (평균 제곱 오차)
- **분류 문제:** 오분류율, F1-Score

---

### 검증셋의 단점

#### 1. 결과의 가변성

어떤 표본이 훈련/검증셋에 들어가느냐에 따라 테스트 오류 추정치가 크게 달라질 수 있습니다.

```
분할 시도 1: 훈련 70% + 검증 30% → 테스트 오류 추정: 15%
분할 시도 2: 다른 무작위 분할  → 테스트 오류 추정: 23%
                               ↑
                       같은 모델인데 결과가 다름!
```

#### 2. 테스트 오류 과대 추정

훈련셋(전체의 일부)만으로 모델을 학습하기 때문에:

- 전체 데이터로 학습했을 때보다 성능이 낮게 나옴
- → **실제 모델의 성능을 과소평가 (테스트 오류를 과대 추정)**

---

## 6. K-fold 교차 검증 (K-fold Cross-Validation)

> **테스트 오류 추정의 표준적인 접근 방식**

---

### 기본 아이디어

데이터 전체를 K개로 나눠서, **각각 돌아가며 검증셋이 되도록** 반복합니다.

```
K = 5인 경우 (5-fold Cross Validation):

1번 실험: [검증] [훈련] [훈련] [훈련] [훈련]  → MSE₁
2번 실험: [훈련] [검증] [훈련] [훈련] [훈련]  → MSE₂
3번 실험: [훈련] [훈련] [검증] [훈련] [훈련]  → MSE₃
4번 실험: [훈련] [훈련] [훈련] [검증] [훈련]  → MSE₄
5번 실험: [훈련] [훈련] [훈련] [훈련] [검증]  → MSE₅

최종 테스트 오류 = (MSE₁ + MSE₂ + MSE₃ + MSE₄ + MSE₅) / 5
```

---

### 수식

폴드 집합 $C_1, ..., C_K$, 각 폴드 크기가 $n_k$일 때:

$$CV(K) = \sum_{k=1}^{K}\frac{n_k}{n}MSE_k$$

(각 폴드의 크기 비율을 가중치로 하여 MSE의 가중 평균을 계산)

---

### K-fold 교차 검증의 단계

1. **데이터 셔플링** → 무작위로 섞기 (순서 편향 제거)
2. **K개 그룹으로 분할** → 보통 K = 5 또는 K = 10
3. **반복 실험** → 각 그룹이 한 번씩 검증셋이 되도록 K번 반복
4. **평균 오류 계산** → K개의 MSE를 평균내어 최종 테스트 오류 추정

---

### 검증셋 방법 vs K-fold 비교

| 항목 | 검증셋 (Hold-out) | K-fold CV |
|------|------------------|-----------|
| **데이터 활용** | 일부만 훈련에 사용 | 전체를 균등하게 활용 |
| **결과 안정성** | 분할 방법에 따라 크게 변함 | 안정적 |
| **계산 비용** | 빠름 (1번만 학습) | K배 느림 |
| **오류 추정** | 과대 추정 경향 | 더 정확한 추정 |
| **권장 상황** | 데이터가 매우 많을 때 | 일반적인 경우 |

---

## 7. LOOCV (Leave-One-Out Cross-Validation)

> **K-fold에서 K = n (데이터 수)인 극단적인 경우**

---

### 어떻게 동작하나?

```
데이터가 100개라면:

1번: 데이터 1개를 검증, 나머지 99개를 훈련 → MSE₁
2번: 데이터 2개를 검증, 나머지 99개를 훈련 → MSE₂
...
100번: 데이터 100개를 검증, 나머지 99개를 훈련 → MSE₁₀₀

최종: 100개의 MSE 평균
```

---

### LOOCV의 특징

#### 장점
- 거의 모든 데이터를 훈련에 사용 → 편향이 매우 낮음
- 모든 데이터를 검증에 활용 → 결과가 매우 안정적

#### 단점
- **연산량이 K = n배** → 데이터가 많으면 극도로 느림
- 100개의 데이터면 100번 모델을 학습해야 함

---

### LOOCV vs K-fold 언제 무엇을 쓸까?

```
데이터가 적은 경우 (100개 미만):
→ LOOCV 고려 가능 (모든 데이터를 최대한 활용)

데이터가 중간인 경우 (수백~수천 개):
→ K = 10 또는 K = 5 권장

데이터가 매우 많은 경우 (수만 개 이상):
→ 단순 검증셋 방법도 충분

💡 LOOCV와 K-fold의 성능 향상이 크지 않다면, 
   적절한 K를 선택한 K-fold를 사용하는 것이 좋습니다.
```

---

## 8. 전체 정리

```
모델 학습 및 평가 전략:

데이터
  ↓
[훈련셋 | 검증셋] ← Hold-out 방법
  또는
[폴드1 | 폴드2 | ... | 폴드K] ← K-fold 방법
  또는
[하나씩 떼어놓기] ← LOOCV
  ↓
여러 번 학습 → 여러 MSE 계산
  ↓
평균 테스트 오류 추정
  ↓
모델의 실제 성능 파악 → 더 좋은 모델 선택
```

> **핵심 메시지:** 
> 훈련 오류만 보면 안 됩니다.
> 교차 검증을 통해 처음 보는 데이터에 대한 성능을 정확히 추정해야
> 실제로 잘 동작하는 모델을 만들 수 있습니다.
