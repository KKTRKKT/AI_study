### 1차 — 문제 정의
- 무엇을 예측/분류할 건지 명확히 정해요
- 평가 지표 결정 (정확도, F1, RMSE 등)
- 성공 기준 수치 설정 ("F1 0.85 이상이면 성공" 같은 식으로)

> ⚠️ **이게 없으면 나중에 결과가 좋은지 나쁜지 판단을 못 해요. 가장 먼저 해야 해요.**

---

### 2차 — 데이터 현황 파악
- 데이터 양 확인
- 품질 확인 (결측값, 중복, 이상치, 클래스 불균형)
- 사용 가능 여부 판단 → 추가 수집 필요한지 결정

---

### 3차 — 데이터 전처리
- 정제 (결측값 처리, 이상치 제거/보정)
- 변환 (정규화, 인코딩)
- 분할 (train / validation / test)

> ⚠️ **전처리 없이 모델 먼저 돌리면, 결과가 나빠도 모델 탓인지 데이터 탓인지 구분이 안 돼요.**

---

### 4차 — 모델 후보 선정
- 문제 유형에 맞는 모델 탐색
- 데이터 규모에 맞는 모델 크기 고려
- 베이스라인 모델 설정 (비교 기준점)

---

### 5차 — 소규모 파일럿 테스트
- 데이터 일부(10~20%)로 모델 후보 빠르게 비교
- 이 단계는 **기본값 그대로** 사용해요 — 옵티마이저/규제 건드리지 않아요
- 방향이 맞는지만 확인하는 단계예요
- 결과 기록 후 모델 최종 선택

---

### 6차 — 하이퍼파라미터 설정
- **규제 방식 결정** (L1 / L2 / 없음)
  - 과적합 징후가 보이면 반드시 설정해요
  - 설정 안 하면 훈련 데이터엔 잘 맞는데 새 데이터에선 망하는 상황이 생겨요
- **옵티마이저 및 학습률 설정**
  - 기본값이 항상 최선은 아니에요
  - 학습이 너무 느리거나, 중간에 멈추거나, 수렴을 못 하면 여기가 원인이에요
- validation 성능 보면서 조정하고, 설정값 반드시 기록해요

> 💡 **설정값을 기록 안 하면 나중에 재현이 안 돼요. 어떤 값으로 돌렸는지 항상 남겨요.**

---

### 7차 — 전체 데이터로 본 실험
- 6차에서 결정된 설정으로 전체 데이터 학습
- 실험 조건 고정 (랜덤 시드 포함)

---

### 8차 — 결과 분석 및 판단
- 1차에서 정한 평가 지표로 성공 기준 달성 여부 확인
- 실패 사례 분석 (오분류, 오차 큰 샘플 등)
- 재실험 필요 여부 결정 → 필요하면 6차부터 반복

---

### 9차 — 결과 정리 및 문서화
- 실험 조건, 설정값, 결과, 결론 기록
- 다음 단계 또는 배포 여부 결정
