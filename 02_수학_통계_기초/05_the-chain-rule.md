# 연쇄 법칙 (The Chain Rule)

---

## 📌 이 문서에서 배우는 것

- 연쇄 법칙(Chain Rule)이 왜 필요한지 이해해요
- 두 식이 연결된 경우 미분을 어떻게 하는지 알아요
- 눈에 보이지 않는 연결고리를 찾아 연쇄 법칙을 적용하는 방법을 알아요
- 머신러닝의 손실 함수(잔차 제곱합)에 연쇄 법칙이 어떻게 적용되는지 이해해요
- 역전파(Backpropagation) 이해의 기반을 마련해요

---

## 용어 정리

| 용어 | 원문 | 한 줄 설명 |
|---|---|---|
| 미분 | Derivative | 어떤 함수가 입력에 따라 얼마나 빠르게 변하는지를 나타내요 |
| 연쇄 법칙 | Chain Rule | 중간 변수로 연결된 함수를 미분하는 방법이에요 |
| 기울기 | Slope | 그래프에서 얼마나 가파른지를 나타내요. 미분값과 같아요 |
| 잔차 | Residual | 실제 관측값과 모델 예측값의 차이예요 |
| 잔차 제곱합 | Residual Sum of Squares | 모든 잔차를 제곱해서 더한 것이에요. 모델의 오차를 나타내요 |
| 손실 함수 | Loss Function | 모델이 얼마나 틀렸는지를 측정하는 함수예요 |
| 역전파 | Backpropagation | 신경망에서 연쇄 법칙을 이용해 기울기를 계산하는 방법이에요 |

## 1. 미분이 뭔지 잠깐 복습해요

### 미분의 핵심 아이디어

> 💡 **비유:** 차를 타고 가다가 속도계를 봤어요. "시속 100km"는 현재 이 순간, 차가 얼마나 빠르게 움직이는지를 나타내요. 미분도 이와 같아요. "이 점에서 함수가 얼마나 빠르게 변하는가"예요.

어떤 데이터에 포물선(y = x²)을 맞췄다고 해요. 이 포물선의 **기울기**는 위치에 따라 달라져요. 어떤 점에서든 기울기를 구하는 방법이 미분이에요.

**거듭제곱 법칙 (Power Rule):**
```
f(x) = x²  →  f'(x) = 2x
f(x) = x³  →  f'(x) = 3x²
f(x) = xⁿ  →  f'(x) = n × x^(n-1)
```

이 문서에서는 미분 자체보다 **연쇄 법칙**에 집중할게요.

## 2. 연쇄 법칙이 필요한 상황

### 🤔 왜 필요한가요?

때로는 A → B → C처럼 두 함수가 연결돼 있어요. "A가 변하면 C가 얼마나 변하는가"를 알고 싶을 때, 직접 미분하기 어려운 경우가 많아요. 이때 연쇄 법칙을 사용해요.

### 한 줄 정의

> **연쇄 법칙(Chain Rule)이란:** A → B → C처럼 중간 변수(B)를 통해 연결된 함수에서, "A의 변화가 C에 미치는 영향"을 중간 단계의 미분들의 곱으로 구하는 방법이에요.

```
dC/dA = dC/dB × dB/dA
```

**이 수식이 말하는 것:**
C가 A에 대해 얼마나 빠르게 변하는지 = (C가 B에 대해 얼마나 빠르게 변하는지) × (B가 A에 대해 얼마나 빠르게 변하는지)

---

## 3. 가장 단순한 예시 — 몸무게, 키, 신발 크기

### 상황 설정

두 가지 직선 관계가 있어요.

**관계 1: 몸무게 → 키**
- 몸무게가 1단위 증가할 때마다 키가 2단위 증가해요
- 기울기(미분값) = 2
- 공식: 키 = 2 × 몸무게

**관계 2: 키 → 신발 크기**
- 키가 1단위 증가할 때마다 신발 크기가 1/4단위 증가해요
- 기울기(미분값) = 1/4
- 공식: 신발 크기 = (1/4) × 키

### 연쇄 법칙 적용

"몸무게가 1단위 증가하면 신발 크기가 얼마나 변할까요?"

직접 공식을 만들 수 있어요:
```
신발 크기 = (1/4) × 키
           = (1/4) × (2 × 몸무게)
           = (1/2) × 몸무게
```

그래서 몸무게 1단위 증가 → 신발 크기 1/2단위 증가예요.

**연쇄 법칙으로도 같은 결과가 나와요:**
```
d(신발 크기)/d(몸무게)
= d(신발 크기)/d(키) × d(키)/d(몸무게)
= (1/4) × 2
= 1/2
```

> 💡 **핵심:** 연쇄 법칙은 "중간 변수(키)를 통한 두 미분의 곱"이에요. 키가 몸무게와 신발 크기를 연결하는 다리 역할이에요.

## 4. 조금 더 복잡한 예시 — 배고픔, 시간, 아이스크림

### 상황 설정

두 가지 비선형 관계가 있어요.

**관계 1: 마지막 간식으로부터 시간(t) → 배고픔**
- 배고픔 = t² + 0.5
- 시간이 지날수록 기하급수적으로 배가 고파요

**관계 2: 배고픔 → 아이스크림 먹고 싶은 정도**
- 아이스크림 욕구 = √배고픔
- 배가 많이 고플수록 욕구가 커지지만, 일정 이상이면 증가 속도가 느려져요

### 직접 공식으로 합치면?

```
아이스크림 욕구 = √(t² + 0.5)
```

이 식을 직접 미분하려면... 머리가 아파요. 연쇄 법칙을 쓰면 훨씬 쉬워요.

### 연쇄 법칙 적용

**단계 1:** 각각을 따로 미분해요

```
d(아이스크림)/d(배고픔) = 1 / (2 × √배고픔)    (거듭제곱 법칙)
d(배고픔)/dt           = 2t                    (거듭제곱 법칙)
```

**단계 2:** 연쇄 법칙으로 곱해요

```
d(아이스크림)/dt = d(아이스크림)/d(배고픔) × d(배고픔)/dt
                = [1 / (2 × √배고픔)] × 2t
                = t / √배고픔
                = t / √(t² + 0.5)    ← 배고픔 식을 대입해요
```

**이 결과가 말하는 것:**
마지막 간식으로부터 시간이 t만큼 지났을 때, 아이스크림 욕구가 얼마나 빠르게 변하는지를 나타내요.

> 💡 **비유:** 두 개의 기어가 연결된 기계를 생각해봐요. 첫 번째 기어가 돌면 두 번째 기어가 따라서 돌아요. "첫 번째 기어 1회전이 최종 출력축에 미치는 영향"을 구할 때, 중간 기어비를 모두 곱하는 것과 같아요. 연쇄 법칙은 이런 "연결된 변화율의 곱셈"이에요.

## 5. 수식이 하나로 합쳐진 경우

실제 머신러닝에서는 두 함수가 따로 주어지지 않고 이미 합쳐진 형태로 나올 때가 많아요.

### 같은 문제, 합쳐진 형태

```
아이스크림 욕구 = √(t² + 0.5)
```

여기서 "숨겨진 연결고리"를 찾아야 해요. 그 연결고리는 괄호 안의 내용이에요.

**"괄호 안의 내용"을 중간 변수로 설정해요:**
```
내부 = t² + 0.5
아이스크림 욕구 = √(내부)
```

**연쇄 법칙:**
```
d(아이스크림)/dt = d(아이스크림)/d(내부) × d(내부)/dt
                = [1/(2√내부)] × 2t
                = t / √(t² + 0.5)
```

똑같은 결과가 나왔어요.

> 💡 **팁:** 수식이 합쳐진 형태에서 연쇄 법칙을 적용하는 핵심 요령은 **"괄호로 묶을 수 있는 부분"을 찾는 거예요.** 그 부분이 중간 변수가 되어요.

## 6. 머신러닝에서의 연쇄 법칙 — 잔차 제곱합

### 🤔 왜 머신러닝에서 중요한가요?

머신러닝에서 모델을 학습한다는 것은 "모델의 오차를 최소화하는 파라미터를 찾는다"는 뜻이에요. 그 오차를 측정하는 대표적인 함수가 **잔차 제곱합(Residual Sum of Squares)** 이에요.

잔차 제곱합을 최소화하려면 그것이 파라미터에 대해 어떻게 변하는지(미분)를 알아야 해요. 여기서 연쇄 법칙이 필요해요.

### 상황 설정

몸무게(x)와 키(y) 데이터에 직선을 맞추려고 해요.

```
직선 방정식: 예측 키 = 절편(intercept) + 1 × 몸무게
```

지금은 기울기는 1로 고정하고 **절편만 바꿀 수 있어요.** 최적의 절편을 찾고 싶어요.

### 연결 구조 파악

```
절편 → 잔차(실제 - 예측) → 잔차² → 잔차 제곱합
```

잔차가 절편과 잔차 제곱합을 연결하는 **중간 변수**예요.

### 연쇄 법칙 적용

**목표:** d(잔차²) / d(절편) = ?

```
d(잔차²)/d(절편) = d(잔차²)/d(잔차) × d(잔차)/d(절편)
```

**각 미분 계산:**

1. **d(잔차²)/d(잔차):**
잔차² 를 잔차로 미분하면 거듭제곱 법칙에 의해:
```
d(잔차²)/d(잔차) = 2 × 잔차
```

2. **d(잔차)/d(절편):**
잔차 = 실제 키 - (절편 + 1 × 몸무게)
절편에 대해 미분하면:
- "실제 키"는 절편과 무관 → 미분 = 0
- "-(절편)"의 미분 = -1
- "-1 × 몸무게"는 절편과 무관 → 미분 = 0
```
d(잔차)/d(절편) = -1
```

**연쇄 법칙으로 합치기:**
```
d(잔차²)/d(절편) = 2 × 잔차 × (-1) = -2 × 잔차
```

### 최적의 절편 찾기

잔차 제곱합이 최솟값이 되려면 미분이 0이어야 해요.

```
-2 × 잔차 = 0
잔차 = 0
실제 키 - (절편 + 1 × 몸무게) = 0
절편 = 실제 키 - 1 × 몸무게
```

실제 데이터를 넣어서 풀면 절편의 최적값이 나와요.

> 💡 **이것이 왜 중요한가요?** 이 계산이 바로 선형 회귀에서 기울기와 절편을 찾는 과정의 핵심이에요. 더 복잡한 신경망에서도 똑같이 연쇄 법칙을 사용해요. 다만 훨씬 더 많은 파라미터에 대해 반복적으로 계산할 뿐이에요. 그 과정이 **역전파(Backpropagation)** 이에요.

## 7. 신경망과 역전파의 연결

### 연쇄 법칙 → 역전파

신경망은 여러 층이 연결된 구조예요.

```
입력 → 1층 → 2층 → 3층 → 출력 → 손실
```

손실을 줄이려면 각 층의 파라미터(가중치)가 손실에 얼마나 영향을 주는지 알아야 해요. 이걸 계산하는 것이 역전파예요.

역전파의 핵심은 연쇄 법칙이에요:
```
d(손실)/d(1층 가중치) = d(손실)/d(3층) × d(3층)/d(2층) × d(2층)/d(1층) × d(1층)/d(가중치)
```

출력 층부터 입력 층으로 거꾸로(Back) 미분을 **전파(Propagation)** 하기 때문에 역전파예요.

---

## ⚠️ 주의할 점 / 자주 하는 실수

- **연쇄 법칙은 중간 변수가 있을 때만 쓰는 게 아니에요.** 합쳐진 형태의 복잡한 함수에도 똑같이 적용해요. 괄호를 활용해서 중간 변수를 만들어내면 돼요.
- **미분 순서를 잘 지켜야 해요.** dC/dA를 구할 때 "dC/dB × dB/dA"이지, "dB/dA × dC/dB"가 아니에요. (곱셈이라 순서가 달라도 값은 같지만, 개념적으로 올바른 순서를 이해하는 게 중요해요.)
- **각 미분을 단독으로 먼저 구하고 나서 곱해요.** 두 미분을 동시에 구하려 하면 헷갈려요.
- **신경망에서는 같은 중간 변수가 여러 경로에서 나올 수 있어요.** 그런 경우엔 모든 경로의 연쇄 법칙 결과를 더해야 해요.

**Q2.** "몸무게 → 키 → 신발 크기" 예시에서, 몸무게가 1단위 변할 때 신발 크기가 1/2단위 변하는 이유를 연쇄 법칙으로 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:**

d(신발 크기)/d(몸무게) = d(신발 크기)/d(키) × d(키)/d(몸무게) = (1/4) × 2 = 1/2

**설명:** 몸무게가 1 증가하면 키가 2 증가해요(미분 = 2). 키가 1 증가하면 신발 크기가 1/4 증가해요(미분 = 1/4). 연결된 두 변화율을 곱하면 1/4 × 2 = 1/2이에요. 몸무게 1단위 변화가 신발 크기에는 1/2단위 변화를 만들어요.

</details>

**Q3.** 합쳐진 형태의 식 f(x) = (3x + 2)⁴에 연쇄 법칙을 적용하는 방법을 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 괄호 안의 내용(3x + 2)을 중간 변수 u로 설정하면 f = u⁴이 돼요. 그러면:
- df/du = 4u³ (거듭제곱 법칙)
- du/dx = 3

연쇄 법칙 적용:
df/dx = df/du × du/dx = 4u³ × 3 = 12u³ = 12(3x+2)³

**설명:** 합쳐진 형태에서는 "괄호로 묶을 수 있는 내부"를 중간 변수 u로 설정하는 게 핵심이에요. 그러면 바깥 함수(u⁴)와 안 함수(3x+2)를 각각 미분한 후 곱하면 돼요.

</details>

**Q4.** 잔차 제곱합에서 "잔차"가 중간 변수 역할을 한다고 했어요. 왜 잔차가 중간 변수인가요?

<details>
<summary>정답 보기</summary>

**정답:** 잔차는 절편(파라미터)에 의해 결정되고, 잔차 제곱합은 잔차에 의해 결정되기 때문이에요. 절편 → 잔차 → 잔차 제곱합의 연결 구조에서 잔차가 중간을 연결하는 변수예요.

**설명:** 절편이 변하면 예측값이 변하고, 예측값이 변하면 잔차(실제-예측)가 변하고, 잔차가 변하면 잔차²가 변해요. "절편이 잔차 제곱합에 미치는 영향"을 직접 구하기 어렵지만, "절편→잔차"와 "잔차→잔차²" 두 단계로 나눠서 연쇄 법칙을 적용하면 쉬워져요.

</details>

**Q5.** 역전파(Backpropagation)가 연쇄 법칙과 어떤 관계인지 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 역전파는 신경망에서 손실 함수를 각 층의 파라미터로 미분할 때 연쇄 법칙을 반복적으로 적용하는 알고리즘이에요.

**설명:** 신경망은 입력 → 여러 층 → 출력 → 손실의 연결 구조예요. 각 층의 파라미터가 손실에 얼마나 영향을 미치는지 알아야 파라미터를 업데이트할 수 있어요. 연쇄 법칙을 사용하면 출력층부터 시작해서 입력층 방향으로 차례차례 미분을 계산할 수 있어요. 이렇게 뒤에서 앞으로(Back) 전파(Propagation)하며 기울기를 계산하는 과정이 역전파예요.

</details>

### 실전 적용 문제

**문제.** 아래 상황에서 연쇄 법칙을 적용해보세요.

온도가 올라갈수록 아이스크림이 녹고, 아이스크림이 녹을수록 매출이 줄어들어요.

- 온도(T)가 1도 올라갈 때마다 녹음 정도가 3 증가해요
- 녹음 정도가 1 증가할 때마다 매출이 5 감소해요

1. 연쇄 법칙을 사용해서 온도가 1도 올라갈 때 매출이 얼마나 변하는지 계산해보세요
2. 이 계산이 연쇄 법칙의 어떤 원리를 보여주는지 설명해보세요

**힌트:**
- d(매출)/d(온도) = d(매출)/d(녹음) × d(녹음)/d(온도)
- 각 미분값을 먼저 정리해봐요

<details>
<summary>예시 답안 보기</summary>

**1. 연쇄 법칙 적용:**

각 미분값:
- d(녹음)/d(온도) = 3 (온도 1도 증가 → 녹음 3 증가)
- d(매출)/d(녹음) = -5 (녹음 1 증가 → 매출 5 감소 → 미분 = -5)

연쇄 법칙:
```
d(매출)/d(온도) = d(매출)/d(녹음) × d(녹음)/d(온도)
                = (-5) × 3
                = -15
```

온도가 1도 올라갈 때 매출은 **15** 감소해요.

**2. 연쇄 법칙의 원리:**

"온도 → 녹음 → 매출"의 연결 구조에서, 온도와 매출이 직접 연결되어 있지 않지만 녹음이라는 중간 변수를 통해 연결돼요. 연쇄 법칙은 이런 간접적인 관계에서 "온도 1단위 변화가 최종 결과(매출)에 미치는 영향"을 중간 변화율들의 곱으로 정확히 계산할 수 있게 해줘요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>
