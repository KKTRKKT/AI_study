# 오즈와 로그 오즈 (Odds and Log Odds)

---

## 📌 이 문서에서 배우는 것

- 오즈(Odds)가 확률(Probability)과 어떻게 다른지 정확히 이해해요
- 오즈를 확률로부터 어떻게 계산하는지 알아요
- 로그 오즈(Log Odds)가 왜 필요하고 어떤 이점이 있는지 알아요
- 로그 오즈가 로지스틱 회귀(Logistic Regression)에서 왜 핵심인지 이해해요
- 로지트 함수(Logit Function)가 무엇인지 알아요

## 1. 오즈(Odds)란 무엇인가요?

### 🤔 왜 확률 말고 오즈를 쓰나요?

스포츠 경기나 도박에서 "오즈"라는 말을 들어봤을 거예요. 하지만 오즈가 확률이랑 어떻게 다른지 헷갈리는 경우가 많아요. 정확히 구분해두면 통계학을 이해하는 데 큰 도움이 돼요.

### 한 줄 정의

> **오즈(Odds)란:** 어떤 사건이 "일어날 경우의 수"와 "일어나지 않을 경우의 수"의 비율이에요.

### 일상 비유

> 💡 **비유:** 팀이 5번의 경기를 한다고 해요. 1번 이기고 4번 진다면 "이길 오즈는 1대4(1:4)"예요. 승리 1번 대 패배 4번의 비율이에요.

---

## 2. 오즈와 확률의 차이

이게 핵심이에요. 많은 사람들이 헷갈려하는 부분이에요.

### 예시: 8번 중 5번 이기는 팀

| 지표 | 계산 | 결과 |
|---|---|---|
| **오즈** | 이긴 수 / 진 수 = 5 / 3 | **1.67** |
| **확률** | 이긴 수 / 전체 수 = 5 / 8 | **0.625** |

**핵심 차이:**

```
오즈     = 이긴 수    / 진 수       (비교 대상: 지는 것)
확률     = 이긴 수    / 전체 경기 수 (비교 대상: 모든 가능성)
```

- **오즈**는 "이기는 것 vs 지는 것"의 비율이에요
- **확률**은 "이기는 것 vs 모든 가능성"의 비율이에요

> ⚠️ **주의:** 오즈는 확률이 아니에요. 위 예시에서 오즈(1.67)와 확률(0.625)은 전혀 다른 숫자예요. 혼용하지 않도록 주의해요.

### 확률로부터 오즈 계산하기

오즈를 확률로도 표현할 수 있어요.

```
오즈 = 확률(이김) / 확률(짐)
     = 확률(이김) / (1 - 확률(이김))
     = p / (1 - p)
```

예: 이길 확률이 0.625라면
```
오즈 = 0.625 / (1 - 0.625) = 0.625 / 0.375 = 1.67
```

**결론:** 개수로 계산하든 확률로 계산하든 같은 오즈가 나와요.

## 3. 오즈의 불균형 문제

### 🤔 왜 로그 오즈가 필요한가요?

오즈를 그냥 쓰면 불편한 점이 있어요. 바로 **비대칭성**이에요.

팀의 성적이 어떻게 변하는지 보면서 오즈가 어떻게 변하는지 살펴볼게요.

**팀이 약할 때 — 오즈가 0과 1 사이에 있어요:**

| 상황 | 이김:짐 | 오즈 |
|---|---|---|
| 팀이 안 좋음 | 1:4 | 0.25 |
| 팀이 더 안 좋음 | 1:8 | 0.125 |
| 팀이 많이 안 좋음 | 1:16 | 0.0625 |
| 팀이 최악 | 1:32 | 0.031 |

오즈가 0에 가까워지기만 해요.

**팀이 강할 때 — 오즈가 1에서 무한대로 올라가요:**

| 상황 | 이김:짐 | 오즈 |
|---|---|---|
| 팀이 좋음 | 4:3 | 1.33 |
| 팀이 더 좋음 | 8:3 | 2.67 |
| 팀이 아주 좋음 | 32:3 | 10.7 |

오즈가 계속 커지기만 해요.

### 불균형의 문제점

수직선으로 표현하면:
```
← 약팀 ←                              → 강팀 →
0  0.03  0.06  0.125  0.25  1  1.33  2.67  10.7  ···
[───────────────────────]  [─────────────────── ···]
     0~1 사이에 빽빽하게                1부터 무한대로 쭉
```

약팀의 오즈는 0과 1 사이에만 존재하고, 강팀의 오즈는 1부터 무한히 커져요.

예를 들어 1대6으로 약팀인 경우(오즈 ≈ 0.17)와 6대1로 강팀인 경우(오즈 ≈ 6)를 비교해보면:
- 숫자상 차이: 6 - 0.17 = 5.83
- 실제 "얼마나 반대 상황인가" 관점에서는 똑같이 "6배 차이"인데, 숫자는 비대칭이에요

이 비대칭성이 통계 계산을 어렵게 만들어요.

---

## 4. 로그 오즈(Log Odds) — 대칭성 회복

### 해결책: 오즈에 로그를 취해요

```
로그 오즈 = log(오즈) = log(p / (1-p))
```

**이 수식이 말하는 것:**
오즈에 로그를 취하면 숫자가 대칭이 돼요.

예시:
```
오즈가 1대6(약팀):  log(1/6) = log(0.167) ≈ -1.79
오즈가 6대1(강팀):  log(6/1) = log(6)     ≈ +1.79
```

원점(0)으로부터 거리가 **똑같이** 1.79예요! 양수와 음수로 대칭이에요.

### 로그 오즈의 특성 정리

| 상황 | 오즈 | 로그 오즈 |
|---|---|---|
| 이길 가능성이 거의 없음 (1:100) | ~0 | -∞에 가까워요 |
| 이길 확률 = 50% (1:1) | 1.0 | **0** |
| 이길 가능성이 매우 높음 (100:1) | ~100 | +∞에 가까워요 |

**핵심 포인트:**
- 오즈가 1일 때(50대50) → 로그 오즈 = **0** (중심점)
- 오즈 > 1(유리할 때) → 로그 오즈 > 0
- 오즈 < 1(불리할 때) → 로그 오즈 < 0
- 대칭이에요: "1대6"과 "6대1"의 로그 오즈는 절댓값이 같고 부호만 반대예요

> 💡 **비유:** 섭씨 온도처럼 생각해봐요. 0℃(기준점)를 중심으로 -20℃와 +20℃는 똑같이 20만큼 떨어져 있어요. 로그 오즈도 0을 중심으로 음수와 양수가 대칭으로 배치돼요.

---

## 5. 로지트 함수 (Logit Function)

### 한 줄 정의

> **로지트 함수(Logit Function)란:** 확률(p)로부터 로그 오즈를 계산하는 함수예요.

```
logit(p) = log(p / (1-p))
```

이 함수는 로지스틱 회귀(Logistic Regression)의 핵심이에요. 로지스틱 회귀는 어떤 사건의 로그 오즈를 예측하는 선형 모델이에요. 나중에 로지스틱 회귀를 배울 때 이 함수가 계속 등장해요.

---

## 6. 로그 오즈가 정규 분포를 따른다는 것

### 이게 왜 중요한가요?

무작위로 숫자 쌍(두 수의 합이 100)을 만들고 각각에 대해 오즈를 계산한 뒤 히스토그램을 그려보면, **종 모양의 정규 분포**가 나와요.

이게 의미하는 것:
- 로그 오즈는 통계학에서 이미 잘 정립된 **정규 분포 이론**을 그대로 활용할 수 있어요
- 신뢰 구간 계산, p값 계산 등 정규 분포 기반의 검정들을 모두 적용할 수 있어요
- 특히 "예/아니오", "이김/짐" 같은 이진 결과(Binary Outcome)를 다루는 문제에 강력해요

---

## ⚠️ 주의할 점 / 자주 하는 실수

- **오즈와 확률을 혼용하지 마세요.** 오즈 1.5는 "150% 확률"이 아니에요. 오즈 1.5를 확률로 변환하면 1.5 / (1 + 1.5) = 0.6, 즉 60%예요.
- **"오즈비(Odds Ratio)"는 "오즈"가 아니에요.** 이름이 비슷해서 헷갈리지만 전혀 다른 개념이에요. 오즈비는 두 오즈를 비교한 비율이에요. 다음 문서에서 자세히 다뤄요.
- **로그 오즈가 음수라고 나쁜 게 아니에요.** 단순히 그 사건이 일어나지 않을 쪽이 더 유력하다는 의미예요.
- **로그의 밑(base)에 따라 값이 달라져요.** 로지스틱 회귀에서는 보통 자연로그(loge)를 써요.

- 확률 = 5(이긴 수) / 8(전체 수) = 0.625

같은 상황인데도 값이 달라요. 오즈는 "이기는 것 vs 지는 것"의 비율이고, 확률은 "이기는 것 vs 모든 가능성"의 비율이에요.

**Q2.** 이길 확률이 0.8이라면, 이길 오즈는 얼마인가요?

<details>
<summary>정답 보기</summary>

**정답:** 오즈 = 0.8 / (1 - 0.8) = 0.8 / 0.2 = **4**예요.

**설명:** 오즈 = p / (1-p) 공식을 사용해요. 확률 0.8은 80%이고, 이 경우 이길 확률이 질 확률의 4배라는 뜻이에요. "4대1"의 오즈예요.

</details>

---

**Q3.** 로그 오즈를 쓰는 이유가 "대칭성"이라고 했어요. 구체적으로 어떤 비대칭 문제가 해결되나요?

<details>
<summary>정답 보기</summary>

**정답:** 불리한 오즈는 0과 1 사이에, 유리한 오즈는 1에서 무한대 사이에 몰려 있는 비대칭 문제가 해결돼요.

**설명:** 예를 들어 "1대10으로 불리"와 "10대1로 유리"는 정반대의 상황인데, 오즈로는 0.1과 10으로 크기가 비대칭이에요. 로그 오즈로 변환하면 각각 약 -2.30과 +2.30으로, 원점 0으로부터 똑같이 2.30만큼 떨어진 대칭 값이 돼요. 이 덕분에 "얼마나 유리/불리한가"를 대칭적으로 비교할 수 있어요.

</details>

---

**Q4.** 로그 오즈가 0이라는 것은 어떤 상황인가요?

<details>
<summary>정답 보기</summary>

**정답:** 오즈가 1인 상황이에요. 즉, 일어날 확률과 일어나지 않을 확률이 똑같은 50대50 상황이에요.

**설명:** log(1) = 0이에요. 오즈가 1이라는 건 "일어날 수 = 일어나지 않을 수"라는 뜻이에요. 확률로 변환하면 0.5(50%)예요. 로그 오즈 = 0은 "어느 쪽도 더 유리하지 않은 중립 상태"를 의미해요. 로지스틱 회귀에서 로그 오즈 = 0이면 두 클래스의 확률이 동일해요.

</details>

---

**Q5.** 로지트(Logit) 함수가 로지스틱 회귀에서 핵심적인 이유가 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** 로지스틱 회귀는 결과가 "예/아니오"인 이진 분류 문제를 다루는데, 이 결과를 확률로 바로 예측하면 0~1 범위를 벗어나는 문제가 생겨요. 로지트 함수를 통해 확률을 로그 오즈로 변환하면 값의 범위가 -∞에서 +∞로 확장되어 선형 모델로 다룰 수 있게 돼요.

**설명:** 확률은 0~1 사이에만 있어야 해요. 그런데 선형 회귀 예측값은 어떤 숫자든 나올 수 있어요. 로지트 변환으로 확률을 로그 오즈로 바꾸면 -∞에서 +∞까지의 실수 범위가 되어서 선형 모델이 자연스럽게 적용돼요. 나중에 로그 오즈를 다시 시그모이드 함수로 확률로 변환해요.

</details>

### 실전 적용 문제

**문제.** 카드 게임에서 이길 확률이 0.6이에요.

1. 이길 오즈를 계산해보세요
2. 이길 로그 오즈를 계산해보세요 (자연로그 사용, ln(1.5) ≈ 0.405)
3. 만약 질 확률이 0.6이 된다면(이길 확률이 0.4) 로그 오즈는 어떻게 변하나요? 이길 때와 어떤 관계인가요?

**힌트:**
- 오즈 = p / (1-p)
- 로그 오즈 = log(오즈)
- 로그 오즈의 대칭성을 활용해봐요

<details>
<summary>예시 답안 보기</summary>

**1. 이길 오즈:**
```
오즈 = 0.6 / (1 - 0.6) = 0.6 / 0.4 = 1.5
```
이길 오즈는 **1.5**예요. 이길 확률이 질 확률의 1.5배예요.

**2. 로그 오즈:**
```
로그 오즈 = ln(1.5) ≈ 0.405
```
로그 오즈는 양수(0.405)예요. 이기는 쪽이 유리하기 때문이에요.

**3. 이길 확률이 0.4일 때:**
```
오즈 = 0.4 / (1 - 0.4) = 0.4 / 0.6 ≈ 0.667
로그 오즈 = ln(0.667) ≈ -0.405
```

이길 확률이 0.4(불리)일 때의 로그 오즈는 **-0.405**예요.

이기는 쪽이 유리할 때(+0.405)와 불리할 때(-0.405)의 절댓값이 같고 부호만 반대예요. 로그 오즈의 대칭성이 그대로 나타났어요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>
