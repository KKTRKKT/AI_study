# 직선을 데이터에 맞추는 핵심 아이디어 — 최소제곱법

## 📌 이 문서에서 배우는 것

- 데이터에 직선을 맞춘다는 것이 정확히 무슨 의미인지 이해해요
- 잔차(Residual)와 잔차 제곱합(Sum of Squared Residuals)이 무엇인지 알아요
- 왜 거리를 "제곱"해서 더하는지 이해해요
- 최소제곱법(Least Squares)이 어떤 원리로 최적의 직선을 찾는지 알아요
- 미분을 통해 최솟값을 찾는 아이디어를 이해해요

---

## 용어 정리

| 용어 | 원문 | 한 줄 설명 |
|---|---|---|
| 잔차 | Residual | 실제 데이터 값과 직선이 예측한 값의 차이예요 |
| 잔차 제곱합 | Sum of Squared Residuals | 모든 잔차를 제곱해서 더한 값이에요. 직선의 "오차"를 나타내요 |
| 최소제곱법 | Least Squares | 잔차 제곱합이 가장 작은 직선을 찾는 방법이에요 |
| 기울기 | Slope (A) | 직선이 얼마나 가파른지를 나타내요. x가 1 증가할 때 y가 얼마나 변하는지예요 |
| y절편 | Y-intercept (B) | x가 0일 때 직선이 y축을 만나는 점이에요 |
| 미분 | Derivative | 함수가 특정 점에서 얼마나 빠르게 변하는지, 즉 기울기를 구하는 방법이에요 |

## 1. 데이터에 직선을 맞추는 이유

### 🤔 왜 필요한가요?

실험이나 관측으로 데이터를 얻었을 때, 그 데이터에서 경향(Trend)을 파악하고 싶어요. 그리고 나아가 새로운 값을 **예측**하고 싶어요.

예를 들어 여러 데이터 포인트들이 XY 그래프에 찍혀 있을 때, 이 점들을 가장 잘 나타내는 직선은 무엇일까요? 그리고 "가장 잘 나타낸다"는 것을 어떻게 수치로 표현할 수 있을까요?

### 한 줄 정의

> **직선 맞추기(Fitting a Line)란:** 주어진 데이터 점들에 가장 가깝게 지나가는 직선을 찾는 과정이에요.

### 일상 비유

> 💡 **비유:** 화살 여러 개가 과녁판 주변에 꽂혀 있어요. "이 화살들의 평균 위치를 나타내는 직선"을 그린다고 생각해봐요. 각 화살에서 직선까지의 거리가 작을수록 직선이 화살들을 잘 "대표"하는 거예요.

---

## 2. 시작점 — 수평선과 평균값

### 기준선으로서의 수평선

가장 단순한 "직선"은 데이터의 평균 y값에서 그은 수평선이에요.

예를 들어 데이터의 평균 y값이 3.5라면, 수평선의 방정식은 `y = 3.5`예요.

이 수평선은 아마도 가장 나쁜 직선이에요. 데이터가 오른쪽으로 갈수록 커지는 경향이 있다면, 수평선은 그 경향을 전혀 반영하지 못해요.

그러나 이 수평선은 중요한 역할을 해요. **비교의 기준점**이 되어줘요. 우리가 만드는 어떤 직선도 이 수평선보다는 데이터를 더 잘 설명해야 해요.

### 수평선을 수식으로

```
y = B
```

여기서 B는 y절편이에요. 이 특정 데이터에서는 B = 3.5예요.

---

## 3. 잔차(Residual) — 직선과 데이터 사이의 거리

### 🤔 왜 "거리"가 중요한가요?

직선이 얼마나 데이터를 잘 설명하는지는 직선과 각 데이터 점 사이의 거리로 측정할 수 있어요. 거리가 작을수록 직선이 데이터를 잘 맞추는 거예요.

### 잔차란?

각 데이터 점과 직선 사이의 수직 거리를 **잔차(Residual)** 라고 해요.

```
잔차 = 실제 y값 − 직선이 예측한 y값
```

수평선(y = B)에 대한 잔차는:
```
잔차 = B − Y₁
잔차 = B − Y₂
잔차 = B − Y₃
...
```

---

## 4. 왜 "잔차를 제곱"하나요?

### 문제: 음수 잔차

데이터 점이 직선 **위**에 있으면 잔차가 음수예요.
데이터 점이 직선 **아래**에 있으면 잔차가 양수예요.

그냥 잔차를 다 더하면 어떻게 될까요? 양수와 음수가 서로 상쇄돼서 "오차가 없는 것처럼" 보일 수 있어요. 이건 잘못된 결과예요.

### 해결책: 제곱

역사적으로 절댓값을 쓰는 방법도 시도했어요. 그런데 절댓값은 수학적으로 다루기가 까다로워요(미분이 불편해요). 그래서 **제곱**을 사용해요.

제곱하면:
- 음수도 양수가 돼요 (상쇄 문제 해결)
- 큰 오차에 더 큰 벌점을 줘요 (작은 오차보다 큰 오차가 훨씬 나쁜 것으로 취급)
- 수학적으로 미분이 편리해요

### 잔차 제곱합 (Sum of Squared Residuals)

```
잔차 제곱합 = (B - Y₁)² + (B - Y₂)² + (B - Y₃)² + ...
             = Σ(직선 예측값 - 실제값)²
```

이 값이 작을수록 직선이 데이터를 잘 맞추는 거예요.

수평선의 경우 예시에서 잔차 제곱합 = **24.62**예요.

---

## 5. 더 좋은 직선 찾기 — 직선을 회전해봐요

### 직선을 조금씩 기울여보면?

수평선을 조금 기울이면(회전하면) 잔차 제곱합이 어떻게 될까요?

| 회전 정도 | 잔차 제곱합 |
|---|---|
| 수평선 (기준) | 24.62 |
| 조금 기울임 | 18.72 (개선됨 ✅) |
| 더 기울임 | 14.05 (더 개선됨 ✅) |
| 너무 많이 기울임 | 31.71 (오히려 나빠짐 ❌) |

처음에는 회전할수록 잔차 제곱합이 줄어들다가, 어느 순간부터 다시 커져요. 중간 어딘가에 최솟값이 있어요.

> 💡 **비유:** 활 시위를 당길 때 너무 조금 당기면 화살이 멀리 안 가고, 너무 많이 당기면 부러져요. 최적의 힘이 존재하는 것처럼, 직선에도 최적의 기울기가 있어요.

---

## 6. 최소제곱법(Least Squares) — 최솟값 찾기

### 직선의 방정식

```
y = Ax + B
```

- **A:** 기울기 (Slope)
- **B:** y절편 (Y-intercept)

A와 B의 값을 바꾸면 다른 직선이 돼요. A와 B 각각의 최적값을 찾아야 해요.

### 그래프로 이해하기

y축에 잔차 제곱합, x축에 직선의 회전(기울기 A) 값을 두면 그래프를 그릴 수 있어요.

```
잔차 제곱합
     ↑
     |     ×
     |   ×   ×
     | ×       ×
     |×           ×
     +────────────── → 기울기 A
             ↑
           최솟값 (여기서 기울기 = 0)
```

그래프는 아래로 볼록한 곡선 모양이에요. 최솟값에서 기울기(미분값)가 0이에요.

### 미분으로 최솟값 찾기

최적의 직선을 찾는 방법:

1. 잔차 제곱합을 A(기울기)와 B(y절편)의 함수로 표현해요
2. 이 함수를 A와 B 각각에 대해 미분해요
3. 미분값 = 0인 지점이 최솟값이에요 → 이게 최적의 A와 B예요

> 💡 **보충 설명:** 이 계산을 손으로 하는 사람은 없어요. 컴퓨터가 대신 해줘요. 중요한 건 계산 방법이 아니라 **"잔차 제곱합을 최소화하는 A와 B를 찾는다"** 는 핵심 개념이에요.

### 2D에서 3D로 — 기울기와 절편 동시에 최적화

기울기 A와 y절편 B를 동시에 최적화하려면 3D 그래프가 필요해요.

- x축: 기울기 A
- z축(깊이): y절편 B
- y축(높이): 잔차 제곱합

이 3D 그래프에서 가장 낮은 점(그릇의 바닥 같은 곳)이 최적의 (A, B) 조합이에요. 이 점을 찾기 위해 A와 B 각각에 대한 편미분을 0으로 놓고 풀어요.

---

## 7. 최적의 직선

이 과정의 결과로 나온 최적의 직선:

```
y = 0.77x + 0.66
```

- 기울기 A = 0.77
- y절편 B = 0.66

이 직선이 주어진 데이터에 대해 잔차 제곱합을 최소화하는 최적의 직선이에요.

---

## 8. 두 가지 핵심 개념 정리

> 🎯 **핵심 개념 1:** 실제 값과 직선 사이의 거리(잔차)를 **제곱해서 더한 값**(잔차 제곱합)을 최소화하는 직선을 찾아요.

> 🎯 **핵심 개념 2:** 최솟값은 **미분값이 0**인 지점에서 찾아요.

이 두 가지가 최소제곱법의 전부예요.

---

## ⚠️ 주의할 점 / 자주 하는 실수

- **잔차와 오차를 혼용하지 마세요.** 엄밀하게는 잔차(Residual)는 모델 예측값과 관측값의 차이이고, 오차(Error)는 모집단의 진짜 값과의 차이예요. 현실에서는 잔차를 통해 오차를 추정해요.
- **직선을 맞추는 게 목적이 아니에요.** 직선은 예측과 관계 파악을 위한 도구예요. "잔차 제곱합이 작다"는 것은 데이터를 잘 설명한다는 뜻이지, 예측이 정확하다는 보장이 아니에요.
- **이상치(Outlier)에 민감해요.** 제곱을 하기 때문에 멀리 떨어진 점이 잔차 제곱합에 훨씬 큰 영향을 줘요. 이상치가 있으면 최적의 직선이 크게 왜곡될 수 있어요.
- **직선이 항상 최선은 아니에요.** 데이터가 곡선 패턴을 가지고 있다면 직선으로는 잘 맞출 수 없어요. 편향(Bias)이 높아지는 상황이에요.

<summary>정답 보기</summary>

**정답:** 잔차는 실제 데이터 값과 직선의 예측값 사이의 차이예요. 잔차를 그냥 더하면 양수와 음수가 상쇄되기 때문에 제곱해서 더해요.

**Q4.** 최적의 기울기와 절편을 찾을 때 "미분값이 0인 지점"을 찾는 이유는 무엇인가요?

<details>
<summary>정답 보기</summary>

**설명:** 아래로 볼록한 곡선에서는 가장 낮은 지점(최솟값)의 좌우에서 기울기가 반대예요. 왼쪽으로 가면 내리막(음의 기울기), 오른쪽으로 가면 오르막(양의 기울기), 최솟값 지점에서는 기울기가 정확히 0이에요. 그래서 미분값 = 0인 지점이 최솟값을 가리켜요.

</details>

**Q5.** 잔차 제곱합이 아니라 잔차의 절댓값 합을 최소화하면 안 될까요? 왜 제곱을 선택했을까요?

<details>
<summary>정답 보기</summary>

**정답:** 절댓값은 수학적으로 다루기 어려워요. 미분이 까다롭기 때문에 최솟값을 찾기 불편해요. 제곱은 미분이 쉽고, 컴퓨터가 효율적으로 계산할 수 있어요.

**설명:** 절댓값 함수는 0에서 미분이 정의되지 않아요. 그래서 최솟값을 찾는 수학적 과정이 복잡해져요. 반면 제곱 함수는 어디서든 미분이 가능하고, 미분값 = 0인 지점을 방정식으로 풀 수 있어요. 역사적으로 이 계산 편의성이 제곱을 선택하게 만든 주요 이유 중 하나예요.

</details>

### 실전 적용 문제

**문제.** 세 개의 데이터 점이 있어요: (1, 2), (2, 4), (3, 5)

수평선 y = 3.67(평균값)과 직선 y = 1.5x + 0.33 중 어느 것이 이 데이터를 더 잘 맞추는지 잔차 제곱합을 계산해서 비교해보세요.

**힌트:**
- 각 점에서 잔차 = 실제 y값 − 직선의 y값
- 잔차 제곱합 = 모든 잔차²의 합
- 잔차 제곱합이 작을수록 더 잘 맞는 직선이에요

<details>
<summary>예시 답안 보기</summary>

**수평선 y = 3.67의 잔차 제곱합:**
- (1, 2): 잔차 = 2 - 3.67 = -1.67 → 잔차² = 2.79
- (2, 4): 잔차 = 4 - 3.67 = 0.33 → 잔차² = 0.11
- (3, 5): 잔차 = 5 - 3.67 = 1.33 → 잔차² = 1.77
- **잔차 제곱합 = 2.79 + 0.11 + 1.77 = 4.67**

**직선 y = 1.5x + 0.33의 잔차 제곱합:**
- x=1: 예측값 = 1.5×1 + 0.33 = 1.83 → 잔차 = 2 - 1.83 = 0.17 → 잔차² = 0.03
- x=2: 예측값 = 1.5×2 + 0.33 = 3.33 → 잔차 = 4 - 3.33 = 0.67 → 잔차² = 0.45
- x=3: 예측값 = 1.5×3 + 0.33 = 4.83 → 잔차 = 5 - 4.83 = 0.17 → 잔차² = 0.03
- **잔차 제곱합 = 0.03 + 0.45 + 0.03 = 0.51**

**결론:** 직선(0.51)이 수평선(4.67)보다 잔차 제곱합이 훨씬 작아요. 직선이 이 데이터를 훨씬 잘 설명해요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>
