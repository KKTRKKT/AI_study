# 📈 선형 회귀와 최소제곱법

---

## 1. 최소제곱법 (Least Squares)이란?

> **데이터를 가장 잘 설명하는 직선을 찾기 위해, 잔차(오차)의 제곱합을 최소화하는 방법**

---

### 💡 핵심 목표: "가장 잘 맞는 직선 찾기"

데이터 점들이 여러 개 있을 때, 그 점들을 **가장 잘 설명하는 직선 하나**를 그어야 합니다.  
수많은 직선 후보 중에서 "가장 적합한 직선"을 찾는 게 목표입니다.

---

### 📌 직선의 방정식

```
ŷ = β̂₀ + β̂₁ × x
```

| 기호 | 의미 | 쉬운 말 |
|------|------|---------|
| `ŷ` (y햇) | 예측값 | 직선이 예측한 y값 |
| `β̂₀` (베타0햇) | 절편 | 직선이 y축과 만나는 점 |
| `β̂₁` (베타1햇) | 기울기 | 직선이 얼마나 가파른지 |
| `x` | 입력값 | 내가 넣는 값 |

> 💡 **햇(^) 기호**는 "추정한 값"이라는 뜻입니다. 정확한 값은 모르지만 데이터로 추정했다는 표시예요.

---

### 📌 잔차(Residual)란?

직선과 **실제 데이터 점** 사이의 차이를 **잔차(residual)** 라고 합니다.

```
잔차 eᵢ = 실제값 yᵢ - 예측값 ŷᵢ
```

```
실제 점  •  ← 실제 관측값 (yᵢ)
         |
         |  ← 이 세로 길이가 잔차(eᵢ)
         |
─────────•─────  ← 직선 (예측값 ŷᵢ)
```

- 잔차 **양수** → 실제값이 직선보다 **위**에 있음
- 잔차 **음수** → 실제값이 직선보다 **아래**에 있음
- 잔차 **0** → 직선이 그 점을 **정확히** 통과함

---

### 📌 RSS (잔차제곱합)

잔차를 그냥 더하면 양수·음수가 상쇄되어 문제가 생깁니다. 그래서 **제곱해서 더합니다.**

| 방법 | 문제점 |
|------|--------|
| 그냥 합산 | +3 + (-3) = 0 → 잔차가 없는 것처럼 보임 |
| **제곱 합산** ✅ | 3² + (-3)² = 18 → 항상 양수, 수학적으로 다루기 쉬움 |

```
RSS = e₁² + e₂² + ... + eₙ²
    = Σᵢ(yᵢ - ŷᵢ)²
    = Σᵢ(yᵢ - β̂₀ - β̂₁xᵢ)²
```

> 💡 **RSS가 작을수록** 직선이 데이터에 **잘 맞는** 것입니다!

---

### 📌 최적 계수 공식 (Closed-form Solution)

RSS를 최소로 만드는 β̂₁과 β̂₀을 수학적으로 유도할 수 있습니다.

**먼저 평균을 구합니다:**
```
x̄ = x값들의 평균 = (x₁ + x₂ + ... + xₙ) / n
ȳ = y값들의 평균 = (y₁ + y₂ + ... + yₙ) / n
```

**기울기 공식:**
```
         Σᵢ(xᵢ - x̄)(yᵢ - ȳ)
β̂₁ = ─────────────────────────
           Σᵢ(xᵢ - x̄)²
```

**절편 공식:**
```
β̂₀ = ȳ - β̂₁x̄
```

#### 🔢 예시로 이해하기

공부시간(x)과 점수(y) 데이터 3개:

| 학생 | 공부시간(x) | 점수(y) |
|------|------------|--------|
| A | 1시간 | 50점 |
| B | 2시간 | 70점 |
| C | 3시간 | 90점 |

**1) 평균:** x̄ = 2, ȳ = 70

**2) 기울기 계산:**

| 학생 | (xᵢ - x̄) | (yᵢ - ȳ) | 곱 | (xᵢ - x̄)² |
|------|-----------|-----------|-----|------------|
| A | -1 | -20 | 20 | 1 |
| B | 0 | 0 | 0 | 0 |
| C | 1 | 20 | 20 | 1 |
| **합계** | | | **40** | **2** |

β̂₁ = 40 / 2 = **20**  
β̂₀ = 70 - 20×2 = **30**

**최종 직선:** `점수 = 30 + 20 × 공부시간`

## 2. 단순선형회귀: 광고 데이터 예시

> **목표: TV 광고비(X)와 제품 판매량(Y)의 선형 관계 예측**

---

### 📊 회귀 결과 해석

최소제곱법을 적용한 결과:

```
판매량(ŷ) = 7.03 + 0.0475 × TV광고비(x)
```

| 항목 | 계수 | 표준오차 | t-통계량 | p-value |
|------|------|---------|--------|---------|
| 절편 | 7.03 | 0.4578 | 15.36 | < 0.0001 |
| TV | 0.0475 | 0.0027 | 17.67 | < 0.0001 |

#### 절편 = 7.03 의미
- TV 광고비가 0원이어도 기본 판매량은 **703만원**
- 브랜드 인지도, 기존 고객 등에 의한 기본 판매량

#### 기울기 = 0.0475 의미
- TV 광고비를 **1백만원 더 쓰면** 판매량이 평균 **4만 7500원 증가**

---

### 📌 표 항목 상세 설명

| 항목 | 의미 |
|------|------|
| **Coefficient** | β 계수 (절편, 기울기) |
| **Std. Error** | 표준오차 — 계수의 불확실성. 작을수록 믿을 만함 |
| **t-statistic** | 계수 ÷ 표준오차. 클수록 계수가 0이 아닐 가능성 높음 |
| **p-value** | 이 결과가 우연일 확률. **0.05 미만이면 유의미** |

> ⚠️ **p-value 기준:**
> - p < 0.05: 통계적으로 유의미 ✅
> - p < 0.0001: 극도로 유의미 ✅✅✅
> - p > 0.05: 유의미하지 않음 ❌

---

### 📈 모형 적합도: R²

```
R² = 0.612
→ "판매량 변동의 61.2%를 TV 광고비로 설명할 수 있다"
```

```
판매량 변동 이유:
├── TV 광고비 때문: 61.2% ← 우리 모델이 설명하는 부분
└── 다른 이유: 38.8%  ← 라디오, 신문, 계절 등
```

## 3. 다중선형회귀 (Multiple Linear Regression)

> **여러 개의 독립변수(X)를 동시에 고려해서 Y를 예측하는 방법**

---

### 💡 단순 vs 다중 선형회귀 차이

| 구분 | 단순선형회귀 | 다중선형회귀 |
|------|------------|------------|
| 입력변수 | 1개 | 여러 개 |
| 수식 | y = β₀ + β₁x | y = β₀ + β₁x₁ + β₂x₂ + ... |
| 그래프 | 2D 직선 | 다차원 평면(hyperplane) |

---

### 📌 각 기호의 의미

```
Y = β₀ + β₁X₁ + β₂X₂ + β₃X₃ + ... + βₚXₚ + ε
```

광고 데이터 예시:
```
판매량 = β₀ + β₁×TV광고비 + β₂×라디오광고비 + β₃×신문광고비 + ε
```

| 기호 | 이름 | 의미 |
|------|------|------|
| Y | 종속변수 | 예측하고 싶은 것 (판매량) |
| X₁, X₂... | 독립변수 | 입력으로 넣는 것들 (각종 광고비) |
| β₀ | 절편 | 모든 X가 0일 때의 Y값 |
| β₁, β₂... | 계수 | 각 변수의 영향력 크기와 방향 |
| ε | 오차 | 모델이 설명하지 못하는 나머지 부분 |

> 💡 **핵심 해석:** "다른 변수를 고정한 채" Xⱼ가 1단위 증가하면 Y가 βⱼ만큼 변화

---

### 📌 RSS와 Closed-form 해

다중선형회귀의 RSS:
```
RSS = Σᵢ(yᵢ - β̂₀ - β̂₁xᵢ₁ - ... - β̂ₚxᵢₚ)²
```

**행렬 표현으로 최적 해:**
```
β̂ = (XᵀX)⁻¹Xᵀy
```

여기서 X는 데이터 행렬, y는 레이블 벡터입니다.

---

### 📊 다중선형회귀 결과: 광고 데이터

```
판매량 = 2.939 + 0.046×TV + 0.189×라디오 - 0.001×신문
```

| 변수 | 계수 | p-value | 유의미? |
|------|------|---------|--------|
| 절편 | 2.939 | < 0.0001 | ✅ |
| TV | 0.046 | < 0.0001 | ✅ |
| 라디오 | 0.189 | < 0.0001 | ✅ |
| 신문 | -0.001 | 0.8599 | ❌ |

- **R² = 0.897** → 단순선형회귀(0.612)보다 훨씬 향상됨!

#### 🔍 핵심 발견: 신문 광고비의 반전

```
단순선형회귀로 신문만 분석 → 유의미해 보임
다중선형회귀로 함께 분석  → 유의미하지 않음!

이유: 신문 광고를 많이 하는 회사 = 라디오 광고도 많이 함
→ 라디오의 효과가 신문의 효과로 착각되었던 것!
→ 다중선형회귀는 각 변수의 순수한 독립적 효과를 분리해줌
```

## 4. 주의사항: 상관관계 ≠ 인과관계

> **두 변수가 함께 변한다고 해서, 한 변수가 다른 변수의 원인은 아닙니다!**

### 🍦 아이스크림과 상어 공격 예시

```
데이터 분석: 아이스크림 소비량↑ → 상어에 물리는 사건↑

실제 이유:
여름 → 더움 → 아이스크림 소비↑
여름 → 더움 → 해수욕장 방문↑ → 상어 공격↑

"아이스크림이 상어를 부르는 게 아니라
 '더운 날씨'라는 제3의 변수(교란변수)가 둘 다 영향을 준 것!"
```

### ⚠️ 과적합 주의

모델을 훈련 데이터에만 맞추면 새로운 데이터에서 성능이 떨어집니다.  
반드시 **테스트 데이터로 검증**하고, 변수가 너무 많으면 **과적합** 위험을 주의하세요.

## 5. 전체 흐름 정리

```
1. 데이터 수집
   Feature(X)와 Label(Y) 준비
          ↓
2. 모델 선택
   단순선형회귀 (변수 1개) or 다중선형회귀 (변수 여러 개)
          ↓
3. 최소제곱법으로 β 추정
   RSS를 최소화하는 β̂ 계산
          ↓
4. 결과 해석
   p-value, R²로 유의미성과 설명력 확인
          ↓
5. 예측 및 주의사항
   새 데이터 예측 + 상관관계≠인과관계 주의
```

> **핵심 요약:** 최소제곱법은 "잔차의 제곱합(RSS)이 가장 작아지는 직선을 찾는 방법"이며, 이를 통해 데이터를 가장 잘 설명하는 선형 모델을 만들 수 있습니다.
