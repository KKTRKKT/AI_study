## 1. 디자인 행렬이 왜 필요한가요? <!-- ⏱️ 00:40 -->

### 🤔 왜 필요한가요?

선형 회귀, t-검정, ANOVA는 서로 다른 기법처럼 보여도, 사실 모두 같은 수학적 틀(F통계량과 잔차제곱합)을 쓰고 있어요. 그런데 이 기법들을 컴퓨터가 자동으로 처리하려면, 서로 다른 형태의 문제들을 **같은 형식**으로 표현해야 해요.

바로 그 "같은 형식"이 디자인 행렬이에요.

### 한 줄 정의

> **디자인 행렬(Design Matrix)이란:** 회귀식의 각 파라미터에 어떤 값을 곱할지 정해주는 숫자 표예요. 각 데이터 포인트가 한 행이 되고, 각 파라미터가 한 열이 돼요.

### 일상 비유

> 💡 **비유:** 냉장고 안에 여러 재료(파라미터)가 있다고 상상해봐요. 디자인 행렬은 각 요리(데이터 포인트)에서 어떤 재료를 넣고(1) 빼는지(0)를 알려주는 레시피 표예요. 재료는 같아도 레시피(디자인 행렬)에 따라 전혀 다른 요리(t-검정, 회귀, ANOVA 등)가 만들어져요.

---

## 2. 디자인 행렬의 기본 구조 — "스위치" 개념 <!-- ⏱️ 02:00 -->

### 어떻게 동작하나요?

디자인 행렬에서 **1은 켜기, 0은 끄기** 스위치 역할을 해요.

예를 들어 t-검정을 한다고 해봐요. 대조군과 돌연변이 쥐의 유전자 발현량을 비교해요.

**회귀식:**
```
y = 대조군 평균 × (스위치 A) + 돌연변이 평균 × (스위치 B)
```

**디자인 행렬:**

| 데이터 포인트 | 스위치 A (대조군 평균 ON/OFF) | 스위치 B (돌연변이 평균 ON/OFF) |
|---|---|---|
| 대조군 데이터 1 | 1 | 0 |
| 대조군 데이터 2 | 1 | 0 |
| 대조군 데이터 3 | 1 | 0 |
| 돌연변이 데이터 1 | 0 | 1 |
| 돌연변이 데이터 2 | 0 | 1 |
| 돌연변이 데이터 3 | 0 | 1 |

- 대조군 데이터: 스위치 A = 1(켜짐), 스위치 B = 0(꺼짐) → 대조군 평균만 적용
- 돌연변이 데이터: 스위치 A = 0(꺼짐), 스위치 B = 1(켜짐) → 돌연변이 평균만 적용

이렇게 하면 두 그룹의 평균을 **하나의 식**으로 표현할 수 있어요.

---

## 3. 표준 디자인 행렬 — 오프셋 형식 <!-- ⏱️ 04:00 -->

### 어떻게 동작하나요?

사실 가장 흔히 쓰이는 디자인 행렬은 위의 방식과 조금 달라요. 더 일반적인 형식은 이렇게 생겼어요:

**표준 회귀식:**
```
y = 대조군 평균 × 1 + (돌연변이 평균 - 대조군 평균) × (스위치 B)
```

두 번째 항은 "돌연변이 그룹이 대조군보다 얼마나 다른가"를 나타내는 **오프셋(offset)** 이에요.

**표준 디자인 행렬:**

| 데이터 포인트 | 열 1 (대조군 평균) | 열 2 (돌연변이 오프셋) |
|---|---|---|
| 대조군 데이터 | 1 | 0 |
| 돌연변이 데이터 | 1 | 1 |

- 대조군: 1 × 대조군 평균 + 0 × 오프셋 = 대조군 평균
- 돌연변이: 1 × 대조군 평균 + 1 × 오프셋 = 대조군 평균 + 오프셋 = 돌연변이 평균

두 방식은 결과가 완전히 동일해요. F값도 같고 p값도 같아요. 표준 형식이 조금 더 일반적으로 쓰여요.

> 💡 **팁:** 두 방식이 같은 결과를 내는 이유는, 파라미터 수가 같아서 F통계량 계산에 들어가는 값들이 동일하기 때문이에요.

## 4. 선형 회귀에서의 디자인 행렬 <!-- ⏱️ 06:00 -->

### 어떻게 동작하나요?

t-검정에서는 0과 1만 썼지만, **선형 회귀에서는 다른 숫자도 쓸 수 있어요.**

쥐의 몸무게로 키를 예측하는 회귀식:
```
키 = 절편 × 1 + 기울기 × 몸무게
```

**디자인 행렬 (4마리 쥐의 경우):**

| 쥐 | 열 1 (절편 ON/OFF) | 열 2 (몸무게 값) |
|---|---|---|
| 쥐 1 (몸무게 1.0) | 1 | 1.0 |
| 쥐 2 (몸무게 2.0) | 1 | 2.0 |
| 쥐 3 (몸무게 1.5) | 1 | 1.5 |
| 쥐 4 (몸무게 2.5) | 1 | 2.5 |

- 첫 번째 열: 항상 1 → 절편을 항상 켜두는 역할이에요
- 두 번째 열: 실제 몸무게 값 → 기울기에 곱해져서 예측값에 반영돼요

각 행을 회귀식에 대입하면 선 위의 해당 점이 나와요. 거기서 잔차를 계산하고, 잔차제곱합을 구해서 F값과 p값을 계산해요.

## 5. 복합 모델 — t-검정과 회귀를 동시에! <!-- ⏱️ 07:30 -->

### 🤔 왜 필요한가요?

이게 디자인 행렬의 진짜 강점이에요. 실제 데이터에서는 단순한 t-검정이나 단순 회귀만으로 부족한 경우가 많아요.

예를 들어 "보통 쥐와 돌연변이 쥐가 키에서 다른가?"를 비교하고 싶은데, 쥐마다 몸무게도 다르다면 어떻게 할까요?

단순 t-검정: 몸무게 차이를 무시해서 부정확해요
단순 회귀: 종류 차이를 반영 못 해요

두 가지를 동시에 쓰면 되는데, 이걸 하나의 식으로 표현하는 게 복합 디자인 행렬이에요.

### 어떻게 동작하나요?

**복합 회귀식:**
```
키 = 절편 + 돌연변이 오프셋 × (종류) + 기울기 × 몸무게
```

이 식에는 파라미터가 3개예요:
- 절편 (보통 쥐의 y절편)
- 돌연변이 오프셋 (돌연변이 쥐가 보통 쥐보다 얼마나 더 큰지)
- 기울기 (몸무게-키 관계, 두 종류 쥐에게 동일하게 적용)

**복합 디자인 행렬:**

| 쥐 | 열 1 (절편) | 열 2 (돌연변이 오프셋) | 열 3 (몸무게) |
|---|---|---|---|
| 보통 쥐 1 | 1 | 0 | 1.2 |
| 보통 쥐 2 | 1 | 0 | 2.1 |
| 돌연변이 1 | 1 | 1 | 1.5 |
| 돌연변이 2 | 1 | 1 | 2.3 |

- 보통 쥐: 오프셋 = 0 → 오프셋 항이 꺼짐 → 보통 쥐의 회귀선을 따라요
- 돌연변이: 오프셋 = 1 → 오프셋 항이 켜짐 → 보통 쥐보다 오프셋만큼 높은 회귀선을 따라요

이렇게 하면 몸무게를 제어한 상태에서 종류 차이가 유의한지 알 수 있어요.

### 실제 예시 — 세 가지 비교

이 복합 모델을 기준으로 다양한 비교를 할 수 있어요:

**비교 1: 복합 모델 vs 평균**
- 질문: "몸무게 + 종류 조합이 평균만 쓰는 것보다 유의하게 낫나?"
- p값 = 0.003 → 네, 유의해요

**비교 2: 복합 모델 vs 단순 회귀(몸무게만)**
- 질문: "종류 변수가 몸무게만 쓰는 것보다 예측을 개선하나?"
- p값 = 0.002 → 네, 종류도 중요해요

**비교 3: 복합 모델 vs t-검정(종류만)**
- 질문: "몸무게 변수가 종류만 쓰는 것보다 예측을 개선하나?"
- p값 = 0.003 → 네, 몸무게도 중요해요

## 6. 배치 효과 보정 — 실험실이 다를 때 <!-- ⏱️ 10:00 -->

### 🤔 왜 필요한가요?

실험실 A와 실험실 B가 같은 실험을 했는데, B의 측정값이 전반적으로 낮게 나왔어요. 이런 체계적 차이를 **배치 효과(Batch Effect)** 라고 해요. 두 데이터를 합치려면 이걸 보정해야 해요.

### 어떻게 동작하나요?

디자인 행렬에 "실험실 B 오프셋" 열을 추가해요:

**보정 회귀식:**
```
유전자 발현 = 대조군 평균 + (B 실험실 오프셋) × (실험실) + (돌연변이 오프셋) × (종류)
```

| 데이터 | 열 1 (대조군 평균) | 열 2 (B 실험실 오프셋) | 열 3 (돌연변이 오프셋) |
|---|---|---|---|
| A 실험실 대조군 | 1 | 0 | 0 |
| A 실험실 돌연변이 | 1 | 0 | 1 |
| B 실험실 대조군 | 1 | 1 | 0 |
| B 실험실 돌연변이 | 1 | 1 | 1 |

이렇게 하면 실험실 차이를 통제한 상태에서 대조군과 돌연변이 차이만을 검정할 수 있어요.

---

## ⚠️ 주의할 점 / 자주 하는 실수

- ❌ "디자인 행렬은 0과 1만 쓴다" → 회귀에서는 실제 값(몸무게, 나이 등)도 들어가요.
- ❌ "디자인 행렬의 형식은 하나뿐이다" → 같은 분석을 다른 형식의 디자인 행렬로도 할 수 있어요. 결과는 같아요.
- ❌ "복잡한 모델을 쓰면 무조건 좋다" → 파라미터를 너무 많이 추가하면 데이터보다 파라미터가 많아지는 문제가 생겨요.
- ✅ "디자인 행렬을 쓰면 어떤 복잡한 상황도 단일 식으로 표현할 수 있다" → 이게 디자인 행렬의 진짜 장점이에요.

### 개념 확인 퀴즈

**Q1.** 디자인 행렬에서 "1"과 "0"이 각각 무슨 역할을 하나요?

<details>
<summary>정답 보기</summary>

**정답:** 1은 해당 파라미터를 "켜는" 역할이고, 0은 "끄는" 역할이에요.

**설명:** 예를 들어 어떤 데이터 포인트가 대조군이라면, "대조군 파라미터" 열에 1이 들어가서 그 파라미터가 예측에 반영되고, "돌연변이 파라미터" 열에 0이 들어가서 그 파라미터는 무시돼요. 이 스위치 메커니즘 덕분에 여러 그룹의 데이터를 단일 식으로 표현할 수 있어요.

</details>

---

**Q2.** 표준 디자인 행렬에서 두 번째 열이 "두 그룹 평균의 차이(오프셋)"인 이유는 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** 대조군을 기준(절편)으로 삼고, 돌연변이 그룹이 거기서 얼마나 다른지를 오프셋으로 표현하기 위해서예요.

**설명:** "돌연변이 평균"을 직접 파라미터로 쓰는 대신, "돌연변이 평균 - 대조군 평균"이라는 차이를 파라미터로 써요. 이렇게 하면 "두 그룹이 얼마나 다른가"를 직접 추정할 수 있어서 해석이 더 직관적이에요. 또 다양한 모델을 비교할 때도 이 오프셋 형식이 더 유용해요.

</details>

---

**Q3.** t-검정과 선형 회귀를 동시에 수행하는 복합 모델이 왜 필요한가요?

<details>
<summary>정답 보기</summary>

**정답:** 그룹 차이와 연속 변수의 영향을 동시에 분석하고 싶을 때 필요해요.

**설명:** 예를 들어 "돌연변이 쥐가 보통 쥐보다 큰가?"라는 질문에 답할 때, 쥐마다 몸무게가 다르면 몸무게 차이가 결과에 영향을 줘요. 단순 t-검정은 몸무게를 무시하고, 단순 회귀는 종류를 무시해요. 복합 모델은 두 가지를 동시에 고려해서 더 정확한 답을 줘요. 이걸 가능하게 하는 게 디자인 행렬이에요.

</details>

---

**Q4.** 배치 효과(Batch Effect)가 무엇이고, 디자인 행렬로 어떻게 보정하나요?

<details>
<summary>정답 보기</summary>

**정답:** 배치 효과는 실험실이 다르거나 측정 시기가 달라서 생기는 체계적인 값의 차이예요. 디자인 행렬에 "어느 실험실 데이터인지"를 나타내는 열을 추가해서 보정해요.

**설명:** B 실험실 데이터에는 "B 오프셋" 열에 1을 넣고, A 실험실 데이터에는 0을 넣어요. 그러면 최소제곱법이 B 실험실의 전반적 오차를 자동으로 추정하고 보정해줘요. 이렇게 실험실 차이를 통제한 상태에서 진짜 관심 변수(예: 대조군 vs 돌연변이)의 영향만 분리할 수 있어요.

</details>

### 실전 적용 문제

**문제.** 세 개 학교(A, B, C)에서 수업 방식(전통 vs 플립드 러닝)의 효과를 비교하고 싶어요. 그런데 학교마다 학생들의 평균 성적 수준이 다르게 시작해요 (학교 효과). 시험 점수를 예측하는 복합 모델의 디자인 행렬 구조를 설계해보세요.

**힌트:**
- 파라미터: 기준 평균(학교 A, 전통 수업), 학교 B 오프셋, 학교 C 오프셋, 플립드 러닝 오프셋
- 각 학생 데이터에 어떤 열에 1이 들어가야 할지 생각해보세요

**파라미터 (열) 설계:**
- 열 1: 기준값 (항상 1) → 학교 A, 전통 수업의 평균
- 열 2: 학교 B 오프셋 → 학교 B 학생에게만 1
- 열 3: 학교 C 오프셋 → 학교 C 학생에게만 1
- 열 4: 플립드 러닝 오프셋 → 플립드 러닝 수업 학생에게만 1

**디자인 행렬:**

| 학생 | 열1 (기준) | 열2 (B 오프셋) | 열3 (C 오프셋) | 열4 (플립드 오프셋) |
|---|---|---|---|---|
| A학교, 전통 수업 | 1 | 0 | 0 | 0 |
| A학교, 플립드 | 1 | 0 | 0 | 1 |
| B학교, 전통 수업 | 1 | 1 | 0 | 0 |
| B학교, 플립드 | 1 | 1 | 0 | 1 |
| C학교, 전통 수업 | 1 | 0 | 1 | 0 |
| C학교, 플립드 | 1 | 0 | 1 | 1 |

이 모델로 학교 효과를 보정한 상태에서 수업 방식(열 4)의 효과를 검정할 수 있어요.

| 🗂️ 디자인 행렬 | 각 데이터 포인트에 어떤 파라미터를 적용할지 정해주는 숫자 표 |
| 🔛 1과 0 | 1은 파라미터를 켜고, 0은 꺼요 — 그룹 소속을 표현해요 |
| 🔄 오프셋 형식 | 기준 그룹에서 비교 그룹이 얼마나 다른지 차이를 파라미터로 표현해요 |

| 🔀 복합 모델 | t-검정 + 회귀를 한 번에 — 범주형 + 연속형 변수를 함께 분석해요 |
