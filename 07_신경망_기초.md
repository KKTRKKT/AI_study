# 🧠 신경망(Neural Network) 기초

---

## 1. 왜 신경망이 필요한가?

### 💡 선형 모델의 한계

```
선형회귀: y = φ₀ + φ₁x   ← 직선만 표현 가능

현실 데이터는 대부분 직선이 아님!
→ 더 복잡한 패턴을 표현할 모델이 필요!
```

### 선형 모델로 표현 가능한 것 vs 불가능한 것

```
✅ 가능: 직선 관계 (광고비 → 판매량)
❌ 불가능: 꺾이는 관계, 복잡한 곡선, XOR 패턴
```

---

## 2. Shallow 네트워크 (은닉층 1개)

> **입력층 → 은닉층(Hidden Layer, 1개) → 출력층으로 구성된 네트워크**

---

### 📌 구조

```
입력(x) → [은닉층: h₁, h₂, h₃] → 출력(y)
```

| 층 이름 | 역할 |
|--------|------|
| 입력층 | 데이터(x)를 받아들임 |
| 은닉층 | 내부 계산 (사람 눈에 안 보임) |
| 출력층 | 최종 예측값(y) 출력 |

---

### 📌 수식

```
y = f[x, φ]
  = φ₀ + φ₁·a[θ₁₀ + θ₁₁x]
       + φ₂·a[θ₂₀ + θ₂₁x]
       + φ₃·a[θ₃₀ + θ₃₁x]
```

**두 부분으로 분리해서 생각하기:**

```
은닉 유닛 계산:          출력 계산:
h₁ = a[θ₁₀ + θ₁₁x]     y = φ₀ + φ₁h₁ + φ₂h₂ + φ₃h₃
h₂ = a[θ₂₀ + θ₂₁x]
h₃ = a[θ₃₀ + θ₃₁x]
```

---

### 📌 활성화 함수 ReLU

> **음수는 0으로, 양수는 그대로 통과시키는 함수**

```
        { 0   (z < 0 일 때)
a[z] = {
        { z   (z ≥ 0 일 때)

z = 3  → a[3] = 3   (그대로)
z = -5 → a[-5] = 0  (0으로)
```

```
그래프:
출력
5|              /
 |            /
0|──────────/──── z
 (음수 구간은 전부 0)
```

**왜 이렇게 간단한 함수를 쓰나요?**
```
1. 계산이 매우 빠름
2. 여러 ReLU를 조합하면 어떤 복잡한 함수도 근사 가능
3. 딥러닝에서 실제로 매우 잘 작동함
```

---

### 📌 모수(Parameter)

```
위 네트워크의 모수 (총 10개):
φ = {φ₀, φ₁, φ₂, φ₃,    ← 출력층 가중치 (4개)
     θ₁₀, θ₁₁,            ← 은닉 유닛 1 (2개)
     θ₂₀, θ₂₁,            ← 은닉 유닛 2 (2개)
     θ₃₀, θ₃₁}            ← 은닉 유닛 3 (2개)
```

**학습 원리:**
```
훈련 데이터로 손실함수(Loss) 정의
→ Loss를 최소화하는 방향으로 10개의 모수를 반복 조정
→ 최적의 φ 발견 = 학습 완료!
```

---

## 3. 조각별 선형 함수 (Piecewise Linear)

> **여러 개의 ReLU를 합치면 꺾이는 복잡한 곡선을 표현할 수 있습니다!**

```
ReLU 하나:          ReLU 여러 개 합치기:
         /               /\
────────/          ──────  ──────
                      산 모양!

더 많은 ReLU:
  /\/\
 /    \
/      \──
→ 더 복잡한 모양 가능!
```

---

## 4. 보편적 근사 정리 (Universal Approximation Theorem)

> **"Hidden Unit을 충분히 많이 갖는다면, 얕은 신경망은 임의의 연속함수를 임의의 정밀도로 근사할 수 있다"**

이건 수학적으로 **증명된 정리**입니다!

### 📊 Hidden Unit 수에 따른 근사 정확도

```
a) 5개 Linear Regions → 대충 비슷
b) 10개 Linear Regions → 꽤 비슷
c) 20개 Linear Regions → 거의 같음
...
무한히 많으면 → 완벽히 일치!
```

### ⚠️ 현실적인 한계

```
이론: "충분히 많은 유닛이면 어떤 함수든 표현 가능"
현실:
1. "충분히 많은"이 얼마인지 모름
2. 표현 가능 ≠ 학습 가능
3. 유닛이 너무 많으면 과적합 위험
4. 계산 비용 증가
```

## 5. 네트워크 일반화

### 📌 출력이 여러 개인 경우 (1 input, 4 hidden, 2 output)

```
y₁ = φ₁₀ + φ₁₁h₁ + φ₁₂h₂ + φ₁₃h₃ + φ₁₄h₄
y₂ = φ₂₀ + φ₂₁h₁ + φ₂₂h₂ + φ₂₃h₃ + φ₂₄h₄

→ 은닉층은 공유하지만, 출력층 가중치는 각각 다름
→ 같은 은닉 특징(h)을 보고 y₁, y₂ 각자의 방식으로 조합
```

### 📌 입력이 여러 개인 경우 (2 input, 3 hidden, 1 output)

```
h₁ = a[θ₁₀ + θ₁₁x₁ + θ₁₂x₂]  ← x₁, x₂ 둘 다 받음
h₂ = a[θ₂₀ + θ₂₁x₁ + θ₂₂x₂]
h₃ = a[θ₃₀ + θ₃₁x₁ + θ₃₂x₂]
y = φ₀ + φ₁h₁ + φ₂h₂ + φ₃h₃
```

### 📌 완전 일반화 (Dᵢ개 입력, D개 은닉, Dₒ개 출력)

```
              Dᵢ
hd = a[ θd₀ + Σ θdi·xᵢ ]     (d번째 은닉 유닛)
             i=1

         D
yⱼ = φⱼ₀ + Σ φⱼd·hd           (j번째 출력)
           d=1
```

**총 파라미터 수:**
```
= D × (Dᵢ + 1) + Dₒ × (D + 1)
```

## 6. 딥 네트워크 (Deep Network) — 층 쌓기

> **Shallow Network 여러 개를 직렬 연결 = Deep Network**

---

### 💡 왜 층을 쌓으면 강력해지나? — "접기" 비유

```
층 1개 (Shallow): 종이를 한 번 접음 → 꺾임 3~4개

층 2개 (Deep):    접힌 종이를 또 접음 → 꺾임이 기하급수적 증가!

구체적으로:
Shallow (은닉 유닛 6개): 최대 7개 선형 구역
Deep (3+3 유닛): 최대 16개 선형 구역!

→ 비슷한 파라미터 수여도 Deep이 훨씬 복잡한 함수를 표현!
```

---

### 📌 2층 네트워크 수식

```
1층 계산: h₁ = a[θ₀ + θ·x]
2층 계산: h₂ = a[ψ₀ + Ψ·h₁]
출력:      y  = φ₀ + φᵀh₂
```

---

### 📌 K층 일반화 공식 (행렬 표기)

```
h₁ = a[β₀ + Ω₀x]           ← 1층: 입력 → 1번째 은닉
h₂ = a[β₁ + Ω₁h₁]          ← 2층: 1번째 → 2번째 은닉
h₃ = a[β₂ + Ω₂h₂]          ← 3층: 2번째 → 3번째 은닉
  ⋮
y  = βₖ + Ωₖhₖ              ← 출력층
```

**하나로 합친 수식 (중첩 함수):**
```
y = βₖ + Ωₖ·a[βₖ₋₁ + Ωₖ₋₁·a[...β₁ + Ω₁·a[β₀ + Ω₀x]]...]]
```

---

### 📌 Shallow vs Deep 비교

| 항목 | Shallow (1층, 6유닛) | Deep (2층, 3+3유닛) |
|------|---------------------|---------------------|
| 파라미터 | 19개 | 20개 (거의 같음!) |
| 선형 구역 | 최대 7개 | 최대 16개 |
| 표현력 | 낮음 | **훨씬 높음** |

---

## 7. 2D 입력에서의 신경망

입력이 2개(x₁, x₂)이면, 공간을 여러 구역으로 나누는 **결정 경계**가 만들어집니다.

```
1D 입력 네트워크: 꺾이는 선 → 1D 함수 근사
2D 입력 네트워크: 접히는 평면 → 2D 함수 근사

각 은닉 유닛 = 2D 공간을 나누는 "경계선" 하나
은닉 유닛이 많을수록 더 복잡한 경계 표현 가능!
```

---

## 8. 전체 흐름 정리

```
선형 모델의 한계
(직선만 표현 가능)
        ↓
활성화 함수(ReLU) 도입
(비선형 표현 가능)
        ↓
Hidden Units (은닉 유닛)
각각 다른 각도로 x를 처리
        ↓
Shallow Network (1층)
조각별 선형 함수로 복잡한 패턴 근사
        ↓
Deep Network (여러 층)
층을 쌓을수록 표현력 기하급수적 증가
        ↓
K층 일반화
hₖ = a[βₖ₋₁ + Ωₖ₋₁hₖ₋₁]
```

> **핵심 요약:** 신경망은 ReLU 활성화 함수를 가진 은닉 유닛들을 여러 층으로 쌓아, 어떤 복잡한 함수도 근사할 수 있는 강력한 모델입니다. 층이 깊어질수록 표현력이 기하급수적으로 증가합니다.
