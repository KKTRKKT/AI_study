# 조건부 확률과 시그모이드 함수 정리

> 분류(Classification) 문제의 핵심 개념 두 가지를 정리한 문서입니다.

---

## 1. P(Y = k | X) — 조건부 확률

### 정의

$$P(Y = k \mid X)$$

**"X(입력 정보)가 주어졌을 때, 정답 Y가 k라는 범주에 속할 확률"**

| 기호 | 의미 |
|------|------|
| `P(...)` | ~일 확률 |
| `Y = k` | 정답이 k 범주다 (예: 스팸, 고양이, 암) |
| `\| X` | X(입력 데이터)가 주어졌을 때 |

---

### 왜 사용하는가?

분류 모델은 단순히 "정답이 뭐야?"라고 답하는 것보다 **확률**로 답하는 게 훨씬 유용합니다.

```
직접 예측: "이 메일은 스팸입니다"         ← 이분법적, 확신도 모름

확률 예측: "이 메일이 스팸일 확률은 94%"  ← 유연한 의사결정 가능
```

확률 형태로 출력하면 세 가지가 가능해집니다.

1. **불확실한 상황에서 신중한 의사결정** → 확률이 55%이면 추가 검토
2. **임계값(threshold) 조절** → 상황에 따라 기준을 70% 또는 30%로 설정 가능
3. **여러 범주 동시 비교** → 가장 높은 확률의 범주를 최종 예측으로 선택

---

### 어떻게 사용하는가?

```
1. 입력 X를 모델에 넣는다
      ↓
2. 각 범주 k에 대해 확률을 계산한다
   P(스팸|X) = 0.94
   P(정상|X) = 0.05
   P(피싱|X) = 0.01
      ↓
3. 모든 확률의 합 = 1 (100%)
      ↓
4. 가장 높은 확률의 범주를 최종 예측으로 선택
   → argmax_k P(Y=k|X) = "스팸"
```

#### 실제 예시: 이메일 스팸 분류

| 입력 X | 정상 메일 | 스팸 | 피싱 | 판정 |
|--------|---------|------|------|------|
| 스팸 단어 20%, 신뢰 발신자, 링크 1개 | 80% | 15% | 5% | 정상 |
| 스팸 단어 70%, 불신 발신자, 링크 8개 | 5% | 90% | 5% | 스팸 |

#### 확신도에 따른 처리 방식

| 최고 확률 | 처리 방식 |
|---------|---------|
| 85% 이상 | 자동 처리 가능 |
| 60~84% | 표준 처리 |
| 60% 미만 | 추가 검토 권장 |

## 2. 시그모이드 함수 (Sigmoid Function)

### 정의

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

**어떤 숫자든 0과 1 사이의 값으로 압축해주는 S자 모양의 함수**

---

### 핵심 성질

| 성질 | 내용 |
|------|------|
| **출력 범위** | 항상 (0, 1) — 절대 0 미만이나 1 초과가 되지 않음 |
| **z = 0일 때** | σ(0) = 0.5 → 정확히 50% |
| **비선형 변환** | z가 선형으로 변해도 σ(z)는 비선형으로 변함 |

#### z 값에 따른 출력 변화

| z 값 | σ(z) 출력 | 해석 |
|------|---------|------|
| -∞ | → 0 | 확률 거의 0% |
| -6 | ≈ 0.002 | 약 0.2% |
| -2 | ≈ 0.119 | 약 12% |
| **0** | **= 0.5** | **정확히 50%** |
| +2 | ≈ 0.880 | 약 88% |
| +6 | ≈ 0.998 | 약 99.8% |
| +∞ | → 1 | 확률 거의 100% |

---

### 왜 사용하는가?

분류 문제에서 모델의 출력은 반드시 **확률(0~1)**이어야 합니다.  
그런데 선형 모델 `z = β₀ + β₁x`의 출력은 −∞ ~ +∞로 어떤 값도 될 수 있습니다.

```
선형 모델 그대로 사용하면:
x가 매우 작을 때: y = -0.5  ← 확률이 음수?! ❌
x가 매우 클 때:   y = 1.5   ← 확률이 1 초과?! ❌

시그모이드를 사용하면:
σ(-0.5) = 0.378  ✅  (0~1 사이)
σ(1.5)  = 0.818  ✅  (0~1 사이)
```

시그모이드는 이 무한한 범위의 숫자를 0~1 사이로 눌러줘서 **확률로 해석할 수 있게 만들어주는 변환기** 역할을 합니다.

---

### 어떻게 사용하는가? — 로지스틱 회귀와의 연결

시그모이드는 단독으로 쓰이지 않고, 항상 **선형 모델의 출력 z를 받아서 확률로 바꾸는** 역할을 합니다.

```
1단계: 선형 모델로 z 계산
z = β₀ + β₁x₁ + β₂x₂ + ...

       ↓ 시그모이드 적용

2단계: 확률로 변환
P(Y=1|X) = σ(z) = 1 / (1 + e^{-z})
```

#### 실제 예시: 신용카드 연체 예측

자료의 로지스틱 회귀 모형식:

$$\log\left(\frac{p}{1-p}\right) = -10.6513 + 0.0055 \times \text{Balance}$$

**Balance = 1,000일 때:**

```
z = -10.6513 + 0.0055 × 1000 = -5.15
σ(-5.15) = 1 / (1 + e^5.15) ≈ 0.006
→ 연체 확률 약 0.6% (정상 판정)
```

**Balance = 2,000일 때:**

```
z = -10.6513 + 0.0055 × 2000 = 0.35
σ(0.35) = 1 / (1 + e^{-0.35}) ≈ 0.587
→ 연체 확률 약 58.7% (연체 위험 판정)
```

> **비선형성 포인트:** 입력이 2배 증가했는데 확률은 0.6% → 58.7%로 약 100배 상승!  
> 이것이 시그모이드의 비선형 변환 효과입니다.

---

### 선형 회귀 vs 로지스틱 회귀(시그모이드) 비교

| 구분 | 선형 회귀 | 로지스틱 회귀 |
|------|---------|------------|
| **출력** | `y = z` (무제한) | `y = σ(z)` (0~1) |
| **출력 범위** | −∞ ~ +∞ | 0 ~ 1 |
| **확률로 사용** | ❌ 불가 | ✅ 가능 |
| **사용 목적** | 연속값 예측 | 분류 (확률 예측) |

---

### 딥러닝에서의 시그모이드

딥러닝에서는 시그모이드를 **출력층 활성화 함수**로 사용합니다.

```
이진 분류 출력층: 시그모이드 사용 ✅
다중 분류 출력층: 소프트맥스(Softmax) 사용
은닉층(Hidden Layer): ReLU 사용 권장

⚠️ 시그모이드를 은닉층에 잘 쓰지 않는 이유:
   z가 극단값(±5 이상)일 때 기울기 ≈ 0
   → 경사 소실(Vanishing Gradient) 문제 발생
```

## 3. 전체 흐름 정리

```
분류 문제 발견
(출력이 카테고리)
        ↓
확률로 표현: P(Y=k|X)
"이 데이터가 k 범주일 확률이 얼마인가?"
        ↓
선형 모델로 z 계산
z = β₀ + β₁x₁ + β₂x₂ + ...
        ↓
시그모이드로 0~1 변환
σ(z) = 1 / (1 + e^{-z})
        ↓
확률 비교 후 범주 선택
argmax_k P(Y=k|X)
        ↓
MLE로 파라미터 β 학습
```

## 4. 핵심 한줄 요약

| 개념 | 한줄 요약 |
|------|---------|
| **P(Y=k\|X)** | X가 주어졌을 때 Y가 k일 확률 — 분류 모델의 출력 형태 |
| **시그모이드** | 어떤 숫자든 0~1 사이로 압축하는 S자 함수 |
| **두 개념의 연결** | 선형 모델 출력 z → 시그모이드 → P(Y=1\|X) |
| **사용 이유** | 확률은 반드시 0~1이어야 하기 때문 |
| **판정 기준** | σ(z) ≥ 0.5 → 양성 / σ(z) < 0.5 → 음성 |
