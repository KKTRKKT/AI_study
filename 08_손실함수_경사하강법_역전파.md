# ⚙️ 손실함수 · 경사하강법 · 역전파

> 딥러닝 학습의 핵심 메커니즘을 단계별로 이해합니다.

---

## 1. 손실함수 (Loss Function)

> **현재 모델이 얼마나 잘못 예측하고 있는지를 숫자 하나로 나타내는 함수**

---

### 💡 핵심 개념

```
좋은 모델 → 손실 작음 → L[φ] ≈ 0
나쁜 모델 → 손실 큼  → L[φ] = 7.11
```

### 📌 손실함수 수식

```
L[φ] = L[φ, f[xᵢ, φ], {xᵢ, yᵢ}ᵢ₌₁ᴵ]

간단 표현: L[φ]
→ "파라미터 φ가 주어졌을 때의 손실값"
→ φ를 잘 선택할수록 L[φ]가 작아짐
```

### 📌 선형회귀의 손실함수 (MSE)

```
        I
L[φ] = Σ (f[xᵢ, φ] - yᵢ)²
       i=1

     I
   = Σ (φ₀ + φ₁xᵢ - yᵢ)²
    i=1

f[xᵢ, φ] = φ₀ + φ₁xᵢ  ← 모델의 예측값
yᵢ                      ← 실제 정답값
```

---

## 2. 학습이란?

> **손실함수를 최소화하는 파라미터 φ를 찾는 것**

```
φ̂ = argmin L[φ]
      φ

"φ에 대해 L[φ]를 최소로 만드는 φ를 찾아라"
```

### 🏔️ 산에서 골짜기 찾기 비유

```
L[φ]를 산의 높이라고 생각하면:

파라미터(φ) = 산 위의 위치 좌표
목표 = 가장 낮은 골짜기(최솟값) 찾기!
학습 = 산 위에서 가장 낮은 지점으로 내려가는 과정
```

---

## 3. 미분을 이용한 최적화

### 💡 미분이란?

> **"현재 위치에서 함수가 얼마나 가파른가?"를 알려주는 도구**

```
∂y/∂x = "x가 조금 변할 때 y가 얼마나 변하는가"

기울기 < 0 → 오른쪽 가면 y 감소 → 오른쪽으로 이동
기울기 > 0 → 오른쪽 가면 y 증가 → 왼쪽으로 이동
기울기 = 0 → 최솟값(또는 최댓값) 도달!
```

### 🔢 예시: y = x² - 4x + 5

```
미분: ∂y/∂x = 2x - 4

x = 1일 때: 기울기 = 2(1)-4 = -2 (음수 → 오른쪽 이동)
x = 2일 때: 기울기 = 2(2)-4 = 0  (최솟값! 멈춤)
x = 4일 때: 기울기 = 2(4)-4 = +4 (양수 → 왼쪽 이동)

→ x = 2에서 최솟값 (y = 1)
```

## 4. 경사하강법 (Gradient Descent)

> **손실이 줄어드는 방향으로 파라미터를 조금씩 이동시키는 알고리즘**

---

### 📌 업데이트 공식

```
φ ← φ - α × (∂L/∂φ)

풀어쓰면:
φ₀ ← φ₀ - α × (∂L/∂φ₀)
φ₁ ← φ₁ - α × (∂L/∂φ₁)
```

### 📌 α (학습률, Learning Rate)

```
α 너무 클 때:  너무 멀리 이동 → 최솟값 지나침 😱
α 너무 작을 때: 너무 느리게 이동 → 학습이 오래 걸림 😴
α 적당할 때:  빠르고 안정적으로 수렴 😊
```

---

### 📌 선형회귀의 기울기 계산

```
전체 손실의 기울기 = 각 데이터 기울기의 합:

∂L/∂φ = Σᵢ ∂ℓᵢ/∂φ

각 데이터 i의 기울기:
∂ℓᵢ/∂φ₀ = 2(φ₀ + φ₁xᵢ - yᵢ)       = 2 × 잔차
∂ℓᵢ/∂φ₁ = 2xᵢ(φ₀ + φ₁xᵢ - yᵢ)     = 2xᵢ × 잔차
```

### 🔢 업데이트 예시

```
가정: φ₀=0.4, φ₁=0.2, 학습률 α=0.1
데이터: x₁=1.0, y₁=1.5

잔차 = 0.4 + 0.2×1.0 - 1.5 = -0.9

∂ℓ₁/∂φ₀ = 2×(-0.9) = -1.8
∂ℓ₁/∂φ₁ = 2×1.0×(-0.9) = -1.8

업데이트:
φ₀ ← 0.4 - 0.1×(-1.8) = 0.58  (절편 증가)
φ₁ ← 0.2 - 0.1×(-1.8) = 0.38  (기울기 증가)
→ 예측값이 실제값(1.5)에 더 가까워짐!
```

## 5. Convex vs Non-convex 문제

### 📌 Convex (볼록 함수) ✅

```
특징: U자 모양 (아래로 볼록)
→ 어디서 시작해도 항상 같은 전역 최솟값으로 수렴
→ 지역 최솟값이 없음!
→ 경사하강법으로 반드시 최적해 찾을 수 있음!

예시: 선형회귀 손실함수
```

### 📌 Non-convex (비볼록 함수) ⚠️

```
특징: 봉우리·골짜기가 여러 개
→ 지역 최솟값(Local Minimum) 문제 발생!
→ 시작 위치에 따라 다른 결과

예시: 딥러닝 손실함수
```

| 용어 | 의미 | 위험성 |
|------|------|--------|
| **전역 최솟값 (Global Minimum)** | 함수 전체에서 가장 낮은 점 | 이게 진짜 최적해 |
| **지역 최솟값 (Local Minimum)** | 주변에서만 가장 낮은 점 | 여기서 멈춰버림 ⚠️ |
| **안장점 (Saddle Point)** | 기울기=0이지만 최솟값 아님 | 학습이 느려짐 ⚠️ |

## 6. 확률적 경사하강법 (SGD)

### ⚠️ 기존 경사하강법의 문제점

```
문제 1: 지역 최솟값에 빠지면 탈출 불가
문제 2: 매 스텝마다 전체 데이터 계산 → 느림

예) 이미지넷: 120만 장
→ 파라미터 1번 업데이트에 120만 번 계산 → 사실상 불가능
```

### 💡 해결책: 미니배치 SGD

> **전체 데이터 대신, 무작위로 선택한 일부(미니배치)만 사용해서 기울기 계산**

```
기존 경사하강법 (Full Batch):
φₜ₊₁ ← φₜ - α × Σᵢ₌₁ᴵ [∂ℓᵢ/∂φ]  ← i=1부터 I까지 전부

확률적 경사하강법 (SGD):
φₜ₊₁ ← φₜ - α × Σᵢ∈𝓑ₜ [∂ℓᵢ/∂φ]  ← 𝓑ₜ = 무작위 선택된 배치만
```

### 📌 미니배치란?

```
전체 데이터: 10,000개
배치 크기:   100개

→ 매 업데이트마다 무작위로 100개만 선택해서 사용!

같은 계산량으로:
GD:  1번 업데이트 가능
SGD: 10,000번 업데이트 가능!
```

---

### 📊 SGD의 4가지 핵심 특성

#### 1. 지역 최솟값 탈출 ✅

```
Full Batch GD: 지역 최솟값 → 기울기=0 → 완전히 멈춤 ❌
SGD:          지역 최솟값 → 미니배치 기울기 ≠ 0 → 탈출 가능! ✅

비유:
GD = 무거운 당구공 (구멍에 빠지면 절대 못 나옴)
SGD = 작은 고무공 (구멍에서도 튀어 나올 수 있음!)
```

#### 2. 계산 비용 절감 ✅

```
Full Batch: 1번 업데이트 = 1,000,000번 계산
SGD (배치 100): 1번 업데이트 = 100번 계산 (10,000배 빠름!)
```

#### 3. 노이즈가 있지만 타당한 업데이트 ✅

```
미니배치 기울기 ≠ 정확한 전체 기울기
하지만: E[미니배치 기울기] ≈ 전체 기울기
→ 평균적으로 올바른 방향을 가리킴!
```

#### 4. 수렴 특성 (지그재그) ⚠️

```
Full Batch GD: → → → ● (깔끔하게 수렴, 지역 최솟값에 갇힘)
SGD:          ↗↙↘↗↙↘ ≈● (지그재그, 전역 최솟값 근처 탐색)
```

### 📋 배치 크기별 비교

| 항목 | Pure SGD(1) | Mini-batch(32~512) | Full Batch |
|------|------------|-------------------|------------|
| 계산량/스텝 | 매우 적음 | 적당 | 매우 많음 |
| 기울기 정확도 | 낮음 | 중간 | 정확 |
| 지역 탈출 | 쉬움 | 가능 | 어려움 |
| 실용성 | 낮음 | **최고** ✅ | 대용량 불가 |

## 7. 역전파 (Backpropagation)

> **출력의 손실에서 시작해서 그래프를 거꾸로 따라가며 연쇄법칙으로 각 파라미터의 기울기를 계산하는 절차**

---

### 💡 왜 필요한가?

```
딥 네트워크에서 첫 번째 층(Ω₀)은 손실(ℓ)에 직접 연결되지 않음!

경로: Ω₀ → f₀ → h₁ → f₁ → h₂ → f₂ → h₃ → f₃ → ℓ

→ 이 긴 경로를 통해 기울기를 계산해야 함
→ 연쇄법칙(Chain Rule)이 핵심 도구!
```

---

### 📌 연쇄법칙 (Chain Rule)

> **합성함수의 변화율 = 바깥 함수의 변화율 × 안쪽 함수의 변화율**

```
y = f₁(f₀(x; Ω)) 일 때:

∂y/∂Ω = [∂f₁/∂f₀] × [∂f₀/∂Ω]
          ↑              ↑
       바깥 함수 미분   안쪽 함수 미분

예시:
y = (3x + 2)²

u = 3x + 2, y = u²
dy/dx = (dy/du) × (du/dx)
      = 2u × 3 = 6(3x+2)
```

---

### 📌 순방향(Forward) vs 역방향(Backward)

```
순방향 전파 (예측값 계산):
x → f₀ → h₁ → f₁ → h₂ → f₂ → h₃ → f₃ → ℓ

역방향 전파 (기울기 계산):
ℓ → ∂ℓ/∂f₃ → ∂ℓ/∂h₃ → ∂ℓ/∂f₂ → ∂ℓ/∂h₂ → ∂ℓ/∂f₁ → ∂ℓ/∂h₁ → ∂ℓ/∂f₀
         ↓              ↓              ↓              ↓
    ∂ℓ/∂Ω₃,β₃    ∂ℓ/∂Ω₂,β₂    ∂ℓ/∂Ω₁,β₁    ∂ℓ/∂Ω₀,β₀
```

---

### 📌 각 층의 파라미터 기울기 공식

**선형 층 fₖ = βₖ + Ωₖhₖ 에서:**

```
∂ℓ/∂Ωₖ = (∂ℓ/∂fₖ) × hₖᵀ    ← 역전파 기울기 × 입력의 전치
∂ℓ/∂βₖ = ∂ℓ/∂fₖ             ← 역전파 기울기 그대로
```

---

### 💡 역전파가 효율적인 이유

```
순진한 방법: 각 파라미터마다 처음부터 다시 계산 (중복 계산 엄청남!)

역전파:     계산된 값을 저장하고 재사용!
→ 각 값을 한 번만 계산하고 재활용
→ 계산량이 순방향과 거의 동일하게 효율적!
```

## 8. 전체 학습 과정 정리

```
[순방향 전파 - Forward Pass]
입력 x
  ↓ Ω₀ 적용
f₀ = β₀ + Ω₀x    → 저장
  ↓ ReLU 활성화
h₁ = a[f₀]        → 저장
  ↓ ...반복...
  ↓
ℓ 계산 (손실)

[역방향 전파 - Backward Pass]
손실 ℓ에서 출발
  ↑ 연쇄법칙 적용
f₃ 기울기 계산    → ∂ℓ/∂Ω₃, ∂ℓ/∂β₃ 추출
  ↑
h₃ 기울기 계산
  ↑
f₂ 기울기 계산    → ∂ℓ/∂Ω₂, ∂ℓ/∂β₂ 추출
  ↑ ...반복...
f₀ 기울기 계산    → ∂ℓ/∂Ω₀, ∂ℓ/∂β₀ 추출

[파라미터 업데이트 - SGD]
Ωₖ ← Ωₖ - α × ∂ℓ/∂Ωₖ
βₖ ← βₖ - α × ∂ℓ/∂βₖ
```

## 9. 핵심 개념 비교표

| 개념 | 의미 | 비유 |
|------|------|------|
| **손실함수** | 모델이 얼마나 틀렸는가 | 시험 점수의 반대 (낮을수록 좋음) |
| **기울기(Gradient)** | 손실이 증가하는 방향 | 산의 경사 방향 |
| **경사하강법** | 기울기 반대 방향으로 φ 조정 | 산에서 내려오기 |
| **학습률 α** | 한 번에 이동하는 크기 | 걸음 크기 |
| **SGD** | 미니배치로 기울기 근사 | 전체 지도 대신 주변만 보며 내려오기 |
| **역전파** | 연쇄법칙으로 모든 기울기 계산 | 한 번에 모든 계단 정보 수집 |
| **Convex** | 전역 최솟값 1개 | U자 그릇 |
| **Non-convex** | 지역 최솟값 여러 개 | 울퉁불퉁한 지형 |

---

> **핵심 요약:** 딥러닝 학습은 ①손실함수로 오차를 측정하고, ②역전파로 모든 파라미터의 기울기를 효율적으로 계산한 뒤, ③SGD로 파라미터를 업데이트하는 과정을 반복합니다. 이 세 가지가 딥러닝 학습의 심장입니다.
