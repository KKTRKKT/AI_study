## 용어 정리

| 용어 | 원문 | 한 줄 설명 |
|---|---|---|
| 민감도 | Sensitivity | 실제 양성 중에서 양성으로 맞게 예측한 비율이에요 |
| 특이도 | Specificity | 실제 음성 중에서 음성으로 맞게 예측한 비율이에요 |
| 진양성 | True Positive (TP) | 양성이라고 예측했고 실제로 양성이에요 |
| 진음성 | True Negative (TN) | 음성이라고 예측했고 실제로 음성이에요 |
| 위양성 | False Positive (FP) | 양성이라고 예측했지만 실제론 음성이에요 |
| 위음성 | False Negative (FN) | 음성이라고 예측했지만 실제론 양성이에요 |

## 1. 왜 민감도와 특이도가 필요한가요?

### 🤔 문제 상황

혼동 행렬만으로는 모델을 선택하기 어려운 경우가 있어요.

예를 들어 로지스틱 회귀와 랜덤 포레스트를 비교해봤더니 두 혼동 행렬이 아주 비슷했어요. 어느 게 더 낫다고 딱 잘라 말하기 어려운 상황이에요. 그럴 때 더 명확한 기준이 필요해요.

또 다른 문제도 있어요. 두 모델이 "전체적인 정확도"는 비슷한데, 한 쪽은 양성 환자를 잘 잡아내고 다른 쪽은 음성 환자를 잘 걸러낸다면? 상황에 따라 어느 모델이 더 중요한지가 달라져요.

### 한 줄 정의

> **민감도(Sensitivity):** 실제로 양성인 사람 중에서, 모델이 양성으로 맞게 예측한 비율이에요. "진짜 환자를 얼마나 잡아냈나"를 측정해요.

> **특이도(Specificity):** 실제로 음성인 사람 중에서, 모델이 음성으로 맞게 예측한 비율이에요. "건강한 사람을 얼마나 정확히 걸러냈나"를 측정해요.

---

## 2. 민감도 계산하기

### 공식

```
민감도(Sensitivity) = TP / (TP + FN)
```

**분모(TP + FN):** 실제 양성인 사람 전체예요. 양성을 맞게 예측한 것(TP) + 양성을 놓친 것(FN)이 합쳐진 게 "실제 양성 전체"예요.
**분자(TP):** 그 중 모델이 맞게 잡아낸 것만이에요.

### 일상 비유

> 💡 **비유:** 학교에서 독감 걸린 학생 100명을 검사했어요. 민감도는 "그 100명 중 몇 명을 독감이라고 제대로 찾아냈는가"예요. 민감도가 90%면 100명 중 90명을 잡아냈고 10명은 놓쳤다는 뜻이에요.

### 실제 계산 — 로지스틱 회귀

혼동 행렬:

| | 실제: 심장 질환 있음 | 실제: 심장 질환 없음 |
|---|---|---|
| **예측: 있음** | 139 (TP) | 20 (FP) |
| **예측: 없음** | 32 (FN) | 112 (TN) |

```
민감도 = TP / (TP + FN)
       = 139 / (139 + 32)
       = 139 / 171
       ≈ 0.81 (81%)
```

**해석:** 심장 질환이 있는 환자 171명 중 81%인 139명을 제대로 잡아냈어요. 32명은 놓쳤어요.

---

## 3. 특이도 계산하기

### 공식

```
특이도(Specificity) = TN / (TN + FP)
```

**분모(TN + FP):** 실제 음성인 사람 전체예요. 음성을 맞게 예측한 것(TN) + 음성을 잘못 양성이라고 한 것(FP)이 합쳐진 게 "실제 음성 전체"예요.
**분자(TN):** 그 중 모델이 맞게 음성이라고 판정한 것이에요.

### 일상 비유

> 💡 **비유:** 학교에서 독감에 걸리지 않은 학생 200명을 검사했어요. 특이도는 "그 200명 중 몇 명을 독감 아님이라고 제대로 판단했는가"예요. 특이도가 85%면 200명 중 170명은 정확히 "건강"이라고 했고, 30명은 잘못 "독감"이라고 했다는 뜻이에요.

### 실제 계산 — 로지스틱 회귀

```
특이도 = TN / (TN + FP)
       = 112 / (112 + 20)
       = 112 / 132
       ≈ 0.85 (85%)
```

**해석:** 심장 질환이 없는 사람 132명 중 85%인 112명을 정확히 건강하다고 했어요. 20명은 잘못 환자로 봤어요.

---

## 4. 두 모델 비교 — 로지스틱 회귀 vs 랜덤 포레스트

### 랜덤 포레스트 계산

혼동 행렬:

| | 실제: 심장 질환 있음 | 실제: 심장 질환 없음 |
|---|---|---|
| **예측: 있음** | 142 (TP) | 22 (FP) |
| **예측: 없음** | 29 (FN) | 110 (TN) |

```
민감도 = 142 / (142 + 29) = 142 / 171 ≈ 0.83 (83%)
특이도 = 110 / (110 + 22) = 110 / 132 ≈ 0.83 (83%)
```

### 두 모델 최종 비교

| 지표 | 로지스틱 회귀 | 랜덤 포레스트 |
|---|---|---|
| 민감도(Sensitivity) | 81% | **83%** |
| 특이도(Specificity) | **85%** | 83% |

- **민감도:** 랜덤 포레스트가 약간 더 높아요. 양성 환자를 더 잘 잡아내요.
- **특이도:** 로지스틱 회귀가 약간 더 높아요. 건강한 사람을 더 잘 걸러내요.

### 어떤 모델을 선택해야 할까요?

답은 **"목적에 따라 달라요."**

> **심장 질환 환자를 절대로 놓쳐서는 안 되는 상황이라면?**
> → 민감도가 높은 **랜덤 포레스트**를 선택해요. 양성 환자를 더 잘 잡아내요.

> **건강한 사람이 불필요한 치료를 받는 걸 최소화해야 하는 상황이라면?**
> → 특이도가 높은 **로지스틱 회귀**를 선택해요. 건강한 사람을 더 정확히 가려내요.

> 💡 **비유:** 두 명의 경비원을 비교해봐요. 경비원 A는 수상한 사람을 99% 잡아내지만 멀쩡한 사람도 가끔 막아요. 경비원 B는 멀쩡한 사람을 99% 통과시키지만 수상한 사람을 가끔 놓쳐요. 건물 보안이 최우선이라면 A를, 방문객 편의가 중요하다면 B를 고르는 거예요.

## 5. 3×3 혼동 행렬에서의 민감도와 특이도

### 🤔 왜 복잡해지나요?

카테고리가 3개 이상이면 "양성"과 "음성"이라는 개념이 명확하지 않아요. "트롤 2를 좋아하는 사람"을 기준으로 보면, 나머지 두 영화를 좋아하는 사람이 모두 "음성"이 되는 식이에요. 즉, **카테고리마다 별도로** 민감도와 특이도를 계산해야 해요.

### 3×3 혼동 행렬

세 영화(트롤 2, 고어 폴리스, 쿨 애즈 아이스) 선호도 예측 결과예요.

| | 실제: 트롤2 | 실제: 고어폴리스 | 실제: 쿨애즈아이스 |
|---|---|---|---|
| **예측: 트롤2** | 12 | 102 | 77 |
| **예측: 고어폴리스** | 112 | 23 | 93 |
| **예측: 쿨애즈아이스** | 83 | 92 | 17 |

### "트롤 2" 기준으로 계산

**트롤 2를 "양성"으로, 나머지 두 영화를 "음성"으로 설정해요.**

- **TP:** 트롤 2를 좋아하는데 트롤 2로 맞게 예측 = **12**
- **FN:** 트롤 2를 좋아하는데 다른 영화로 잘못 예측 = 112 + 83 = **195**
- **TN:** 다른 영화를 좋아하는데 다른 영화로 맞게 예측 = 23 + 77 + 92 + 17 = **209**
- **FP:** 다른 영화를 좋아하는데 트롤 2로 잘못 예측 = 102 + 77 = **179**

> ⚠️ **잠깐!** 위양성 계산이 헷갈릴 수 있어요. "다른 영화를 좋아하는데 트롤 2로 예측"한 것들이에요. 표에서 "예측: 트롤 2" 행 중 TP를 제외한 나머지예요 (102 + 77 = 179).

```
민감도(트롤2) = TP / (TP + FN)
             = 12 / (12 + 195)
             = 12 / 207
             ≈ 0.06 (6%)
```

트롤 2를 좋아하는 사람 중 고작 **6%**만 올바르게 예측했어요.

```
특이도(트롤2) = TN / (TN + FP)
             = 209 / (209 + 179)
             = 209 / 388
             ≈ 0.54 (54%)
```

트롤 2를 좋아하지 않는 사람 중 **54%**를 제대로 "트롤 2 아님"으로 예측했어요.

### "고어 폴리스" 기준으로 계산

- **TP:** 23
- **FN:** 102 + 92 = 194 (고어폴리스를 좋아하지만 다른 영화로 예측)
- **TN:** 12 + 77 + 83 + 17 = 189 (다른 영화를 좋아하고 다른 영화로 맞게 예측)
- **FP:** 112 + 93 = 205 (다른 영화를 좋아하지만 고어폴리스로 예측)

```
민감도(고어폴리스) = 23 / (23 + 194) = 23 / 217 ≈ 0.11 (11%)
특이도(고어폴리스) = 189 / (189 + 205) = 189 / 394 ≈ 0.48 (48%)
```

### 전체 카테고리 요약

| 카테고리 | 민감도 | 특이도 |
|---|---|---|
| 트롤 2 | 6% | 54% |
| 고어 폴리스 | 11% | 48% |
| 쿨 애즈 아이스 | 각자 계산해봐요 | 각자 계산해봐요 |

전반적으로 민감도가 아주 낮아요. 이 모델은 어느 영화를 좋아하는지 잘 맞추지 못하고 있어요.

> 💡 **보충 설명:** 카테고리가 많아질수록 카테고리별로 민감도/특이도를 각각 계산해야 해서 복잡해져요. 실무에서는 이걸 "클래스별 리포트"로 자동 계산해주는 도구들을 많이 사용해요.

## 6. 정리 — 민감도 vs 특이도, 언제 무엇을 중시할까요?

### 핵심 원칙

| 상황 | 중시해야 할 지표 | 이유 |
|---|---|---|
| 양성을 절대 놓치면 안 되는 경우 (암 검진, 바이러스 감지) | **민감도** | 위음성(놓치는 것)이 치명적이에요 |
| 음성을 잘못 양성으로 보면 안 되는 경우 (법원 판결, 신용 대출 거절) | **특이도** | 위양성(잘못 걸어내는 것)이 불공정해요 |
| 둘 다 중요한 경우 | 민감도와 특이도 **모두** 고려, 또는 ROC 곡선 활용 | 상황에 맞게 균형을 잡아요 |

> 💡 **ROC 곡선(ROC Curve)과 AUC:** 민감도와 특이도의 균형을 시각적으로 보여주는 도구예요. 이건 이후 강의에서 자세히 다뤄요!

## ⚠️ 주의할 점 / 자주 하는 실수

- **민감도와 특이도를 헷갈리지 마세요.** 민감도는 "실제 양성 중 맞춘 비율", 특이도는 "실제 음성 중 맞춘 비율"이에요. 분모가 다르다는 게 핵심이에요.
- **민감도와 특이도는 항상 같이 봐야 해요.** 민감도만 높이면 특이도가 낮아지는 경우가 많아요 (이것도 트레이드오프예요).
- **3개 이상 카테고리에서는 카테고리별로 따로 계산해요.** 전체에 대한 단일 민감도/특이도는 없어요.
- **민감도가 높다고 무조건 좋은 게 아니에요.** 극단적으로 "모든 사람을 양성"이라고 하면 민감도는 100%지만 특이도는 0%예요. 이 모델은 쓸모없어요.

**Q2.** 아래 혼동 행렬에서 민감도와 특이도를 각각 계산해보세요.

| | 실제: 양성 | 실제: 음성 |
|---|---|---|
| **예측: 양성** | 200 | 40 |
| **예측: 음성** | 50 | 160 |

<details>
<summary>정답 보기</summary>

**민감도 계산:**
- TP = 200, FN = 50
- 민감도 = 200 / (200 + 50) = 200 / 250 = **0.80 (80%)**

**특이도 계산:**
- TN = 160, FP = 40
- 특이도 = 160 / (160 + 40) = 160 / 200 = **0.80 (80%)**

**해석:** 이 모델은 양성 환자의 80%를 맞게 예측하고, 음성 환자의 80%를 맞게 예측해요.

</details>

**Q3.** "모든 환자를 양성"이라고 예측하는 모델의 민감도와 특이도는 각각 얼마인가요?

<details>
<summary>정답 보기</summary>

**정답:** 민감도 = 100%, 특이도 = 0%예요.

**설명:**
- 모든 사람을 "양성"이라고 하면, 실제 양성인 사람은 모두 TP가 돼요. FN이 0이에요. → 민감도 = TP / (TP + 0) = **100%**
- 실제 음성인 사람은 모두 FP가 돼요. TN이 0이에요. → 특이도 = 0 / (0 + FP) = **0%**

이 모델은 민감도는 완벽하지만 특이도는 0%예요. 쓸모없는 모델이에요. 민감도만 보면 완벽해 보이지만, 특이도까지 함께 봐야 이 함정을 피할 수 있어요.

</details>

**Q4.** 음주운전 단속 장비를 평가한다고 해봐요. 민감도와 특이도 중 어느 쪽을 더 중시해야 하나요?

<details>
<summary>정답 보기</summary>

**정답:** 두 가지 관점이 모두 있지만, 일반적으로 **민감도**를 더 중시해야 해요.

**설명:**
- 민감도를 중시하는 이유: 음주운전자를 놓치는 것(위음성)은 사고 위험이에요. 음주운전자가 도로에서 사고를 내면 생명을 위협해요. 이 위험을 최소화해야 해요.
- 특이도도 중요한 이유: 멀쩡한 사람을 음주운전으로 잘못 단속하면(위양성) 억울한 피해자가 생겨요.

현실에서는 둘 다 중요하기 때문에 두 지표의 균형을 고려해요. 하지만 공중 안전 측면에서는 민감도를 더 높이는 방향으로 설계하는 게 일반적이에요.

</details>

---

**Q5.** 민감도가 높고 특이도가 낮은 모델과, 민감도가 낮고 특이도가 높은 모델 중 어느 쪽이 "더 좋은" 모델인가요?

<details>
<summary>정답 보기</summary>

**정답:** 어느 쪽이 더 좋은지는 사용하는 **목적과 상황**에 따라 달라요.

**설명:**
- 민감도 높고 특이도 낮은 모델: 양성을 잘 잡아내지만 건강한 사람도 자주 양성으로 오해해요. 암 조기 발견처럼 "놓치면 안 되는" 상황에 적합해요.
- 민감도 낮고 특이도 높은 모델: 건강한 사람을 잘 걸러내지만 양성 환자를 가끔 놓쳐요. 잘못 양성으로 분류하면 심각한 부작용이 있는 상황(예: 매우 위험한 치료)에 적합해요.

"더 좋은 모델"은 절대적으로 없어요. 어떤 실수가 더 나쁜 결과를 낳는가를 먼저 정하고, 그에 맞는 지표를 기준으로 선택해요.

</details>

### 실전 적용 문제

**문제.** 유방암 조기 검진 AI 모델 A와 B가 있어요. 아래 혼동 행렬을 보고 어떤 모델을 선택해야 할지 결정해보세요.

**모델 A:**

| | 실제: 암 있음 | 실제: 암 없음 |
|---|---|---|
| **예측: 있음** | 180 | 90 |
| **예측: 없음** | 20 | 710 |

**모델 B:**

| | 실제: 암 있음 | 실제: 암 없음 |
|---|---|---|
| **예측: 있음** | 150 | 30 |
| **예측: 없음** | 50 | 770 |

1. 모델 A와 B의 민감도와 특이도를 각각 계산해보세요
2. 유방암 조기 검진이라는 맥락에서 어떤 모델을 선택하고 싶은지 이유와 함께 설명해보세요

**힌트:**
- 민감도 = TP / (TP + FN)
- 특이도 = TN / (TN + FP)
- 유방암에서 위음성(놓치는 것)과 위양성(잘못 걸어내는 것) 중 어느 게 더 심각한지 생각해봐요

<details>
<summary>예시 답안 보기</summary>

**모델 A:**
- 민감도 = 180 / (180 + 20) = 180 / 200 = **90%**
- 특이도 = 710 / (710 + 90) = 710 / 800 = **88.75%**

**모델 B:**
- 민감도 = 150 / (150 + 50) = 150 / 200 = **75%**
- 특이도 = 770 / (770 + 30) = 770 / 800 = **96.25%**

**선택: 모델 A**

유방암 조기 검진에서는 암이 있는 환자를 놓치는 것(위음성, FN)이 훨씬 더 위험해요. 암을 발견하지 못하면 치료 시기를 놓쳐서 생명을 위협할 수 있어요.

모델 A는 민감도가 90%로 더 높아서, 암 환자 200명 중 180명을 잡아내요 (모델 B는 150명). 20명만 놓쳐요 (모델 B는 50명).

모델 B는 특이도가 더 높아서 건강한 사람을 더 정확히 걸러내지만, 조기 검진 단계에서는 "양성 의심"이 나와도 추가 검사로 확인할 수 있어요. 건강한 사람이 추가 검사를 받는 건 불편하지만, 암 환자를 놓치는 것만큼 치명적이진 않아요.

따라서 유방암 조기 검진에서는 민감도를 더 중시하는 모델 A를 선택해요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>

## 🎯 핵심 요약

| 개념 | 한 줄 설명 |
|---|---|
| 🎯 민감도(Sensitivity) | 실제 양성 중 양성으로 맞게 예측한 비율이에요. TP / (TP + FN)으로 계산해요 |
| 🛡️ 특이도(Specificity) | 실제 음성 중 음성으로 맞게 예측한 비율이에요. TN / (TN + FP)로 계산해요 |
| 🚨 위음성(FN)이 치명적인 경우 | 암 검진, 바이러스 감지처럼 놓치면 안 되는 상황 → 민감도를 높여요 |
| 🔒 위양성(FP)이 치명적인 경우 | 잘못 분류하면 심각한 부작용이 있는 상황 → 특이도를 높여요 |
| 📊 3개 이상 카테고리 | 카테고리별로 민감도와 특이도를 각각 따로 계산해요 |
| ⚖️ 민감도-특이도 트레이드오프 | 민감도를 높이면 특이도가 낮아지는 경향이 있어요. 목적에 맞게 균형을 잡아요 |
| 📈 다음 단계 | 이 트레이드오프를 시각적으로 보여주는 ROC 곡선과 AUC를 나중에 배워요 |
