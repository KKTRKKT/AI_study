## 1. 엔트로피는 어디에 쓰이나요?

### 🤔 왜 배워야 하나요?

엔트로피라는 이름은 물리학에서도 나오고 정보이론에서도 나와서 왠지 어렵게 느껴지는 개념이에요. 하지만 데이터 과학에서 쓰이는 엔트로피는 아주 직관적인 아이디어에서 출발해요.

엔트로피는 데이터 과학의 여러 핵심 도구에서 사용돼요.

- **결정 트리(Classification Tree):** 데이터를 가장 잘 분류하는 방법을 찾을 때 엔트로피를 활용해요
- **상호 정보량(Mutual Information):** 두 변수가 얼마나 관련 있는지 측정할 때 엔트로피에서 파생돼요
- **교차 엔트로피(Cross Entropy):** 신경망 학습에서 예측과 정답의 차이를 계산하는 손실 함수예요
- **t-SNE, UMAP:** 고차원 데이터를 2D로 시각화할 때 사용하는 알고리즘들도 엔트로피 기반이에요

이 모든 도구들에는 공통점이 있어요. 엔트로피(또는 거기서 파생된 것)로 **"얼마나 비슷하고 얼마나 다른가"** 를 수치화해요. 그러니 엔트로피를 이해하면 이 도구들을 훨씬 깊이 이해할 수 있어요.

### 한 줄 정의

> **엔트로피(Entropy)란:** 어떤 집합에서 무작위로 하나를 뽑을 때, 평균적으로 기대되는 "놀람의 양"이에요.

---

## 2. 놀람(Surprise)이란?

### 🤔 왜 "놀람"이 필요한가요?

엔트로피를 이해하려면 먼저 "놀람"이라는 개념이 필요해요. 놀람은 어떤 사건이 일어났을 때 "얼마나 예상 밖인가"를 나타내요.

### 일상 비유

> 💡 **비유:** 로또 1등에 당첨되면 엄청나게 놀라요. 확률이 아주 낮기 때문이에요. 반면 매일 하는 출근길에 지하철이 왔을 때는 전혀 놀랍지 않아요. 이미 당연히 올 거라고 예상했으니까요.

**놀람과 확률의 관계:**
- 확률이 **높은** 사건 → 놀람이 **낮아요** (예상했던 일)
- 확률이 **낮은** 사건 → 놀람이 **높아요** (예상 밖의 일)

즉, 놀람은 확률과 반비례 관계예요.

### 닭 예시로 이해하기

주황 닭과 파란 닭이 세 구역(A, B, C)에 나뉘어 있다고 상상해요.

| 구역 | 주황 닭 | 파란 닭 | 파란 닭 뽑을 때 놀람 |
|---|---|---|---|
| A | 6마리 | 1마리 | 높음 (파란 닭이 드물어요) |
| B | 1마리 | 10마리 | 낮음 (파란 닭이 많아요) |
| C | 3마리 | 3마리 | 중간 (둘 다 같은 수예요) |

무작위로 닭을 집었을 때, 흔한 색깔의 닭이 잡혀도 별로 놀랍지 않아요. 드문 색깔의 닭이 잡혀야 놀라워요.

---

## 3. 놀람을 어떻게 계산하나요?

### 🤔 왜 단순히 "확률의 역수"를 쓰면 안 되나요?

놀람이 확률의 역수와 반비례한다면 그냥 1/확률을 쓰면 되지 않을까요? 문제가 있어요.

동전을 100% 확률로 앞면만 나오는 나쁜 동전으로 던진다고 해봐요. 앞면이 나와도 전혀 놀랍지 않아요. 놀람이 **0**이어야 해요. 그런데 1/확률 = 1/1 = **1**이 나와요. 원하는 값(0)이 아니에요.

### 해결책: 로그를 사용해요

놀람의 올바른 공식은 이렇게 생겼어요.

```
놀람 = log(1 / 확률) = -log(확률)
```

**이 수식이 말하는 것:**

확률이 1(100%)일 때: log(1/1) = log(1) = **0** → 전혀 놀랍지 않아요 ✅  
확률이 0.5(50%)일 때: log(1/0.5) = log(2) = **1** → 어느 정도 놀라워요  
확률이 0에 가까울 때: 값이 **아주 커져요** → 매우 놀라워요 ✅

> 💡 **보충 설명:** log(0)은 정의되지 않아요. 이건 수학적으로 맞아요. 확률이 0인 사건(절대 일어나지 않는 사건)의 놀람을 계산하는 건 의미가 없으니까요. "절대 일어나지 않는 일이 일어났을 때 얼마나 놀라운가"는 질문 자체가 이상한 거예요.

> 💡 **보충 설명:** 이진 분류(두 가지 결과만 있는 경우)에서는 **log₂(2를 밑으로 하는 로그)** 를 주로 써요. 그러면 놀람의 단위가 "비트(bit)"가 돼요. 머신러닝에서는 자연로그(loge = ln)를 많이 써요.

## 4. 여러 사건의 놀람을 합산하기

### 동전 던지기 예시

앞면이 90%, 뒷면이 10%인 동전이 있어요.

| 결과 | 확률 | 놀람 |
|---|---|---|
| 앞면 | 0.9 | log₂(1/0.9) ≈ 0.15 |
| 뒷면 | 0.1 | log₂(1/0.1) ≈ 3.32 |

뒷면이 훨씬 드물기 때문에 뒷면이 나왔을 때 놀람이 훨씬 커요.

이 동전을 3번 던져서 앞, 앞, 뒤가 나왔다고 해봐요.

```
총 놀람 = (앞면 놀람) + (앞면 놀람) + (뒷면 놀람)
        = 0.15 + 0.15 + 3.32
        = 3.62
```

**핵심 발견:** 여러 번의 사건에 대한 총 놀람은, 각 사건의 놀람을 **단순히 더하면** 돼요.

> 💡 **보충 설명:** 로그를 쓰는 또 다른 이유가 바로 이거예요. 여러 사건의 확률은 서로 **곱해야** 하지만, 놀람은 서로 **더하면** 돼요. 로그는 곱셈을 덧셈으로 바꿔주는 성질이 있거든요. 덕분에 계산이 훨씬 쉬워져요.

## 5. 엔트로피 = 평균 놀람

### 🤔 놀람에서 엔트로피로

동전을 100번 던지면 총 놀람이 얼마가 될까요?

- 앞면은 약 90번 나올 거예요 → 앞면 놀람 0.15 × 90번 = 13.5
- 뒷면은 약 10번 나올 거예요 → 뒷면 놀람 3.32 × 10번 = 33.2
- **총 놀람 = 13.5 + 33.2 = 46.7**

이걸 던진 횟수(100)로 나누면 "한 번 던질 때 평균적으로 기대되는 놀람"이 나와요.

```
평균 놀람 = 46.7 / 100 = 0.47
```

이게 바로 **엔트로피(0.47)** 예요! 이 동전을 한 번 던질 때마다 평균 0.47만큼의 놀람을 기대할 수 있어요.

### 엔트로피 공식 유도

100을 곱하고 다시 100으로 나누면 100이 상쇄돼요. 그러면 자연스럽게 이런 공식이 나와요.

```
엔트로피 = (앞면 확률 × 앞면 놀람) + (뒷면 확률 × 뒷면 놀람)
         = Σ [확률(x) × 놀람(x)]
         = Σ [확률(x) × log(1/확률(x))]
```

**이 수식이 말하는 것:**
각 사건이 "얼마나 자주 일어나는지(확률)"와 "얼마나 놀라운지(놀람)"를 곱해서 전부 더한 거예요. 즉, **놀람의 기댓값(Expected Value)** 이에요.

### 클로드 섀넌(Claude Shannon)의 공식

수식을 정리하면 1948년 클로드 섀넌이 발표한 엔트로피 공식이 나와요.

```
H = -Σ [p(x) × log(p(x))]
```

**이 수식이 말하는 것:**
각 결과의 확률에 그 확률의 로그를 곱한 뒤 모두 더하고, 맨 앞에 음수 부호를 붙여요. log(1/p) = -log(p)이기 때문에 수학적으로 위의 합산 공식과 완전히 같은 식이에요.

> ⚠️ **주의:** 이 공식이 표준 형태예요. 대부분의 교재와 코드에서 이 형태를 볼 수 있어요. 하지만 "놀람의 기댓값"이라는 직관을 기억하면 이 공식이 왜 이 모양인지 이해할 수 있어요.

## 6. 엔트로피 실제로 계산해보기

### 구역 A — 주황 6마리, 파란 1마리

```
주황 닭 확률 = 6/7
파란 닭 확률 = 1/7

엔트로피 = (6/7 × log₂(7/6)) + (1/7 × log₂(7/1))
          ≈ 0.59
```

### 구역 B — 주황 1마리, 파란 10마리

```
주황 닭 확률 = 1/11
파란 닭 확률 = 10/11

엔트로피 ≈ 0.44
```

### 구역 C — 주황 3마리, 파란 3마리

```
주황 닭 확률 = 3/6 = 0.5
파란 닭 확률 = 3/6 = 0.5

엔트로피 = (0.5 × log₂(2)) + (0.5 × log₂(2))
         = 0.5 × 1 + 0.5 × 1
         = 1.0
```

### 세 구역 비교

| 구역 | 구성 | 엔트로피 | 의미 |
|---|---|---|---|
| A | 주황 6, 파란 1 | 0.59 | 어느 정도 편향됐어요 |
| B | 주황 1, 파란 10 | 0.44 | 더 많이 편향됐어요 |
| C | 주황 3, 파란 3 | **1.0** | 완전히 균형 잡혔어요 |

**엔트로피가 말해주는 것:**
- 구역 C처럼 두 종류가 **정확히 반반**이면 엔트로피가 가장 높아요 → 어떤 닭이 잡힐지 가장 불확실해요
- 한 종류가 **압도적으로 많을수록** 엔트로피가 낮아요 → 어떤 닭이 잡힐지 어느 정도 예측 가능해요

> 💡 **비유:** 엔트로피는 "불순도(Impurity)" 또는 "불확실성(Uncertainty)"을 나타내기도 해요. 두 종류가 뒤섞여 있을수록 엔트로피가 높아요. 한 종류만 있을수록 엔트로피가 낮아요.

## 7. 엔트로피가 결정 트리와 연결되는 방법

### 🤔 결정 트리에서 엔트로피를 왜 쓰나요?

결정 트리는 데이터를 가장 "잘 분리하는" 질문을 찾아요. 가장 잘 분리한다는 건 각 그룹 내부가 가능한 한 **"같은 종류"** 로 구성된 상태예요. 이걸 수치로 표현하는 도구가 바로 엔트로피예요.

- 분리 후 각 그룹의 엔트로피가 낮을수록 → 잘 분리된 거예요
- 분리 전 엔트로피 - 분리 후 평균 엔트로피 = **정보 이득(Information Gain)**
- 정보 이득이 가장 큰 질문을 선택해요

예를 들어 "색깔이 파란가요?"라는 질문으로 닭을 나눴더니, 한 그룹은 파란 닭만, 다른 그룹은 주황 닭만 있다면 엔트로피가 0이에요. 완벽하게 분리된 거예요.

## ⚠️ 주의할 점 / 자주 하는 실수

- **엔트로피가 높다고 나쁜 게 아니에요.** 엔트로피가 높다는 건 "불확실성이 높다"는 뜻인데, 상황에 따라 다르게 해석해요. 결정 트리에서는 낮은 엔트로피가 좋지만, 다양성을 측정할 때는 높은 엔트로피가 좋을 수도 있어요.
- **엔트로피의 최댓값은 카테고리 수에 따라 달라요.** 카테고리가 2개면 최댓값이 1, 3개면 log₂(3) ≈ 1.58이에요. 서로 다른 카테고리 수의 엔트로피를 직접 비교하면 안 돼요.
- **확률이 0인 항목은 엔트로피 계산에서 제외해요.** 0 × log(0)은 수학적으로 정의되지 않지만, 극한값으로 0이에요. 실제 계산에서는 그냥 0으로 처리해요.
- **로그의 밑(base)을 통일해야 해요.** log₂를 쓰든 loge(자연로그)를 쓰든 일관되게 써야 해요. 섞어 쓰면 결과가 달라져요.

<summary>정답 보기</summary>

**정답:** 앞면 확률이 0.5일 때(반반) 엔트로피가 가장 높고, 0.7/0.3으로 치우칠수록 엔트로피는 낮아져요.

**설명:** 앞면 확률이 0.5일 때는 앞면이 나올지 뒷면이 나올지 가장 불확실해요. 어떤 결과도 예상할 수 없으니 평균 놀람(엔트로피)이 최대예요. 반면 0.7/0.3으로 치우치면 앞면이 더 자주 나올 거라고 예측이 가능해서 불확실성이 줄어들고 엔트로피도 낮아져요. 한쪽이 100%이면 엔트로피는 0이에요. (결과가 뻔하니까요.)

</details>

**Q3.** 결정 트리에서 "분리가 잘 됐다"를 엔트로피로 어떻게 표현하나요?

<details>
<summary>정답 보기</summary>

**정답:** 분리 후 각 그룹의 엔트로피가 낮으면 잘 분리된 거예요.

**설명:** 어떤 질문으로 데이터를 나눴을 때, 각 그룹 안에 같은 종류만 모여 있으면 엔트로피가 0이에요. 뒤섞여 있으면 엔트로피가 높아요. "분리 전 엔트로피 - 분리 후 평균 엔트로피"를 정보 이득(Information Gain)이라고 하고, 이 값이 가장 큰 질문을 결정 트리가 선택해요.

</details>

**Q4.** 카테고리가 5개이고 각각의 확률이 모두 동일할 때(각 0.2), 이 상황의 엔트로피는 최대인가요 최소인가요? 이유를 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 최대 엔트로피예요.

**설명:** 5개 카테고리의 확률이 모두 동일(0.2)하면 어떤 결과가 나올지 전혀 예측할 수 없어요. 불확실성이 가장 높은 상태예요. 엔트로피는 불확실성의 측정값이기 때문에 이 상태가 엔트로피 최대예요. 반대로 한 카테고리 확률이 1이고 나머지가 0이면 엔트로피는 0(최소)이에요.

</details>

**Q5.** 엔트로피가 "놀람의 기댓값"이라는 설명이 맞는 이유를 자신의 말로 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 기댓값은 "각 결과의 값 × 그 결과가 일어날 확률"의 합이에요. 엔트로피도 "각 사건의 놀람 × 그 사건이 일어날 확률"의 합이에요. 그래서 엔트로피는 놀람의 기댓값이에요.

**설명:** 동전 던지기 예시로 보면, 앞면 놀람(0.15)이 90%의 확률로 발생하고 뒷면 놀람(3.32)이 10%의 확률로 발생해요. 가중 평균을 내면 0.15×0.9 + 3.32×0.1 = 0.47이에요. 이게 "한 번 던질 때 평균적으로 기대할 수 있는 놀람"이고, 그게 엔트로피예요.

</details>

### 실전 적용 문제

**문제.** 어떤 과일 바구니에 사과 8개, 바나나 2개가 있어요. 또 다른 바구니에는 사과 5개, 바나나 5개가 있어요. 어느 바구니의 엔트로피가 더 높은가요? 직접 계산하지 않아도 좋으니 직관적인 이유와 함께 설명해보세요. 그리고 이것이 결정 트리에서 어떤 의미를 갖는지도 이야기해보세요.

**힌트:**
- 엔트로피는 "어떤 과일이 나올지 얼마나 불확실한가"를 나타내요
- 결정 트리는 엔트로피가 낮은(순수한) 그룹을 만들려고 해요
- 어느 바구니가 "더 순수"한가요?

<details>
<summary>예시 답안 보기</summary>

**사과 5개, 바나나 5개인 바구니의 엔트로피가 더 높아요.**

직관적 이유: 사과 8개, 바나나 2개인 바구니에서는 꺼낼 과일이 사과일 가능성이 훨씬 높아요. 어느 정도 예측 가능하기 때문에 불확실성이 낮고 엔트로피도 낮아요.

반면 5:5 바구니에서는 사과와 바나나가 반반이라 꺼낼 과일을 전혀 예측할 수 없어요. 불확실성이 최대이고 엔트로피도 최대예요.

**결정 트리에서의 의미:** 결정 트리가 이 두 바구니 중 하나를 선택해서 분류를 수행한다면, 8:2 바구니처럼 엔트로피가 낮은 쪽이 "더 순수한 분류"가 이루어진 거예요. 결정 트리는 분리 후 각 그룹의 엔트로피가 최대한 낮아지도록 질문을 선택해요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>
