## 1. 머신러닝이 뭔가요?

### 🤔 왜 필요한가요?

세상에는 눈으로 보면 대충 패턴이 보이는 것들이 있어요. 예를 들어, "얌을 많이 먹을수록 더 빨리 달린다"는 경향이 데이터에 보인다고 해봐요. 그런데 새로운 사람이 나타났을 때 얼마나 빨리 달릴지 **예측**하려면 어떻게 해야 할까요? 눈으로 보는 것만으로는 부족해요. 데이터에서 패턴을 뽑아내서 예측을 자동으로 해주는 도구가 필요한 거예요. 그게 바로 머신러닝이에요.

### 한 줄 정의

> **머신러닝이란:** 데이터에서 패턴을 찾아 **예측(Prediction)** 과 **분류(Classification)** 를 자동으로 해내는 방법들의 모음이에요.

### 어떻게 동작하나요?

머신러닝이 하는 일은 크게 두 가지예요.

**① 분류(Classification)** — "이 사람이 A 그룹인가, B 그룹인가?"를 판단해요.

> 💡 **강의 예시:** "당신은 silly song을 좋아하나요? 머신러닝에 관심 있나요?" 같은 질문을 이어가면서 마지막에 "이 사람은 StatQuest를 좋아할 것이다 / 아닐 것이다"로 분류하는 결정 트리(Decision Tree)가 분류의 예시예요.

결정 트리는 질문을 나무처럼 가지 치면서 최종 판단을 내리는 방법이에요. 답이 "좋아한다 / 싫어한다"처럼 딱 나뉘는 형태가 분류예요.

**② 회귀(Regression)** — "이 사람의 속도가 얼마일까?"처럼 **숫자**를 예측해요.

> 💡 **강의 예시:** 얌(yam)을 얼마나 먹었는지에 따라 100m 달리기 속도를 예측하는 그래프를 그리면, 데이터 위에 추세선 하나를 그을 수 있어요. 이 선을 이용해서 "얌을 이만큼 먹은 사람은 이 정도 속도로 달릴 것"을 예측할 수 있어요. 이게 회귀예요.

### 실제 예시

분류와 회귀를 나란히 비교하면 이렇게 볼 수 있어요.

| 구분 | 질문 형태 | 결과 |
|---|---|---|
| 분류(Classification) | "이 사람이 심장 질환이 있나요?" | 있다 / 없다 |
| 회귀(Regression) | "이 사람의 달리기 속도는 얼마일까요?" | 9.8초, 11.2초 같은 숫자 |

## 2. 훈련 데이터 vs 테스트 데이터

### 🤔 왜 나눠야 하나요?

머신러닝은 데이터를 보고 패턴을 배워요. 그런데 배운 데이터로 다시 시험을 보면 어떻게 될까요? 마치 **족보를 달달 외우고 똑같은 족보로 시험을 보는 것**과 같아요. 점수는 잘 나오겠지만, 실제로 실력이 있는지는 알 수 없어요.

그래서 데이터를 두 덩어리로 나눠요.

- **훈련 데이터(Training Data):** 모델이 패턴을 배우는 데 쓰는 데이터
- **테스트 데이터(Testing Data):** 모델이 잘 배웠는지 확인하는 데 쓰는 데이터

### 한 줄 정의

> **훈련/테스트 분리란:** 모델이 "처음 보는 데이터"에서도 잘 작동하는지 확인하기 위해, 데이터를 배우는 용도와 평가하는 용도로 나누는 것이에요.

### 어떻게 동작하나요?

추세선(검은 직선)과 구불구불한 선(초록 곡선) 두 가지를 생각해볼게요.

- 훈련 데이터에 더 딱 붙는 건 **초록 곡선**이에요. 훈련 데이터의 모든 점을 거의 완벽하게 통과해요.
- 그런데 테스트 데이터(처음 보는 새 데이터)를 주면 어떻게 될까요?

> 💡 **강의 예시:** 훈련 데이터에 딱 맞게 구불구불하게 생긴 선은, 새로운 사람의 속도를 예측할 때 오히려 더 크게 틀려요. 반면 단순한 직선은 훈련 데이터엔 덜 맞았지만, 새 데이터에서는 더 정확한 예측을 해요.

이게 바로 핵심이에요. 훈련 데이터를 얼마나 잘 맞추느냐가 아니라, **처음 보는 데이터를 얼마나 잘 예측하느냐**가 머신러닝의 진짜 목표예요.

### 실제 예시 — 거리 합산으로 성능 비교하기

두 선 중 어느 게 더 나은지 비교하는 방법은 간단해요.

테스트 데이터의 각 점마다 "실제 값"과 "예측 값"의 차이(거리)를 구해서 모두 더해요. 합이 작을수록 더 정확한 예측을 한 거예요.

```
성능 비교 방법:
1. 테스트 데이터의 각 점에서 → (실제 값) - (예측 값) 차이를 구해요
2. 차이를 모두 더해요
3. 합이 작은 쪽이 더 좋은 모델이에요
```

> ⚠️ **주의:** 직선이 훈련 데이터에 덜 맞는다고 해서 나쁜 모델이 아니에요. 테스트 데이터에서의 성능이 핵심이에요.

## 3. 과적합과 편향-분산 트레이드오프

### 🤔 왜 알아야 하나요?

"훈련 데이터에서 완벽한 점수"를 목표로 하면 반드시 실패해요. 왜냐하면 너무 잘 맞추려다 보면 훈련 데이터의 "노이즈(잡음)"까지 외워버리기 때문이에요. 새로운 데이터에는 그 노이즈가 없으니 예측이 엉망이 되는 거예요.

### 한 줄 정의

> **과적합(Overfitting)이란:** 훈련 데이터에는 아주 잘 맞지만, 새로운 데이터에는 형편없는 성능을 보이는 상태예요.

### 일상 비유

- ❌ **과적합** = 족보 달달 외워서 시험 보기. 족보에 있는 문제는 다 맞지만 응용 문제는 다 틀려요.
- ✅ **적절한 모델** = 개념을 이해하고 시험 보기. 족보 문제도 맞고 응용 문제도 어느 정도 맞아요.

### 어떻게 동작하나요?

이 현상을 **편향-분산 트레이드오프(Bias-Variance Tradeoff)** 라고 불러요. 이건 이후 강의에서 훨씬 자세히 다루니까, 지금은 딱 이것만 기억해요.

> 🎯 **핵심:** "훈련 데이터를 잘 맞추는 것"과 "새 데이터를 잘 예측하는 것"은 서로 다른 목표예요. 머신러닝의 진짜 목표는 후자예요.

## 4. "가장 좋은" 머신러닝 방법은?

### 🤔 왜 중요한가요?

딥러닝, 신경망, 랜덤 포레스트, XGBoost... 이름만 들어도 화려한 방법들이 넘쳐나요. 그런데 이름이 화려하다고 해서 무조건 좋은 건 아니에요. 어떤 기준으로 방법을 고를지를 알아야 해요.

### 한 줄 정의

> **모델 선택 기준:** 어떤 방법이 더 좋은지는 **테스트 데이터에서의 성능**으로만 판단해요. 얼마나 화려한 방법인지는 상관없어요.

### 결정 트리 예시로 보는 모델 평가

결정 트리(Decision Tree)를 훈련 데이터로 만들었어요. 이제 테스트 데이터로 얼마나 잘 작동하는지 확인해볼게요.

1. 테스트 데이터의 첫 번째 사람이 결정 트리를 따라 내려가요
2. 결정 트리가 "이 사람은 StatQuest를 좋아하지 않을 것"이라고 예측해요
3. 그런데 실제로는 그 사람이 StatQuest를 좋아해요 → **틀렸어요** ❌
4. 두 번째 사람도 내려가요 → 예측이 맞았어요 ✅
5. 이렇게 모든 테스트 데이터를 돌려보고, 전체 맞춤 비율을 비교해요

이 과정을 결정 트리뿐만 아니라 다른 머신러닝 방법으로도 해보고, 테스트 데이터에서 가장 잘 맞추는 방법을 선택해요.

```
모델 선택 흐름:
1. 여러 머신러닝 방법을 동일한 훈련 데이터로 각각 학습시켜요
2. 동일한 테스트 데이터로 각 방법의 성능을 비교해요
3. 테스트 데이터에서 가장 좋은 성능을 보인 방법을 선택해요
```

> ⚠️ **주의:** "가장 화려하고 복잡한" 방법이 아니라 "이 데이터에서 가장 잘 작동하는" 방법을 골라요. 문제마다 다른 방법이 최선일 수 있어요.

## 5. 훈련/테스트 데이터를 어떻게 나눌까요?

### 🤔 왜 고민해야 하나요?

데이터를 훈련용과 테스트용으로 나눌 때, "어떤 데이터를 훈련에 쓰고 어떤 걸 테스트에 쓸지"가 결과에 영향을 미쳐요. 그냥 앞의 75%를 훈련, 뒤의 25%를 테스트로 쓰면 될까요? 만약 뒤의 25%가 특이한 데이터였다면 평가가 왜곡될 수 있어요.

### 한 줄 정의

> **데이터 분할 고민:** 어떤 데이터를 훈련/테스트에 쓸지를 잘 결정해야 공정한 평가가 가능해요.

이 문제를 해결하는 방법이 바로 **교차 검증(Cross Validation)** 이에요. 이건 다음 문서에서 자세히 다룰게요!

---

## ⚠️ 주의할 점 / 자주 하는 실수

- **훈련 데이터 성능만 보고 "이 모델은 완벽해!"라고 하면 안 돼요.** 반드시 테스트 데이터 성능을 봐야 해요.
- **복잡한 모델이 무조건 좋다는 생각은 버려요.** 때로는 단순한 직선이 복잡한 곡선보다 더 좋은 예측을 해요.
- **훈련 데이터와 테스트 데이터를 절대 섞어서 쓰면 안 돼요.** 테스트 데이터는 모델이 한 번도 보지 못한 "진짜 새 데이터"여야 해요.

**Q2.** 구불구불한 초록 곡선이 훈련 데이터를 훨씬 완벽하게 맞췄는데, 왜 단순한 검은 직선보다 나쁜 모델이 될 수 있나요?

<details>
<summary>정답 보기</summary>

**정답:** 초록 곡선이 훈련 데이터의 패턴뿐만 아니라 잡음(Noise)까지 외워버렸기 때문이에요.

**설명:** 데이터에는 항상 약간의 무작위적인 잡음이 섞여 있어요. 너무 복잡한 모델은 그 잡음까지 "패턴"으로 인식해서 거기에 맞추려 해요. 그러면 새로운 데이터에서 그 잡음이 없으니 예측이 크게 빗나가요. 이것을 과적합(Overfitting)이라고 해요. 반면 단순한 직선은 훈련 데이터에 완전히 맞추진 못해도, 전체적인 패턴(진짜 경향)을 더 잘 반영하기 때문에 새 데이터에서 더 안정적인 예측을 해요.

</details>

**Q3.** 분류(Classification)와 회귀(Regression)의 차이를 자신만의 예시를 들어 설명해보세요.

<details>
<summary>정답 보기</summary>

**설명:**

- **회귀 예시:** 집 크기와 위치를 보고 "집값이 얼마일까?"를 예측하는 것. 결과가 구체적인 숫자로 나와요.

핵심 차이: 분류는 "어느 상자에 속하는가"를, 회귀는 "얼마짜리인가"를 맞추는 거예요.

**Q4.** 두 머신러닝 모델 중 어느 쪽이 더 좋은지 판단할 때, 무엇을 기준으로 해야 하나요?

<details>
<summary>정답 보기</summary>

**정답:** 테스트 데이터에서의 성능을 기준으로 해야 해요.

**설명:** 훈련 데이터에서 높은 성능을 보인다고 좋은 모델이 아니에요. 훈련 데이터는 모델이 이미 본 데이터라서 당연히 잘 맞춰요. 중요한 건 한 번도 보지 않은 테스트 데이터에서 얼마나 정확한 예측을 하느냐예요. 또한 더 화려하거나 복잡한 방법이 무조건 좋은 것도 아니에요. 데이터마다 가장 잘 맞는 방법이 달라요.

</details>

**Q5.** 과적합(Overfitting)이 발생했다는 걸 어떻게 알 수 있나요?

<details>
<summary>정답 보기</summary>

**정답:** 훈련 데이터에서는 성능이 아주 높은데, 테스트 데이터에서는 성능이 뚝 떨어지는 경우에요.

**설명:** 훈련 성능과 테스트 성능의 차이가 크다면 과적합이 일어난 거예요. 예를 들어 훈련 데이터에서 오차가 거의 0인데 테스트 데이터에서 오차가 크다면, 모델이 훈련 데이터를 "암기"해버린 거예요. 이 간극이 작을수록 더 일반화(Generalization)가 잘 된 좋은 모델이에요.

</details>

### 실전 적용 문제

**문제.** 아래 상황에서 머신러닝 접근법을 설계해보세요.

> 온라인 쇼핑몰에서 고객이 특정 상품을 구매할지 안 할지를 예측하고 싶어요. 고객의 나이, 이전 구매 횟수, 최근 방문 횟수, 평균 체류 시간 데이터가 있어요.

1. 이 문제는 분류인가요, 회귀인가요?
2. 데이터를 어떻게 나눠야 할까요?
3. 어떤 기준으로 모델을 선택해야 할까요?

**힌트:**
- 예측하려는 결과가 숫자인지 카테고리인지 생각해봐요
- 훈련/테스트 분리의 목적을 떠올려봐요
- "최신 방법"이 "최선의 방법"은 아니에요

<details>
<summary>예시 답안 보기</summary>

1. **분류 문제예요.** "구매한다(1) / 구매하지 않는다(0)"처럼 결과가 카테고리로 나오기 때문이에요.

2. **데이터 나누기:** 전체 데이터 중 일부(예: 80%)를 훈련 데이터로 쓰고, 나머지(20%)를 테스트 데이터로 남겨둬요. 훈련에 쓴 데이터는 절대 테스트에 다시 쓰지 않아요.

3. **모델 선택 기준:** 로지스틱 회귀, KNN, 랜덤 포레스트 등 여러 방법을 훈련 데이터로 학습시키고, 테스트 데이터에서 가장 정확하게 구매 여부를 맞추는 방법을 선택해요. 이름이 화려한 방법이 아니라 이 데이터에서 실제로 잘 되는 방법을 골라요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>

## 🎯 핵심 요약

| 개념 | 한 줄 설명 |
|---|---|
| 🤖 머신러닝(Machine Learning) | 데이터에서 패턴을 찾아 예측과 분류를 자동으로 해내는 방법들의 모음이에요 |
| 🗂️ 분류(Classification) | 결과가 카테고리로 나오는 예측이에요 (예: 심장 질환 있다/없다) |
| 📈 회귀(Regression) | 결과가 숫자로 나오는 예측이에요 (예: 달리기 속도가 몇 초인지) |
| 📚 훈련 데이터(Training Data) | 모델이 패턴을 배우는 데 쓰는 데이터예요 |
| 🧪 테스트 데이터(Testing Data) | 모델이 얼마나 잘 배웠는지 확인하는 데이터예요 (모델이 처음 보는 데이터예요) |
| 🌀 과적합(Overfitting) | 훈련 데이터엔 완벽하지만 새 데이터에서 형편없는 상태예요 (족보 암기와 같아요) |
| ⚖️ 편향-분산 트레이드오프(Bias-Variance Tradeoff) | 단순한 모델과 복잡한 모델 사이의 균형을 찾아야 한다는 개념이에요 |
| 🌲 결정 트리(Decision Tree) | 질문을 나무처럼 이어가며 분류를 수행하는 대표적인 머신러닝 방법이에요 |
