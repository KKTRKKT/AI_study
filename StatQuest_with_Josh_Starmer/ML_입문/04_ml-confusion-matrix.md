## 📌 이 문서에서 배우는 것

- 혼동 행렬(Confusion Matrix)이 무엇인지 이해해요
- 진양성, 진음성, 위양성, 위음성이 각각 무엇인지 정확히 구별할 수 있어요
- 혼동 행렬로 두 모델의 성능을 비교하는 방법을 알아요
- 2×2를 넘어서는 더 큰 혼동 행렬도 이해할 수 있어요
- 단순한 정확도만으로는 부족한 이유를 알아요

## 용어 정리

| 용어 | 원문 | 한 줄 설명 |
|---|---|---|
| 혼동 행렬 | Confusion Matrix | 모델이 맞춘 것과 틀린 것을 표 형태로 정리한 것이에요 |
| 진양성 | True Positive (TP) | 양성이라고 예측했는데 실제로 양성인 경우예요 |
| 진음성 | True Negative (TN) | 음성이라고 예측했는데 실제로 음성인 경우예요 |
| 위양성 | False Positive (FP) | 양성이라고 예측했는데 실제로 음성인 경우예요 |
| 위음성 | False Negative (FN) | 음성이라고 예측했는데 실제로 양성인 경우예요 |

## 1. 왜 단순 정확도로는 부족한가요?

### 🤔 왜 필요한가요?

심장 질환 예측 모델을 여러 개 만들었어요. 어떤 모델이 가장 좋은지 어떻게 비교할까요?

가장 쉬운 방법은 **정확도(Accuracy)** 를 보는 거예요.

```
정확도 = 맞게 예측한 수 / 전체 예측 수
```

그런데 이것만으로는 충분하지 않아요. 예를 들어 이런 경우를 생각해봐요.

> 100명의 환자 중 5명만 심장 질환이 있어요. 모델이 "모두 심장 질환 없음"이라고 예측하면? **정확도 95%!** 하지만 정작 심장 질환이 있는 5명을 모두 놓쳤어요.

이 모델은 쓸모없는 모델이에요. 그런데 정확도만 보면 "훌륭한 모델"처럼 보여요. 정확도 하나만으로는 모델의 진짜 실력을 볼 수 없어요.

### 한 줄 정의

> **혼동 행렬(Confusion Matrix)이란:** 모델이 무엇을 맞게 예측했고 무엇을 틀리게 예측했는지를, 카테고리별로 세밀하게 보여주는 표예요.

---

## 2. 혼동 행렬의 구조

### 2×2 혼동 행렬 읽는 법

심장 질환(있다/없다)을 예측하는 경우, 혼동 행렬은 2행 2열로 이루어져요.

```
                  [실제: 심장 질환 있음]  [실제: 심장 질환 없음]
[예측: 있음]      진양성 (TP)             위양성 (FP)
[예측: 없음]      위음성 (FN)             진음성 (TN)
```

- **행(Row):** 모델이 예측한 것
- **열(Column):** 실제 정답

**대각선(초록 박스)** = 맞게 예측한 것들 (TP + TN)
**대각선 외(빨간 박스)** = 틀리게 예측한 것들 (FP + FN)

### 네 칸 자세히 알아보기

#### ✅ 진양성 (True Positive, TP) — 왼쪽 위

> "심장 질환이 있다고 예측했는데, 실제로 있어요."

좋은 결과예요. 병이 있는 환자를 정확히 잡아냈어요.

#### ✅ 진음성 (True Negative, TN) — 오른쪽 아래

> "심장 질환이 없다고 예측했는데, 실제로 없어요."

좋은 결과예요. 건강한 사람을 정확히 건강하다고 했어요.

#### ❌ 위음성 (False Negative, FN) — 왼쪽 아래

> "심장 질환이 없다고 예측했는데, 실제로 있어요."

위험한 실수예요. 병이 있는 환자를 건강하다고 했어요. 치료가 필요한 사람을 놓친 거예요.

#### ❌ 위양성 (False Positive, FP) — 오른쪽 위

> "심장 질환이 있다고 예측했는데, 실제로 없어요."

불필요한 불안과 추가 검사를 유발하는 실수예요. 건강한 사람을 환자로 오해한 거예요.

### 일상 비유로 기억하기

> 💡 **비유:** 화재 경보기를 생각해봐요.
> - **진양성 (TP):** 불이 났는데 경보가 울렸어요 → **정상, 잘 작동했어요** ✅
> - **진음성 (TN):** 불이 없는데 경보가 안 울렸어요 → **정상, 잘 작동했어요** ✅
> - **위음성 (FN):** 불이 났는데 경보가 안 울렸어요 → **위험한 실수! 큰일 나요** ❌
> - **위양성 (FP):** 불이 없는데 경보가 울렸어요 → **귀찮은 실수, 하지만 치명적이진 않아요** ❌

## 3. 실제 예시 — 랜덤 포레스트 vs KNN

### 랜덤 포레스트 결과

| | 실제: 심장 질환 있음 | 실제: 심장 질환 없음 |
|---|---|---|
| **예측: 있음** | 142 (TP) | 22 (FP) |
| **예측: 없음** | 29 (FN) | 110 (TN) |

- ✅ 맞게 예측: 142 + 110 = 252명
- ❌ 틀리게 예측: 29 + 22 = 51명

**읽는 법:**
- 심장 질환이 있는 환자 171명 중 → **142명을 정확히 잡아냈어요** (29명 놓쳤어요)
- 심장 질환이 없는 사람 132명 중 → **110명을 정확히 건강하다고 했어요** (22명은 잘못 환자로 봤어요)

### K-최근접 이웃(KNN) 결과

| | 실제: 심장 질환 있음 | 실제: 심장 질환 없음 |
|---|---|---|
| **예측: 있음** | 107 (TP) | 53 (FP) |
| **예측: 없음** | 64 (FN) | 79 (TN) |

### 두 모델 비교

| 지표 | 랜덤 포레스트 | KNN |
|---|---|---|
| 진양성 (TP) | **142** | 107 |
| 진음성 (TN) | **110** | 79 |
| 위음성 (FN) | 29 | **64 (더 많이 놓쳤어요)** |
| 위양성 (FP) | **22** | 53 |

두 지표 모두에서 랜덤 포레스트가 더 좋아요. 이런 경우엔 명확하게 랜덤 포레스트를 선택해요.

> ⚠️ **주의:** 두 모델 사이에서 어떤 건 좋고 어떤 건 나쁘면 선택이 어려울 수 있어요. 그럴 때는 민감도(Sensitivity)와 특이도(Specificity) 같은 더 구체적인 지표를 사용해요. 이건 다음 문서에서 다뤄요!

## 4. 3×3 이상의 혼동 행렬

### 🤔 왜 필요한가요?

분류할 카테고리가 3개 이상이면 혼동 행렬도 그에 맞게 커져요. 심장 질환 예측은 "있다/없다" 두 가지지만, 좋아하는 영화를 예측하는 건 여러 영화 중 하나를 고르는 거예요.

### 3×3 혼동 행렬 예시

세 영화 중 가장 좋아하는 것을 예측하는 경우예요. (트롤 2, 고어 폴리스, 쿨 애즈 아이스)

```
                [실제: 트롤2]  [실제: 고어폴리스]  [실제: 쿨애즈아이스]
[예측: 트롤2]       12              112                 83
[예측: 고어폴리스]   102              23                 92
[예측: 쿨애즈아이스]  77               93                 17
```

- **대각선 (초록):** 12 + 23 + 17 = 52명 → 맞게 예측했어요
- **나머지 (빨간):** 모두 틀리게 예측했어요

이 모델은 성능이 많이 낮네요. 사실 주어진 영화들이 워낙 비슷하게 나쁜 영화들이라 구별이 어려웠을 거예요.

### 혼동 행렬 크기의 원칙

| 카테고리 수 | 혼동 행렬 크기 |
|---|---|
| 2개 (이진 분류) | 2행 × 2열 |
| 3개 | 3행 × 3열 |
| 4개 | 4행 × 4열 |
| N개 | N행 × N열 |

언제나 **행 수 = 열 수 = 카테고리 수** 예요.

> 💡 **팁:** 카테고리가 많아질수록 혼동 행렬이 복잡해져요. 그래서 대형 분류 문제에서는 혼동 행렬 전체를 보기보다 카테고리별 요약 지표를 많이 활용해요.

---

## ⚠️ 주의할 점 / 자주 하는 실수

- **행과 열의 의미를 헷갈리지 마세요.** 행은 "예측"이고 열은 "실제(정답)"예요. 반대로 기억하는 경우가 많아요.
- **위음성(FN)과 위양성(FP)은 상황에 따라 중요도가 달라요.** 암 진단에서는 위음성(있는데 없다고 함)이 훨씬 위험해요. 스팸 필터에서는 위양성(정상 메일을 스팸으로 분류)이 더 불편할 수 있어요.
- **"대각선만 보면 된다"고 착각하지 마세요.** 어떤 종류의 실수를 더 많이 하는지도 중요해요. 의료 분야에선 특히요.
- **혼동 행렬 하나만으로 최선의 모델을 고를 수 없는 경우도 있어요.** 두 모델의 혼동 행렬이 비슷할 때는 민감도, 특이도, ROC 등 추가 지표가 필요해요.

**Q1.** 위음성(False Negative)이 위양성(False Positive)보다 더 심각한 결과를 낳는 실제 사례를 생각해보고 이유를 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 암 검진, 코로나 검사, 화재 감지기 등이 대표적인 예시예요.

**설명:** 암 검진에서 위음성은 "암이 있는데 없다고 판정"하는 거예요. 환자는 치료를 받지 않고 집에 돌아가고, 암이 악화돼요. 이건 생명과 직결되는 실수예요. 반면 위양성은 "암이 없는데 있다고 판정"하는 거예요. 불필요한 추가 검사와 불안을 주지만, 결국 "건강합니다"라는 결과로 끝날 수 있어요. 위음성이 훨씬 더 심각한 결과를 낳아요.

</details>

**Q2.** 아래 혼동 행렬에서 이 모델의 정확도는 얼마인가요?

| | 실제: 양성 | 실제: 음성 |
|---|---|---|
| **예측: 양성** | 80 | 20 |
| **예측: 음성** | 10 | 90 |

<details>
<summary>정답 보기</summary>

**정답:** 85%예요.

**설명:**
- 전체 샘플 수 = 80 + 20 + 10 + 90 = 200명
- 정확히 예측한 수 = 80(TP) + 90(TN) = 170명
- 정확도 = 170 / 200 = 0.85 = **85%**

</details>

**Q3.** 혼동 행렬에서 "대각선의 합"이 의미하는 것은 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** 모델이 올바르게 예측한 샘플의 총 수예요.

**설명:** 혼동 행렬에서 대각선(왼쪽 위 → 오른쪽 아래)의 값들은 "예측과 실제가 일치하는 경우"예요. 2×2 행렬이면 TP와 TN이 대각선에 위치해요. 3×3 이상이면 세 개 이상의 값이 대각선에 있어요. 이 값들을 모두 더하면 정확히 예측한 총 수가 나오고, 전체로 나누면 정확도가 나와요.

</details>

**Q4.** 스팸 메일 분류기를 만들 때, 위양성(FP)이 많은 모델과 위음성(FN)이 많은 모델 중 어느 쪽이 더 나쁜가요? 이유도 함께 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 일반적으로 **위양성(FP)이 많은 모델**이 더 나쁜 경우가 많아요.

**설명:** 스팸 분류기에서 위음성(FN)은 "스팸인데 정상으로 분류"예요. 스팸 메일이 받은 편지함에 들어오는 거예요. 불편하지만 큰 피해는 없어요.

반면 위양성(FP)은 "정상인데 스팸으로 분류"예요. 중요한 업무 메일, 청구서, 지인의 메일이 스팸함으로 가버리는 거예요. 사용자가 중요한 정보를 놓칠 수 있어요. 일반적으로 이게 더 심각하게 여겨져요.

단, 맥락에 따라 달라질 수 있어요. 스팸이 아주 위험한 피싱 메일이라면 위음성도 심각할 수 있어요.

</details>

**Q5.** 분류 카테고리가 5개인 문제에서 혼동 행렬은 몇 행 몇 열로 이루어지나요? 그리고 "맞게 예측한 경우"는 어디에 표시되나요?

<details>
<summary>정답 보기</summary>

**정답:** 5행 5열이고, 맞게 예측한 경우는 대각선(왼쪽 위에서 오른쪽 아래)에 표시돼요.

**설명:** 혼동 행렬의 크기는 항상 카테고리 수 × 카테고리 수예요. 5개 카테고리라면 5×5 = 25칸이에요. 이 중 대각선의 5칸이 각 카테고리를 맞게 예측한 수예요. 나머지 20칸은 모두 잘못 예측한 경우예요.

</details>

### 실전 적용 문제

**문제.** 코로나 바이러스 검사 키트의 성능을 평가하는 혼동 행렬이 아래와 같아요.

| | 실제: 코로나 양성 | 실제: 코로나 음성 |
|---|---|---|
| **검사 결과: 양성** | 950 | 50 |
| **검사 결과: 음성** | 30 | 970 |

1. 각 칸의 의미를 설명해보세요 (TP, FP, FN, TN)
2. 이 키트의 정확도를 계산해보세요
3. 위음성(FN)과 위양성(FP) 중 어느 쪽이 더 심각한 문제인가요? 이유를 설명해보세요

**힌트:**
- 각 칸이 어떤 상황을 나타내는지 생각해봐요
- 정확도 = 맞게 예측 수 / 전체 수
- 코로나 확산 맥락에서 각 실수가 어떤 결과를 낳는지 생각해봐요

<details>
<summary>예시 답안 보기</summary>

**1. 각 칸의 의미:**
- **950 (TP):** 코로나 양성인 사람을 양성으로 올바르게 판정했어요 ✅
- **50 (FP):** 코로나 음성인 사람을 양성으로 잘못 판정했어요 ❌
- **30 (FN):** 코로나 양성인 사람을 음성으로 잘못 판정했어요 ❌ (위험!)
- **970 (TN):** 코로나 음성인 사람을 음성으로 올바르게 판정했어요 ✅

**2. 정확도 계산:**
- 전체 = 950 + 50 + 30 + 970 = 2,000명
- 맞게 예측 = 950 + 970 = 1,920명
- 정확도 = 1,920 / 2,000 = **96%**

**3. 위음성이 더 심각해요.**

위음성(FN = 30명)은 실제로 코로나에 걸렸는데 "음성"이라고 나온 경우예요. 이 사람들은 격리를 하지 않고 돌아다니면서 다른 사람들에게 바이러스를 전파할 수 있어요. 공중 보건에 심각한 영향을 줘요.

위양성(FP = 50명)은 코로나가 없는데 "양성"이라고 나온 경우예요. 불필요한 격리를 해야 하고 걱정이 되지만, 바이러스를 퍼뜨리는 일은 없어요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>

## 🎯 핵심 요약

| 개념 | 한 줄 설명 |
|---|---|
| 🗺️ 혼동 행렬(Confusion Matrix) | 모델의 예측 결과를 카테고리별로 맞음/틀림으로 나눠 정리한 표예요 |
| ✅ 진양성 (True Positive, TP) | 양성이라고 예측했고 실제로도 양성이에요. 올바른 예측이에요 |
| ✅ 진음성 (True Negative, TN) | 음성이라고 예측했고 실제로도 음성이에요. 올바른 예측이에요 |
| ❌ 위양성 (False Positive, FP) | 양성이라고 예측했지만 실제론 음성이에요. 잘못된 예측이에요 |
| ❌ 위음성 (False Negative, FN) | 음성이라고 예측했지만 실제론 양성이에요. 잘못된 예측이에요 |
| 📏 정확도(Accuracy) | 맞게 예측한 수 / 전체 수. 혼동 행렬의 대각선 합 / 전체 합이에요 |
| 📐 행렬 크기 | 카테고리 수 × 카테고리 수. 2분류면 2×2, 5분류면 5×5예요 |
| 🎯 대각선의 의미 | 대각선(왼위→오아래)이 모두 맞게 예측한 칸들이에요 |
