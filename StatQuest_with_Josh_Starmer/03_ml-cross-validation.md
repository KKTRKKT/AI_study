## 1. 문제 상황 — 어떤 방법이 제일 좋을까요?

### 🤔 왜 필요한가요?

심장 질환을 예측하는 모델을 만들어야 해요. 흉통, 혈액 순환 정도, 막힌 동맥 여부 같은 데이터로 환자가 심장 질환이 있는지를 예측하고 싶어요. 그런데 어떤 머신러닝 방법을 쓸지 결정해야 해요.

- 로지스틱 회귀(Logistic Regression)
- K-최근접 이웃(K-Nearest Neighbors, KNN)
- 서포트 벡터 머신(Support Vector Machine, SVM)
- 그 외 수많은 방법들...

이 중 어느 게 이 데이터에 가장 잘 맞을까요? 그냥 느낌으로 고를 수는 없어요. **데이터로 확인해야 해요.**

### 한 줄 정의

> **교차 검증(Cross Validation)이란:** 여러 머신러닝 방법을 공정하게 비교해서, 실제로 새 데이터에서도 잘 작동할 방법을 고르기 위한 평가 방법이에요.

---

## 2. 데이터를 어떻게 나눠야 할까요?

### 기본 원칙 — 훈련과 테스트의 분리

머신러닝에서 데이터는 두 가지 역할을 해야 해요.

1. **훈련(Training):** 모델이 패턴을 배우는 데 써요. 로지스틱 회귀라면 곡선의 모양을 결정하는 수치들을 계산하는 단계예요.
2. **테스트(Testing):** 모델이 얼마나 잘 작동하는지 평가하는 데 써요. 처음 보는 데이터로 성능을 확인해야 해요.

> ⚠️ **절대 하면 안 되는 것:** 훈련 데이터와 테스트 데이터를 똑같이 쓰는 거예요. 족보로 공부하고 족보로 시험 보면 당연히 100점이지만, 진짜 실력을 모르는 거예요.

### 단순히 75%/25%로 나누면 되지 않나요?

"앞의 75%를 훈련에, 뒤의 25%를 테스트에 쓰자"는 아이디어는 그럴듯해 보여요. 하지만 문제가 있어요.

> 💡 **비유:** 수능 모의고사를 딱 한 번만 봤다고 해봐요. 그 한 번의 점수가 내 실력을 제대로 반영할까요? 운 좋게 잘 본 날일 수도 있고, 컨디션이 나빴던 날일 수도 있어요. 한 번의 결과만으로는 믿기 어려워요.

데이터를 나누는 것도 마찬가지예요. **어느 25%를 테스트에 쓰느냐에 따라 결과가 달라질 수 있어요.**

- 앞의 25%를 테스트로 쓰면?
- 중간의 25%를 테스트로 쓰면?
- 뒤의 25%를 테스트로 쓰면?

결과가 다 달라질 수 있어요. 어느 분할이 "진짜 성능"을 보여주는지 알 수 없어요.

## 3. 교차 검증의 해결 방법

### 🤔 아이디어

"어느 블록이 최선인지 걱정하지 말고, **전부 다 써봐요.**"

교차 검증은 데이터를 여러 블록으로 나누고, 블록을 돌아가며 테스트에 쓰는 방식이에요. 모든 블록이 한 번씩 테스트에 사용돼요.

### 4-겹 교차 검증 예시 (4-Fold Cross Validation)

데이터를 4개의 블록으로 나눠볼게요.

```
전체 데이터: [블록1] [블록2] [블록3] [블록4]
```

**1번째 평가:**
- 훈련: [블록1] [블록2] [블록3]
- 테스트: [블록4]
- → 성능 기록

**2번째 평가:**
- 훈련: [블록1] [블록2] [블록4]
- 테스트: [블록3]
- → 성능 기록

**3번째 평가:**
- 훈련: [블록1] [블록3] [블록4]
- 테스트: [블록2]
- → 성능 기록

**4번째 평가:**
- 훈련: [블록2] [블록3] [블록4]
- 테스트: [블록1]
- → 성능 기록

**최종 성능 = 4번 결과의 평균**

이렇게 하면 모든 데이터가 정확히 한 번씩 테스트 데이터로 사용돼요. 특정 블록 운에 의한 편차가 평균 내면서 상쇄돼요.

### 머신러닝 방법 비교에 적용하기

이걸 각 방법에 적용해요.

```
로지스틱 회귀   → 4-겹 교차 검증 → 평균 성능: 82%
KNN             → 4-겹 교차 검증 → 평균 성능: 79%
서포트 벡터 머신 → 4-겹 교차 검증 → 평균 성능: 85%
```

서포트 벡터 머신이 가장 높은 평균 성능을 보였으니까, 이 데이터에는 서포트 벡터 머신을 쓰는 게 좋아요.

> 💡 **강의 예시:** 강의에서는 서포트 벡터 머신이 가장 좋은 성능을 보인 예시를 들었어요. 어떤 방법이 이기느냐는 데이터마다 달라요.

## 4. 다양한 교차 검증 방법들

### ① k-겹 교차 검증 (k-Fold Cross Validation)

데이터를 k개의 블록으로 나눠서 k번 평가하는 방법이에요. 앞에서 본 4-겹이 이 방법이에요.

- **k = 4**: 4-겹 교차 검증 (데이터를 4번 평가해요)
- **k = 10**: 10-겹 교차 검증 (데이터를 10번 평가해요)

> 💡 **실무에서는?** 보통 10-겹 교차 검증을 가장 많이 사용해요. 10번 평가해서 평균을 내면 꽤 안정적인 성능 추정치를 얻을 수 있어요.

### ② 리브-원-아웃 교차 검증 (Leave One Out Cross Validation, LOOCV)

극단적인 경우로, 데이터 샘플 하나씩을 테스트에 사용하는 방법이에요.

예를 들어 데이터가 100개라면:
- 1번째: 1번 샘플을 테스트, 나머지 99개로 훈련
- 2번째: 2번 샘플을 테스트, 나머지 99개로 훈련
- ...
- 100번째: 100번 샘플을 테스트, 나머지 99개로 훈련

**100번 평가 후 평균**을 내요.

| 방법 | 블록 개수 | 특징 |
|---|---|---|
| 4-겹 교차 검증 | 4개 | 빠르고 간단해요 |
| 10-겹 교차 검증 | 10개 | 실무에서 가장 많이 써요 |
| LOOCV | 샘플 수만큼 | 가장 정밀하지만 시간이 많이 걸려요 |

---

## 5. 교차 검증의 또 다른 활용 — 하이퍼파라미터 튜닝

### 🤔 하이퍼파라미터가 뭔가요?

머신러닝 모델에는 두 종류의 값이 있어요.

- **모델이 스스로 학습하는 값:** 선형 회귀의 기울기, 절편 같은 것들이에요. 훈련하면서 자동으로 정해져요.
- **사람이 미리 정해줘야 하는 값:** 이걸 **하이퍼파라미터(Hyperparameter)** 라고 해요. 모델이 스스로 못 정해요. 예를 들어 릿지 회귀(Ridge Regression)의 "람다(λ)" 값이 있어요.

> 💡 **비유:** 요리를 할 때 "간장을 얼마나 넣을지"는 요리하면서 맛을 보며 조절할 수 있어요. 하지만 "몇 인분을 만들지"는 요리 시작 전에 미리 결정해야 해요. 하이퍼파라미터는 "몇 인분"처럼 사전에 결정해야 하는 값이에요.

### 교차 검증으로 최적의 하이퍼파라미터 찾기

하이퍼파라미터를 어떻게 설정하느냐에 따라 모델 성능이 달라져요. 최적의 값을 찾기 위해 교차 검증을 활용할 수 있어요.

```
예: 릿지 회귀의 람다 값 선택

람다 = 0.1 → 10-겹 교차 검증 → 평균 성능: 80%
람다 = 1.0 → 10-겹 교차 검증 → 평균 성능: 85%
람다 = 10  → 10-겹 교차 검증 → 평균 성능: 82%

→ 람다 = 1.0을 선택해요!
```

이렇게 여러 하이퍼파라미터 값을 교차 검증으로 비교해서 가장 좋은 값을 찾아요.

---

## ⚠️ 주의할 점 / 자주 하는 실수

- **k 값이 클수록 항상 좋은 건 아니에요.** k가 클수록 계산 시간이 늘어나요. 10-겹이 대부분의 상황에서 충분해요.
- **교차 검증은 최종 모델을 만드는 과정이 아니에요.** 어떤 방법/하이퍼파라미터가 좋을지 **선택**하는 과정이에요. 선택이 끝나면 **전체 데이터**로 최종 모델을 다시 훈련해요.
- **데이터가 시간 순서가 있는 경우 (예: 주가 데이터)** 는 일반 교차 검증을 쓰면 안 돼요. 미래 데이터로 과거를 예측하는 상황이 생길 수 있어요. 이런 경우엔 시계열 교차 검증을 사용해요.
- **교차 검증을 거쳐도 과적합이 아예 없어지는 건 아니에요.** 여러 하이퍼파라미터를 비교하다 보면, 교차 검증 성능 기준으로도 과적합이 생길 수 있어요.

**Q4.** 리브-원-아웃 교차 검증(LOOCV)은 언제 유용하고, 언제 비효율적인가요?

</details>

**Q5.** 교차 검증으로 가장 좋은 머신러닝 방법을 선택한 후, 최종 모델은 어떻게 만들어야 하나요?

<details>
<summary>정답 보기</summary>

**정답:** 선택된 방법으로 **전체 데이터**를 사용해서 최종 모델을 다시 훈련해요.

**설명:** 교차 검증은 어떤 방법이 좋을지 "선택"하는 과정이에요. 선택이 끝난 후에는 훈련/테스트로 나눴던 데이터를 다시 합쳐서 전체 데이터로 최종 모델을 훈련해요. 더 많은 데이터로 훈련하면 더 좋은 모델이 되기 때문이에요. 교차 검증 중에 만든 모델은 평가용이었고, 배포할 진짜 모델은 전체 데이터로 새로 만들어요.

</details>

### 실전 적용 문제

**문제.** 영화 리뷰 텍스트를 보고 긍정/부정을 판단하는 감성 분석 모델을 만들려고 해요. 리뷰 데이터가 1,000개 있어요. 교차 검증을 어떻게 활용할지 구체적인 절차를 설계해보세요.

**힌트:**
- k 값을 얼마로 할지 결정해봐요
- 어떤 머신러닝 방법들을 비교해볼지 생각해봐요
- 최종 모델은 어떻게 만들어야 하는지 떠올려봐요

<details>
<summary>예시 답안 보기</summary>

**1단계: k 결정**
1,000개 데이터면 10-겹 교차 검증이 적당해요. 각 블록에 100개씩 들어가고, 10번 평가해요.

**2단계: 방법 비교**
나이브 베이즈, 로지스틱 회귀, 랜덤 포레스트 같은 여러 방법을 10-겹 교차 검증으로 각각 평가해요. 각 방법에서 10번의 평균 정확도를 비교해요.

**3단계: 방법 선택**
10-겹 교차 검증에서 평균 정확도가 가장 높은 방법을 선택해요.

**4단계: 최종 모델 훈련**
선택된 방법으로 1,000개 전체 데이터를 사용해서 최종 모델을 훈련해요. 교차 검증 때 나눴던 데이터 분할은 버리고 전부 다 사용해요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>

## 🎯 핵심 요약

| 개념 | 한 줄 설명 |
|---|---|
| 🔄 교차 검증(Cross Validation) | 데이터를 여러 블록으로 나눠 돌아가며 모두 테스트에 사용하는 평가 방법이에요 |
| 🗂️ k-겹 교차 검증(k-Fold) | 데이터를 k개 블록으로 나눠 k번 평가하고 평균 내는 방법이에요. 실무에선 보통 10-겹을 써요 |
| 👤 리브-원-아웃(LOOCV) | 샘플 하나씩을 테스트로 쓰는 극단적인 교차 검증이에요. 데이터가 적을 때 유용해요 |
| 🎛️ 하이퍼파라미터(Hyperparameter) | 모델이 스스로 학습 못 하고 사람이 미리 설정해야 하는 값이에요 |
| 🏆 방법 선택 기준 | 교차 검증 평균 성능이 가장 높은 방법을 선택해요 |
| 🔧 하이퍼파라미터 튜닝 | 여러 하이퍼파라미터 값을 교차 검증으로 비교해서 최적값을 찾아요 |
| 🎯 최종 모델 훈련 | 방법 선택 후 전체 데이터로 다시 훈련해요. 교차 검증용 모델을 그대로 배포하지 않아요 |
