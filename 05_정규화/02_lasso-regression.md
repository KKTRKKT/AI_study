| L1 노름 | L1 Norm | 파라미터 절댓값의 합이에요. 라쏘 페널티의 다른 이름이에요 |

## 1. 라쏘 vs 릿지 — 딱 하나의 차이

### 공식 비교

**릿지 회귀가 최소화하는 것:**
```
잔차 제곱합 + λ × Σ(기울기)²
                    ↑ 제곱
```

**라쏘 회귀가 최소화하는 것:**
```
잔차 제곱합 + λ × Σ|기울기|
                    ↑ 절댓값
```

차이는 딱 하나예요. 릿지는 파라미터를 **제곱**해서 더하고, 라쏘는 파라미터의 **절댓값**을 더해요.

> 💡 **강의 예시:** "릿지 페널티를 L2 노름이라고 부르는데, 그냥 '제곱 페널티'라고 하면 더 기억하기 쉬워요. 라쏘 페널티는 L1 노름인데 '절댓값 페널티'라고 하면 더 직관적이에요."

### 겉보기엔 비슷해요

두 방법 모두:
- 페널티를 추가해서 파라미터를 줄여요
- 약간의 편향을 허용하는 대신 분산을 줄여요
- λ = 0이면 최소제곱과 동일해요
- λ를 교차 검증으로 찾아요
- 연속형, 이산형 변수에 모두 적용돼요
- 로지스틱 회귀에도 적용돼요

---

## 2. 결정적인 차이 — 라쏘는 파라미터를 0으로 만들 수 있어요

### 릿지의 한계

릿지 회귀에서 λ를 아무리 크게 해도 기울기는 **0에 가깝지만 정확히 0이 되지 않아요.**

```
릿지에서 λ 증가:
기울기 = 1.3 → 0.9 → 0.5 → 0.2 → 0.05 → 0.001 → ...
                                         → 절대 0이 되지 않음
```

### 라쏘의 강점

라쏘 회귀에서 λ가 충분히 크면 기울기가 **정확히 0이 될 수 있어요.**

```
라쏘에서 λ 증가:
기울기 = 1.3 → 0.9 → 0.5 → 0.2 → 0.05 → 0
                                         ↑ 정확히 0!
```

기울기가 0이 된다는 건 해당 변수가 모델에서 **완전히 제거**된다는 뜻이에요.

### 왜 이런 차이가 생기나요?

> 💡 **다음 문서(릿지 vs 라쏘)에서 그래프로 자세히 설명해요.** 간단히 말하면, 절댓값 함수는 0에서 "꺾임(kink)"이 생기는 특성이 있어서 최솟값이 0에서 발생할 수 있어요. 반면 제곱 함수는 0에서 꺾임이 없어서 최솟값이 0에서 발생하지 않아요.

---

## 3. 변수 선택(Feature Selection) — 라쏘의 핵심 강점

### 실제 예시: 쓸모없는 변수가 많은 모델

크기를 예측하는 이런 방정식이 있다고 해봐요.

```
크기 = 절편 + β₁×몸무게 + β₂×식이 + β₃×별자리 + β₄×제비의 비행 속도 + ...
```

- 몸무게, 식이: 크기 예측에 실제로 유용한 변수
- 별자리, 제비의 비행 속도: 크기와 전혀 관련 없는 쓸모없는 변수

**릿지 회귀를 적용하면:**
- λ를 크게 해도 β₃(별자리), β₄(제비 속도)는 아주 작은 값이 되지만 **0이 되지 않아요**
- 방정식에 여전히 존재해요 → 해석하기 복잡해요

**라쏘 회귀를 적용하면:**
- λ가 적당히 크면 β₃(별자리), β₄(제비 속도)가 **정확히 0이 돼요**
- 방정식에서 완전히 사라져요!

```
라쏘 결과:
크기 = 절편 + β₁×몸무게 + β₂×식이
           (별자리, 제비 속도는 사라짐!)
```

이렇게 중요한 변수만 남기고 쓸모없는 변수를 제거하는 것을 **변수 선택(Feature Selection)** 이라고 해요.

### 변수 선택의 장점

1. **모델이 단순해져요:** 해석하기 쉬운 모델이 만들어져요
2. **과적합 위험이 줄어요:** 불필요한 변수가 없으면 노이즈 학습을 방지해요
3. **계산 효율:** 변수가 적으면 예측이 빠르고 저장 공간도 적게 필요해요

---

## 4. 파라미터가 불균등하게 줄어들 수 있어요

릿지와 라쏘 모두 λ가 커질 때 모든 파라미터를 동일하게 줄이지 않아요.

예를 들어 몸무게 기울기(β₁)와 식이 차이(β₂)가 있을 때:

```
λ 증가에 따른 변화:
β₁ (몸무게): 1.2 → 0.9 → 0.6 → 0.4 (조금 줄어듦)
β₂ (식이):   0.7 → 0.3 → 0.1 → 0   (많이 줄어듦)
```

데이터와의 관계가 약한 변수(덜 중요한 변수)가 더 많이 줄어들어요. 이건 **데이터가 스스로 어떤 변수가 더 중요한지 알려주는** 방식이에요.

---

## 5. 릿지 vs 라쏘 — 언제 어느 걸 써야 하나요?

| 상황 | 추천 방법 |
|---|---|
| 대부분의 변수가 유용할 것 같음 | **릿지** (변수를 모두 유지하되 작게 만들어요) |
| 쓸모없는 변수가 많이 섞여 있음 | **라쏘** (쓸모없는 변수를 제거해줘요) |
| 모델을 단순하게 만들고 싶음 | **라쏘** (변수 선택 기능이 있어요) |
| 잘 모르겠음 | **엘라스틱넷** (두 방법을 조합해요 → 다음 문서) |

> 💡 **비유:**
> - **릿지** = 팀원 모두를 데려가되 각자 조금씩 짐을 줄이는 방식이에요
> - **라쏘** = 팀에서 기여가 없는 사람을 완전히 내보내는 방식이에요

---

## ⚠️ 주의할 점 / 자주 하는 실수

- **라쏘도 반드시 변수를 제거하지는 않아요.** λ가 충분히 크지 않으면 파라미터가 0이 되지 않을 수 있어요. λ를 교차 검증으로 적절히 선택해야 해요.
- **변수 스케일링이 필수예요.** 라쏘도 릿지와 마찬가지로, 변수들의 스케일이 다르면 페널티가 불균형하게 적용돼요. 반드시 표준화(standardization)를 먼저 해야 해요.
- **상관이 높은 변수가 있으면 라쏘가 불안정해요.** 두 변수가 강하게 상관되면, 라쏘는 그 중 하나만 남기고 나머지를 제거하는 경향이 있어요. 어떤 변수를 제거할지가 랜덤하게 달라질 수 있어요. 이 경우엔 엘라스틱넷이 더 좋아요.
- **제거된 변수가 진짜 쓸모없다는 보장은 없어요.** 라쏘가 제거한 변수가 실제로는 약간의 기여를 할 수도 있어요. 도메인 지식과 함께 결과를 해석해야 해요.

**설명:** 파라미터가 0인 변수들은 예측에 아무런 기여를 하지 않아요. 라쏘가 이들을 제거했다는 건 페널티와 데이터가 "이 변수들이 없어도 된다"는 것을 보여줬다는 뜻이에요. 결과적으로 100개 변수 대신 3개 변수만으로 예측하는 간결한 모델이 만들어졌어요. 이 희소(sparse) 모델은 해석이 쉽고 과적합 위험도 낮아요.

### 실전 적용 문제

**문제.** 암 진단에서 5,000개의 유전자 발현 데이터를 사용해서 암 유형(A, B, C)을 분류하는 모델을 만들려고 해요. 환자 데이터는 200명뿐이에요.

의사들은 최종 모델에서 "어떤 유전자가 암 유형 구분에 중요한가"를 알고 싶어해요.

1. 릿지 회귀와 라쏘 회귀 중 어느 것이 더 적합한가요?
2. 그 이유를 두 방법의 차이점과 연결해서 설명해보세요.
3. 라쏘 회귀를 사용했더니 5,000개 유전자 중 23개만 0이 아닌 파라미터를 가졌어요. 이것이 의사들에게 어떤 의미인가요?

**힌트:**
- 데이터 수(200) vs 파라미터 수(5,000) 비교를 생각해봐요
- 의사들이 "어떤 변수가 중요한지"를 알고 싶다는 점을 고려해봐요
- 변수 선택(Feature Selection)과 해석 가능성을 연결해봐요

<details>
<summary>예시 답안 보기</summary>

**1. 라쏘 회귀가 더 적합해요.**

**2. 이유:**
- **데이터 vs 파라미터:** 200명 vs 5,000개 파라미터로 데이터가 훨씬 적어요. 릿지도 이 경우를 해결할 수 있지만 5,000개 유전자 파라미터를 모두 유지해요.
- **해석 가능성:** 의사들은 어떤 유전자가 중요한지 알고 싶어해요. 릿지는 5,000개 유전자 모두 작은 파라미터를 갖기 때문에 "어느 유전자가 더 중요한가"를 파악하기 어려워요. 라쏘는 중요하지 않은 유전자를 0으로 만들어서 진짜 중요한 유전자만 남겨줘요.
- **변수 선택:** 5,000개 유전자 중 대부분은 암 유형과 관계없을 가능성이 높아요. 라쏘는 관련 없는 유전자를 자동으로 제거해요.

**3. 의미:**
23개 유전자만 남았다는 건, 라쏘가 이 23개 유전자가 암 유형 A, B, C를 구분하는 핵심 마커라고 판단했다는 뜻이에요. 의사들은 이 23개 유전자에 집중해서 생물학적 의미를 탐색할 수 있어요. 나머지 4,977개 유전자는 이 모델에서 예측력이 없다고 판단된 거예요. 이를 통해 훨씬 더 저렴한 검사(23개 유전자만 측정)도 개발 가능해질 수 있어요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>

| 🔑 핵심 차이 | 라쏘는 파라미터를 정확히 **0**으로 만들 수 있어요 (릿지는 0에 가깝게만 가능) |
| 🗑️ 변수 선택 | 쓸모없는 변수의 파라미터를 0으로 만들어 모델에서 제거해요 |
| 📊 사용 시점 | 쓸모없는 변수가 많을 때 → 라쏘 / 대부분 변수가 유용할 때 → 릿지 |
| ⚠️ 약점 | 상관된 변수가 있으면 하나만 남기고 나머지를 제거해서 불안정할 수 있어요 |
| 🔄 λ 선택 | 교차 검증으로 찾아요. 릿지와 동일한 방식이에요 |
