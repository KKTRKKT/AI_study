## 1. 문제 상황 — 데이터가 적으면 과적합이 생겨요

### 🤔 왜 릿지 회귀가 필요한가요?

생쥐 몸무게(x)와 크기(y) 데이터가 아주 많을 때는 최소제곱법이 잘 작동해요. 데이터가 많으니까 직선이 진짜 패턴을 잘 반영해요.

그런데 데이터가 딱 **2개**뿐이라면 어떨까요?

최소제곱법으로 직선을 맞추면 두 점을 정확히 통과하는 직선이 선택돼요. 잔차 제곱합 = 0이에요. 훈련 데이터에 완벽히 맞지만, 새로운 테스트 데이터에서는 형편없는 성능을 보여요. **과적합(Overfitting)** 이에요.

> 💡 **비유:** 시험 문제가 딱 2개인데 그 2문제 답을 달달 외웠어요. 그 2문제는 100점이지만 비슷한 다른 문제에는 모두 틀려요. 모델이 딱 2개 데이터를 외워버린 것과 같아요.

### 릿지 회귀의 핵심 아이디어

> **릿지 회귀란:** 훈련 데이터에 완벽히 맞추는 대신, **약간의 편향을 허용**하는 대신 **분산을 크게 줄이는** 직선을 찾는 방법이에요.

조금 덜 맞추는 대신, 새 데이터에서도 안정적으로 잘 작동하는 모델을 만들어요.

---

## 2. 릿지 회귀의 작동 원리

### 최소제곱법 vs 릿지 회귀

**최소제곱법이 최소화하는 것:**
```
잔차 제곱합 = Σ(실제값 - 예측값)²
```

**릿지 회귀가 최소화하는 것:**
```
잔차 제곱합 + λ × (기울기)²
```

차이는 딱 하나, **릿지 페널티** 항 `λ × (기울기)²`이 추가됐어요.

### 릿지 페널티가 하는 일

기울기가 크면 페널티도 커요. 페널티를 줄이려면 기울기를 작게 만들어야 해요. 그 결과 릿지 회귀는 최소제곱법보다 기울기가 **더 작은(평평한)** 직선을 선택해요.

### 구체적인 수치 비교

2개 데이터 포인트로 비교해볼게요. (λ = 1로 설정)

**최소제곱 직선:**
- 잔차 제곱합 = 0 (두 점을 완벽히 통과)
- 기울기 = 1.3
- 릿지 식 값 = 0 + 1 × 1.3² = **1.69**

**릿지 회귀 직선:**
- 잔차 제곱합 = 0.3² + 0.1² = 0.10 (훈련 데이터에 덜 맞음)
- 기울기 = 0.8 (더 완만함)
- 릿지 식 값 = 0.10 + 1 × 0.8² = **0.74**

릿지 회귀 직선이 `릿지 식 값`이 더 작아요(0.74 < 1.69). 그래서 릿지 회귀는 이 직선을 선택해요.

> 💡 **핵심:** 릿지 회귀로 선택한 직선은 기울기가 더 작아요. 기울기가 작으면 몸무게 변화에 따른 크기 예측이 덜 민감해져요. 즉, 작은 데이터 잡음에 휘청이지 않아요.

---

## 3. 기울기가 왜 중요한가요?

### 기울기와 예측 민감도의 관계

- **기울기가 클 때:** 몸무게가 조금만 변해도 예측 크기가 크게 달라져요 → 과적합 위험
- **기울기가 작을 때:** 몸무게가 바뀌어도 예측 크기 변화가 작아요 → 안정적인 예측

```
기울기 비교:
- 최소제곱: 기울기 1.3 → 몸무게 1 단위 증가 시 크기 1.3 증가
- 릿지 회귀: 기울기 0.8 → 몸무게 1 단위 증가 시 크기 0.8 증가
```

릿지 회귀는 예측을 훈련 데이터에 덜 "민감"하게 만들어요. 새로운 데이터가 왔을 때 더 안정적으로 예측해요.

## 4. 람다(λ) — 페널티 강도 조절

### 람다는 무엇인가요?

```
릿지가 최소화하는 것: 잔차 제곱합 + λ × 기울기²
```

**λ = 0:** 페널티가 없어요 → 최소제곱과 동일해요
**λ가 커질수록:** 페널티가 강해져서 기울기가 점점 작아져요
**λ → ∞:** 기울기가 0에 점근적으로 수렴해요 (하지만 정확히 0은 되지 않아요)

| λ 값 | 효과 |
|---|---|
| 0 | 페널티 없음 → 최소제곱과 동일 |
| 1 | 약한 페널티 → 기울기가 약간 줄어듦 |
| 10 | 강한 페널티 → 기울기가 많이 줄어듦 |
| 100 | 매우 강한 페널티 → 기울기가 0에 가까워짐 |

> ⚠️ **중요:** 릿지 회귀는 기울기를 **0에 가깝게** 만들 수 있지만 **정확히 0으로** 만들지는 못해요. 기울기를 완전히 0으로 만들고 싶다면 라쏘 회귀(다음 문서)를 써야 해요.

### 최적의 λ는 어떻게 찾나요?

**교차 검증(Cross-Validation)** 으로 찾아요!

여러 λ 값을 시도해보고, 테스트 데이터에서 가장 낮은 분산(MSE)을 보이는 λ를 선택해요. 보통 **10-겹 교차 검증**을 사용해요.

```
최적 λ 탐색:
λ = 0.01 → 10-겹 CV → MSE = 4.2
λ = 0.1  → 10-겹 CV → MSE = 3.1
λ = 1.0  → 10-겹 CV → MSE = 2.8 ← 가장 낮음, 선택!
λ = 10   → 10-겹 CV → MSE = 3.5
```

## 5. 다양한 상황에서의 릿지 회귀

### 이산형 변수에도 적용돼요

연속형(몸무게)뿐만 아니라 이산형(식이 유형: 일반식 vs 고지방식) 변수에도 릿지 회귀를 적용할 수 있어요.

방정식:
```
크기 = 1.5 + 0.7 × 고지방식(0 또는 1)
```

릿지 회귀는:
```
잔차 제곱합 + λ × (식이 차이)²
```

를 최소화해서 "식이 차이" 파라미터를 줄여요. 즉, 식이 유형이 크기 예측에 미치는 영향이 과대 추정되지 않도록 막아요.

### 로지스틱 회귀에도 적용돼요

로지스틱 회귀에서는 잔차 제곱합 대신 우도(Likelihood)를 사용하지만, 릿지 페널티는 동일하게 추가돼요.

```
최대화: 로그 우도 - λ × (기울기)²
```

로지스틱 회귀 기울기도 같은 방식으로 줄어요.

### 복잡한 모델에도 적용돼요

여러 변수를 동시에 쓰는 복잡한 모델에서:

```
크기 = y절편 + β₁×몸무게 + β₂×식이 차이 + β₃×별자리 + ...
```

릿지 페널티:
```
λ × (β₁² + β₂² + β₃² + ...)
```

> ⚠️ **y절편은 페널티에 포함되지 않아요!** y절편은 측정값에 스케일되지 않기 때문에 페널티에서 제외해요. 페널티는 β₁, β₂, β₃ 등 기울기 파라미터들에만 적용돼요.

---

## 6. 릿지 회귀의 마법 — 데이터보다 파라미터가 많을 때

### 🤔 이게 왜 "마법" 같은 일인가요?

일반적으로 최소제곱법은 **파라미터 수 ≤ 데이터 수**여야 작동해요.

- 파라미터 2개(절편 + 기울기)면 → 최소 2개 데이터 필요
- 파라미터 3개면 → 최소 3개 데이터 필요
- 파라미터 10,001개면 → 최소 10,001개 데이터 필요

### 현실의 문제

유전자 발현 데이터를 생각해봐요. 유전자가 10,000개면 파라미터가 10,001개(절편 포함)예요. 10,001마리의 생쥐에서 데이터를 모으는 건 비용이 엄청나요.

실제로는 500마리 정도밖에 없을 때가 많아요. 최소제곱법으로는 이 상황을 풀 수 없어요.

### 릿지 회귀는 풀 수 있어요

릿지 페널티를 추가하면 파라미터 수가 데이터 수보다 많아도 최적 해를 찾을 수 있어요.

> 💡 **직관:** 하나의 데이터 포인트로는 어떤 직선이 최선인지 알 수 없어요. 하지만 릿지 페널티가 "기울기를 작게 만드는 쪽으로" 가이드해줘요. 이 가이드 덕분에 데이터가 적어도 합리적인 해를 찾을 수 있어요.

---

## 7. 릿지 회귀의 한계

릿지 회귀는 파라미터를 **0에 가깝게** 만들 수 있지만, 정확히 **0으로** 만들지는 못해요.

즉, 별자리처럼 쓸모없는 변수를 추가해도 릿지 회귀는 그 파라미터를 완전히 제거하지 않아요. 아주 작게는 만들어주지만 0이 되지 않아요.

→ 이 문제를 해결하는 게 다음 문서의 **라쏘 회귀**예요!

## ⚠️ 주의할 점 / 자주 하는 실수

- **릿지 회귀 전에 변수 스케일링을 해야 해요.** 릿지 페널티는 파라미터 크기를 기준으로 벌점을 줘요. 키(cm)와 체중(kg)처럼 스케일이 다른 변수가 섞이면 페널티가 불공평하게 적용돼요. 먼저 표준화(평균 0, 표준편차 1)를 해야 해요.
- **λ = 0은 최소제곱법과 동일해요.** 릿지 회귀의 효과를 보려면 λ > 0이어야 해요.
- **λ가 너무 크면 편향이 너무 커져요.** 분산을 줄이려다 편향이 너무 높아지면 모델이 데이터를 제대로 설명하지 못해요. 교차 검증으로 균형점을 찾아야 해요.
- **y절편은 페널티에서 제외해요.** 이건 규칙이에요. 페널티는 기울기 파라미터들에만 적용해요.

**Q2.** 릿지 회귀가 최소화하는 목표 함수를 쓰고, 각 항이 무엇을 의미하는지 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** `잔차 제곱합 + λ × Σ(기울기)²`이에요.

**설명:**
- **잔차 제곱합:** 모델의 예측값과 실제값의 차이를 제곱해서 합산한 것이에요. 이걸 줄이면 데이터를 더 잘 설명해요.
- **λ × Σ(기울기)²:** 릿지 페널티예요. 기울기가 클수록 이 값이 커져요. 페널티를 줄이려면 기울기를 작게 만들어야 해요.
- λ가 두 항의 균형을 조절해요. λ가 크면 페널티 항이 지배적이 되어서 기울기가 작아져요.

</details>

**Q3.** λ를 0에서 점점 크게 늘리면 릿지 회귀 직선의 기울기는 어떻게 변하나요? 그리고 완전히 0이 될 수 있나요?

<details>
<summary>정답 보기</summary>

**정답:** 기울기가 점점 작아지고 0에 가까워지지만, 정확히 0이 되지는 않아요.

**설명:** λ = 0이면 최소제곱과 동일해요. λ가 커질수록 페널티가 강해져서 기울기가 줄어요. 하지만 릿지 페널티는 기울기의 제곱을 더하는 형태라서, 기울기가 0이 되면 페널티도 0이 돼서 기울기를 0으로 만들 "강제력"이 없어요. 수학적으로 기울기는 λ → ∞일 때만 0에 수렴해요. 즉, 유한한 λ로는 정확히 0이 되지 않아요.

</details>

---

**Q4.** 릿지 회귀에서 y절편을 페널티에 포함하지 않는 이유는 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** y절편은 측정값(x값)과 무관하게 더해지는 상수라서 스케일에 영향을 받지 않아요. 페널티로 줄여야 할 "과민한 파라미터"가 아니에요.

**설명:** 릿지 페널티의 목적은 예측이 입력 변수(x)의 작은 변화에 과민하게 반응하지 않도록 하는 거예요. 기울기 파라미터만 이런 민감성을 만들고, y절편은 x값에 무관하게 더해지는 상수예요. 그래서 y절편은 페널티 없이 자유롭게 추정돼요.

</details>

**Q5.** 파라미터가 10,001개인데 데이터가 500개밖에 없어요. 최소제곱법으로 이 문제를 해결할 수 없는 이유와, 릿지 회귀가 어떻게 해결하는지 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 최소제곱법은 데이터 수가 파라미터 수보다 많아야 풀 수 있어요. 릿지 회귀는 페널티 덕분에 데이터가 적어도 해를 찾을 수 있어요.

**설명:** 최소제곱법은 데이터 포인트들이 각 파라미터를 고정하는 "제약"을 제공해요. 파라미터 10,001개를 추정하려면 최소 10,001개의 독립적인 데이터가 필요해요. 데이터가 500개뿐이면 무수히 많은 해가 존재해서 어느 게 최선인지 알 수 없어요. 릿지 페널티는 "파라미터가 작은 쪽"을 선호하는 추가 제약을 부여해요. 이 제약 덕분에 데이터가 500개뿐이어도 유일한 최적 해를 찾을 수 있어요.

</details>

### 실전 적용 문제

**문제.** 부동산 가격을 예측하는 모델이 있어요.

- 데이터: 50채의 집 정보
- 변수: 면적, 방 개수, 층수, 주변 학교 수, 지하철 거리, 건축 연도, 리모델링 여부 (총 7개)

최소제곱법으로 모델을 만들었더니 훈련 데이터 R² = 0.98, 테스트 데이터 R² = 0.41이 나왔어요.

1. 이 결과가 나타내는 문제가 무엇인가요?
2. 릿지 회귀를 적용한다면 어떤 효과를 기대할 수 있나요?
3. 최적의 λ는 어떻게 찾아야 하나요?

**힌트:**
- 훈련 R²와 테스트 R²의 큰 차이가 의미하는 것을 생각해봐요
- 릿지 회귀의 핵심 효과(편향 vs 분산)를 떠올려봐요
- λ 최적화 방법은 이 문서에서 소개했어요

<details>
<summary>예시 답안 보기</summary>

**1. 과적합(Overfitting) 문제예요.**
훈련 R² = 0.98인데 테스트 R² = 0.41이면, 모델이 훈련 데이터를 거의 외웠지만 새로운 데이터에는 형편없이 작동하는 거예요. 데이터 50개에 비해 변수가 7개로 비교적 많고, 과적합이 발생한 상태예요.

**2. 릿지 회귀의 기대 효과:**
릿지 회귀는 페널티를 통해 모든 파라미터를 줄여요. 이를 통해 모델이 훈련 데이터의 잡음에 덜 민감해져요. 훈련 R²는 약간 낮아질 수 있지만(약간의 편향 증가), 테스트 R²는 올라갈 거예요(분산 감소). 두 R²의 차이가 줄어드는 게 목표예요.

**3. 최적 λ 찾기:**
10-겹 교차 검증(10-fold Cross-Validation)을 사용해요. λ = 0.001, 0.01, 0.1, 1, 10, 100 등 여러 값을 시도하고, 교차 검증 평균 MSE(또는 R²)가 가장 좋은 λ를 선택해요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>

## 🎯 핵심 요약

| 개념 | 한 줄 설명 |
|---|---|
| 🎯 릿지 회귀 목적 | 과적합을 줄이기 위해 파라미터에 페널티를 부과해요 |
| 📐 목표 함수 | 잔차 제곱합 + λ × Σ(기울기)² 를 최소화해요 |
| ⬇️ 페널티 효과 | 기울기가 작아져서 예측이 입력 변화에 덜 민감해져요 |
| λ | 페널티 강도예요. 0이면 최소제곱과 같고, 클수록 기울기가 0에 수렴해요 |
| 🔄 λ 선택 | 10-겹 교차 검증으로 최적 λ를 찾아요 |
| 🚫 0이 안 됨 | 릿지는 기울기를 0에 가깝게만 만들고 정확히 0으로 만들지는 못해요 |
| 📊 데이터 < 파라미터 | 릿지 회귀는 데이터보다 파라미터가 많아도 해를 찾을 수 있어요 |
| ⚠️ y절편 제외 | y절편은 페널티에서 항상 제외해요 |
