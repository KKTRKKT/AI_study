## 📌 이 문서에서 배우는 것

- 릿지 페널티와 라쏘 페널티의 차이를 그래프로 시각적으로 이해해요

## 1. 시작점 — 기본 설정

### 데이터와 목표

몸무게와 키 데이터로 직선을 맞추는 상황이에요.

y축 = 잔차 제곱합, x축 = 직선의 기울기로 그래프를 그려볼게요.

기울기가 0인 수평선에서 시작해요. 이때 잔차 제곱합은 매우 커요.

기울기를 조금씩 올리면 잔차 제곱합이 줄어들다가, 기울기 = 0.45에서 최솟값을 가져요. 이게 최소제곱 직선의 기울기예요.

```
잔차 제곱합
    │
    │ 포물선 모양
    │    ●
    │  ●   ●
    │ ●     ●
    │●       ●
    └──────────── 기울기
    0  0.2 0.45  0.7
              ↑
          최솟값
```

---

## 2. 릿지 회귀 시각화 — 파라미터를 0으로 만들 수 없는 이유

### λ = 0: 페널티 없음

λ = 0이면 페널티가 없어요. 릿지 식 = 잔차 제곱합이에요. 그래프는 원래 포물선 그대로예요. 최적 기울기 = 0.45예요.

### λ = 10: 약한 페널티

릿지 식 = 잔차 제곱합 + 10 × 기울기²

이제 기울기가 클수록 페널티도 커져요.

```
새 그래프 (λ=10):
    │ 포물선이
    │  ●       오른쪽으로 조금
    │ ● ●      치우쳐요
    │●   ●
    └──────────── 기울기
    0  0.3 0.45
       ↑
   새 최솟값 (0.45보다 작아짐)
```

λ = 0일 때 최적 기울기 0.45가 → λ = 10이면 더 작은 값으로 이동해요.

### λ = 20, 40, ...으로 계속 늘리면

λ가 커질수록 포물선의 최솟값이 점점 0쪽으로 이동해요. 하지만!

**릿지의 핵심 특성: 포물선의 최솟값은 절대 0에 정확히 도달하지 않아요.**

아무리 λ를 크게 해도, 포물선의 바닥은 기울기 = 0을 만들지 않아요. 포물선은 수학적으로 완만한 곡선이라서 최솟값이 x축 위의 어딘가에 있어요. 0에 가까워지기만 할 뿐 도달하지 않아요.

> 💡 **비유:** 계단 없는 부드러운 경사로는 경사가 0인 지점이 없어요. 끝없이 내려가다 수평에 가까워질 뿐이에요. 릿지의 포물선이 바로 이 부드러운 경사로와 같아요.

## 3. 라쏘 회귀 시각화 — 파라미터를 0으로 만드는 이유

### λ = 0: 동일한 시작점

λ = 0이면 라쏘도 릿지와 동일해요. 최적 기울기 = 0.45예요.

### λ = 10으로 늘리면

라쏘 식 = 잔차 제곱합 + 10 × |기울기|

절댓값 함수 |기울기|는 기울기 = 0에서 **꺾임(kink)** 이 생겨요!

```
절댓값 함수 모양:
    │    ╱
    │   ╱
    │  ╱
    │ ╱
    │╱_____________
             기울기 = 0
         ↑ 꺾임(kink)!
```

이 꺾임이 그래프에 영향을 줘요.

```
새 그래프 (라쏘, λ=10):
    │
    │ ●         포물선에 절댓값 페널티를
    │  ●        더하면 기울기 = 0 근처에서
    │   ●_●     꺾임이 생겨요
    └──────────── 기울기
    0  0.35 0.45
       ↑
   새 최솟값
```

### λ를 더 크게 (20, 40)으로 늘리면

λ = 40이 되면:

```
새 그래프 (라쏘, λ=40):
    │
    │ ●
    │  ●
    │   ●
    │    ↘꺾임이 더 뚜렷해짐
    │      ●
    └──────────── 기울기
    0    ↑
       최솟값이 0에 도달!
```

꺾임(kink)이 점점 뚜렷해지다가, λ = 40에서 **최솟값이 정확히 기울기 = 0**이 돼요.

이제 최적 기울기 = 0이에요. 즉, **몸무게 변수가 모델에서 완전히 제거됐어요.**

### 꺾임(kink)이 핵심이에요

> **수학적 이유:**
> - 릿지 페널티(x²): 0에서 기울기(미분)가 부드럽게 변해요. 0에서 특별한 점이 없어요.
> - 라쏘 페널티(|x|): 0에서 기울기(미분)가 급격히 변해요. 0에서 "꺾임"이 있어요.

이 꺾임이 최솟값을 0에서 "잡아당기는" 역할을 해요. λ가 충분히 크면 최솟값이 정확히 0에 고정돼요.

## 4. 릿지 vs 라쏘 최종 비교

| 특성 | 릿지 (L2) | 라쏘 (L1) |
|---|---|---|
| 페널티 함수 모양 | 부드러운 포물선 | 꺾임이 있는 절댓값 |
| λ 증가 시 기울기 | 0에 **수렴**하지만 도달 못 함 | 0에 수렴하다가 **정확히 0** 도달 가능 |
| 최솟값 위치 | 항상 x축 위 (0 초과) | x축 위 또는 정확히 0 |
| 변수 제거 | 불가능 | 가능 |

### 그래프로 기억하는 요약

**릿지:**
```
잔차+페널티
    │ λ=0 λ=10 λ=20 λ=40
    │  ↓   ↓    ↓    ↓
    │  ●   ●    ●    ●
    │   \  \    \    \
    │    \  \    \    \
    └──────────────────── 기울기
    0    최솟값들이 점점 0쪽으로 이동하지만 절대 0에 도달 못 함
```

**라쏘:**
```
잔차+페널티
    │ λ=0 λ=10 λ=20 λ=40
    │  ↓   ↓    ↓    ↓
    │  ●   ●    ●
    │   \  \    \\
    │    \  \   꺾임  ●←최솟값이 0에 도달!
    └──────────────────── 기울기
    0            ↑
              λ=40에서 최솟값이 정확히 0
```

---

## ⚠️ 주의할 점 / 자주 하는 실수

- **꺾임(kink)은 미분이 불연속이에요.** 라쏘 페널티의 0에서의 꺾임은 수학적으로 미분이 불연속이에요. 이 때문에 라쏘는 수치 최적화가 릿지보다 약간 더 복잡해요. 하지만 현대 소프트웨어는 이를 효율적으로 처리해요.
- **λ가 "충분히 클 때만" 0이 돼요.** λ가 작으면 라쏘도 기울기를 작게만 만들지 0으로 만들지 않아요. 변수 제거가 일어나는 λ의 임계값이 있어요.
- **그래프는 단순화된 1차원 예시예요.** 실제로는 여러 파라미터가 동시에 최적화돼요. 원리는 동일하지만 시각화가 복잡해져요.

### 개념 확인 퀴즈

**Q1.** 릿지 회귀에서 λ를 1,000,000으로 설정해도 기울기가 정확히 0이 되지 않는 이유를 시각적으로 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 릿지 페널티(기울기²)를 더한 그래프는 기울기 = 0에서 "꺾임"이 없는 부드러운 포물선이에요. 이 포물선의 최솟값은 항상 기울기 > 0인 지점에 있어요.

**설명:** 포물선의 수학적 특성 때문이에요. 기울기² 함수는 0에서 부드럽게 연결돼요. λ를 크게 해도 포물선이 납작해질 뿐, 최솟값이 0이 되지는 않아요. 최솟값이 0에 점점 가까워지지만 λ가 무한대가 되지 않는 한 정확히 0에 도달하지 않아요.

</details>

---

**Q2.** 라쏘 회귀에서 λ가 충분히 크면 최솟값이 정확히 0이 되는 이유를 "꺾임(kink)" 개념으로 설명해보세요.

<details>
<summary>정답 보기</summary>

**정답:** 절댓값 페널티(|기울기|)는 기울기 = 0에서 꺾임이 생겨요. 이 꺾임이 "0에서 최솟값을 고정시키는" 역할을 해요.

**설명:** |기울기| 함수는 기울기가 음수일 때는 오른쪽 방향으로 내려가고, 양수일 때는 오른쪽으로 올라가요. 0에서 방향이 갑자기 바뀌어요. 이걸 꺾임이라고 해요. 잔차 제곱합에 이 절댓값 페널티를 더하면, λ가 충분히 커졌을 때 0이 전체 그래프의 가장 낮은 점이 돼요. 꺾임이 최솟값을 0에 "고정"시키는 거예요.

</details>

---

**Q3.** λ = 0일 때 릿지와 라쏘의 결과가 동일한 이유는 무엇인가요?

<details>
<summary>정답 보기</summary>

**정답:** λ = 0이면 페널티 항(λ × 페널티)이 0이 되어서 사라져요. 릿지든 라쏘든 순수한 잔차 제곱합만 최소화하게 되므로 최소제곱과 동일한 결과가 나와요.

**설명:** 릿지 식 = 잔차 제곱합 + 0 × 기울기² = 잔차 제곱합. 라쏘 식 = 잔차 제곱합 + 0 × |기울기| = 잔차 제곱합. 둘 다 동일한 최소제곱 문제가 돼요. λ가 클수록 릿지와 라쏘의 결과가 달라지기 시작해요.

</details>

---

**Q4.** 릿지와 라쏘 중 어느 것이 수치 최적화(컴퓨터 계산)가 더 쉬운가요? 그 이유는?

<details>
<summary>정답 보기</summary>

**정답:** 릿지가 더 쉬워요. 릿지의 목적 함수는 모든 점에서 부드럽게 미분 가능하지만, 라쏘의 절댓값 함수는 0에서 미분이 불연속이에요.

**설명:** 컴퓨터는 기울기(미분)를 계산해서 최솟값을 찾아요. 릿지는 모든 점에서 미분이 가능해서 표준적인 최적화 알고리즘을 쓸 수 있어요. 라쏘는 0에서 미분이 정의되지 않아서(꺾임) 특별한 최적화 알고리즘(좌표 하강법 등)이 필요해요. 하지만 현대 소프트웨어는 이를 자동으로 처리해요.

</details>

### 실전 적용 문제

**문제.** 두 개의 데이터 포인트로 선형 모델을 학습했더니 최소제곱 기울기 = 2.0이 나왔어요.

- 릿지 (λ = 5): 기울기가 1.2가 됐어요
- 릿지 (λ = 50): 기울기가 0.3이 됐어요
- 라쏘 (λ = 5): 기울기가 1.0이 됐어요
- 라쏘 (λ = 50): 기울기가 0이 됐어요

1. λ = 5와 λ = 50에서 릿지와 라쏘의 기울기 차이를 비교하고, 이 패턴이 각 방법의 특성을 어떻게 보여주는지 설명해보세요.
2. 라쏘에서 기울기가 0이 됐다는 것은 실질적으로 어떤 의미인가요?

<details>
<summary>예시 답안 보기</summary>

**1. 비교 및 패턴 분석:**

| λ | 릿지 기울기 | 라쏘 기울기 |
|---|---|---|
| 0 | 2.0 | 2.0 |
| 5 | 1.2 | 1.0 |
| 50 | 0.3 | **0** |

λ = 5에서 릿지(1.2)와 라쏘(1.0) 모두 기울기를 줄였지만 비슷한 수준이에요.

λ = 50에서 릿지는 0.3으로 줄었지만 여전히 0보다 커요. 반면 라쏘는 정확히 0이 됐어요.

이 패턴이 보여주는 것:
- 릿지의 부드러운 포물선은 λ를 크게 해도 최솟값이 0에 도달하지 않아요 → 기울기 0.3
- 라쏘의 꺾임(kink)은 λ가 충분히 크면 최솟값을 0에 고정시켜요 → 기울기 0

**2. 기울기 = 0의 실질적 의미:**
몸무게라는 변수가 모델에서 완전히 제거됐어요. 방정식은 이제:
```
키 = 절편 + 0 × 몸무게 = 절편(상수)
```
즉, 어떤 몸무게든 키 예측값이 동일해요. 이 변수(몸무게)가 예측에 기여하지 않게 됐어요. 변수 선택이 이루어진 거예요.

> 정답이 하나가 아닐 수 있어요. 방향이 맞으면 돼요.

</details>
