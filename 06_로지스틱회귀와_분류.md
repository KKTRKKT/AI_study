# 🔵 로지스틱 회귀와 이진 분류

---

## 1. 분류(Classification) 문제란?

> **예측 결과가 숫자가 아닌 "카테고리(범주)" 중 하나인 문제**

---

### 💡 회귀 vs 분류 비교

```
선형회귀: 광고비 → 판매량 (얼마?)   ← 연속적인 숫자
분류:     이메일 → 스팸/정상?       ← 정해진 카테고리
```

| 구분 | 회귀 | 분류 |
|------|------|------|
| 출력 | 연속적인 숫자 | 미리 정해진 범주 |
| 예시 | 집값, 점수, 판매량 | 스팸/정상, 연체/정상 |

---

### 📌 범주형 변수란?

```
✅ 범주형 변수 (순서 없음):
- 혈액형: A, B, O, AB → A > B 라는 게 의미 없음
- 스팸: spam, normal

❌ 연속형 변수 (범주형 아님):
- 키: 170cm > 160cm → 크고 작음이 의미 있음
```

---

### 📌 범주를 직접 예측 vs 확률로 예측

단순히 "스팸이다/아니다"를 예측하는 것보다 **"스팸일 확률"** 을 예측하는 게 더 유용합니다.

```
직접 예측: "이 메일은 스팸입니다" ← 이분법적

확률 예측: "이 메일이 스팸일 확률은 94%입니다"
→ 확률 70% → "애매하네, 한 번 더 확인"
→ 확률 99% → "확실한 스팸, 바로 차단"
→ 확률 10% → "거의 정상 메일"
→ 의사결정의 유연성이 생김!
```

수식: **P(Y = k | X)** = "X가 주어졌을 때, Y가 k 범주에 속할 확률"

## 2. 신용카드 연체 예시 분석

```
입력(X): 신용카드 사용량(Balance), 소득(Income)
출력(Y): 연체 여부 (Yes / No)
```

### 📊 분석 결과

그래프(산점도, 박스플롯)를 분석하면:
- **사용량(Balance) 높음** → 연체 가능성 높음 ✅ (핵심 변수!)
- **소득(Income)** → 연체와 뚜렷한 관계 없음 ❌

```
박스플롯 요약:
연체자  사용량 중앙값 ≈ 1800 (훨씬 높음)
비연체자 사용량 중앙값 ≈ 800

연체자  소득 중앙값 ≈ 32,000 (거의 비슷)
비연체자 소득 중앙값 ≈ 34,000
```

## 3. 선형회귀를 분류에 쓰면 안 되는 이유

### ⚠️ 확률 범위를 벗어나는 문제

```
연체: Yes = 1, No = 0 으로 지정해서
선형회귀로 y = β₀ + β₁x 로 예측하면?

x가 매우 작을 때: y = -0.5  ← 확률이 음수?! ❌
x가 매우 클 때:   y = 1.5   ← 확률이 1 초과?! ❌

확률은 반드시 0 ≤ p ≤ 1 이어야 함!
```

## 4. 시그모이드(Sigmoid) 함수

> **어떤 값을 넣어도 항상 0~1 사이로 눌러주는 S자 함수**

```
        eᶻ          1
y = ───────── = ─────────
      1 + eᶻ    1 + e⁻ᶻ

e ≈ 2.71828 (오일러 수)
```

### 📊 입력 z에 따른 출력

| z값 | 시그모이드 출력 | 의미 |
|-----|--------------|------|
| -∞ | → 0 | 확률 거의 0% |
| -2 | ≈ 0.12 | 약 12% |
| 0 | = 0.5 | 정확히 50% |
| +2 | ≈ 0.88 | 약 88% |
| +∞ | → 1 | 확률 거의 100% |

> 💡 절대로 0 미만이나 1 초과가 되지 않습니다! 이게 핵심입니다.

## 5. 로지스틱 회귀 모형식

### 🔄 선형회귀 → 로지스틱 회귀 변환

```
1단계: 선형회귀 식 준비
z = β₀ + β₁x

2단계: 시그모이드 함수에 대입
         e^(β₀+β₁x)
p(X) = ─────────────── = P(Y=1 | X)
        1+e^(β₀+β₁x)

→ 결과: 항상 0~1 사이의 확률값!
```

## 6. 오즈(Odds)와 로짓 변환

### 📌 오즈(Odds)란?

```
         성공 확률      P(Y=1|x)
오즈 = ─────────── = ─────────────────
         실패 확률    1 - P(Y=1|x)

예시:
연체 확률 25% → 오즈 = 0.25/0.75 = 1/3
              "연체 안 할 가능성이 연체할 가능성의 3배"

연체 확률 80% → 오즈 = 0.8/0.2 = 4
              "연체할 가능성이 안 할 가능성의 4배"
```

### 📌 로짓 변환 (Logit = Log Odds)

```
logit(p) = log(오즈) = log( p / (1-p) )

오즈 범위: 0 ~ +∞  (비대칭)
log(오즈) 범위: -∞ ~ +∞  (대칭, 선형회귀처럼!)
```

### 🔗 핵심 연결고리

```
로지스틱 모형     ←→     로짓 변환 = 선형 모형

        e^(β₀+β₁x)          p(x;β)
p = ────────────────  ←→  log ────────── = β₀ + β₁x
     1+e^(β₀+β₁x)           1-p(x;β)

"로지스틱 모형 = 선형 모형 + 시그모이드 함수"
```

## 7. MLE (최대우도추정)로 β 찾기

### 💡 선형회귀와의 차이

```
선형회귀:    RSS(잔차제곱합) 최소화 → 공식으로 바로 계산 가능
로지스틱회귀: Likelihood(우도) 최대화 → 반복 계산 필요
```

### 📌 우도(Likelihood)란?

> **"현재 β값으로 만든 모델이 실제 데이터를 얼마나 잘 설명하는가"**

```
비유: 형사가 용의자를 찾는 상황
"이 증거들이 나올 확률이 가장 높은 용의자를 선택!"

머신러닝:
"현재 데이터들이 관측될 확률이 가장 높은 β를 선택!"
```

### 📌 Log-Likelihood (로그 우도)

곱셈을 덧셈으로 바꿔 계산하기 쉽게 만듭니다:

```
log L(β) = Σᵢ [yᵢ log p(xᵢ;β) + (1-yᵢ) log(1-p(xᵢ;β))]

- yᵢ = 1 (연체)이면:  log p(xᵢ;β)     ← 연체 예측 확률의 log
- yᵢ = 0 (비연체)이면: log(1-p(xᵢ;β))  ← 비연체 예측 확률의 log
```

> 💡 **log변환해도 같은 β를 얻는 이유:** log는 단조 증가 함수이므로, 최대화하는 β가 동일합니다.

## 8. 로지스틱 회귀 결과 해석: 신용카드 연체

### 📋 단순 로지스틱 회귀 (Balance만 사용)

| 항목 | 계수 | z통계량 | p-value |
|------|------|--------|---------|
| 절편 | -10.6513 | -29.5 | < 0.0001 |
| balance | 0.0055 | 24.9 | < 0.0001 |

**최종 로짓 모형식:**
```
log( p/(1-p) ) = -10.6513 + 0.0055 × Balance
```

**β₁ = 0.0055 해석:** Balance가 1단위 증가하면, 연체의 **로그오즈(log-odds)가 0.0055 증가**

### 🔢 실제 확률 계산 예시

```
Balance = 1000일 때:
z = -10.6513 + 0.0055×1000 = -5.15
p = 0.006  → 연체 확률 0.6%

Balance = 2000일 때:
z = -10.6513 + 0.0055×2000 = 0.35
p = 0.586  → 연체 확률 58.6%

입력 2배 증가 → 확률 97배 증가! (비선형적)
```

---

### 📋 다중 로지스틱 회귀 (Balance + Income + Student)

| 항목 | 계수 | p-value | 유의미? |
|------|------|---------|--------|
| 절편 | -10.8690 | < 0.0001 | ✅ |
| balance | 0.0057 | < 0.0001 | ✅ |
| income | 0.0030 | 0.7115 | ❌ |
| student[Yes] | -0.6468 | 0.0062 | ✅ |

#### 🤔 학생이 연체 확률이 낮다?

```
단순 분석: 학생 → 소득 낮음 → 연체 많을 것 같음

하지만 다중회귀 결과:
사용량(Balance)을 통제하면 → 학생이 오히려 연체 확률 낮음!

이유: 학생들은 카드 사용량 자체가 낮은 경향
→ 사용량을 통제했을 때 순수한 "학생 효과"는 오히려 음수

→ 이게 다중회귀의 강점: 다른 변수를 통제했을 때의 순수한 효과를 봄!
```

## 9. 전체 흐름 정리

```
분류 문제 발견
(출력이 카테고리)
        ↓
선형회귀 사용 불가
(확률 범위 벗어남)
        ↓
로지스틱 회귀 도입
z = β₀ + β₁x (선형)
        ↓ 시그모이드
p(x) = 1/(1+e⁻ᶻ)  (0~1 사이)
        ↓
MLE로 β 추정
log L(β) 최대화
        ↓
결과 해석
- p-value로 유의미성
- 계수는 로그오즈 변화량
- 실제 확률은 수식 대입
```

> **핵심 요약:** 로지스틱 회귀는 선형 결합을 시그모이드로 감싸서 항상 0~1 사이의 확률을 출력합니다. 이를 통해 분류 문제를 확률적으로 접근할 수 있습니다.
