# LLM - Detect AI Generated Text - 1위 솔루션: 다양한 데이터셋 구성이 핵심

**대회**: LLM - Detect AI Generated Text  
**순위**: 1위 (팀)  
**작성자**: Nicholas Broad, Udbhav Bamba, Raja Biswas  
**URL**: https://www.kaggle.com/competitions/llm-detect-ai-generated-text/discussion/473295

---

## 대회 개요

학생 에세이가 AI가 작성한 것인지 인간이 작성한 것인지 예측하는 이진 분류 문제입니다.

---

## 핵심 아이디어: 데이터 다양성이 모델보다 중요하다

모델링 방식보다 **데이터셋의 품질과 다양성**이 승패를 결정했습니다. 단일 모델도 0.970+ 점수를 달성할 만큼 데이터 구성이 핵심이었습니다.

---

## 데이터 구성 전략 (Datamix)

### 인간 작성 텍스트 소스
- PERSUADE corpus (15개 프롬프트 전체)
- OpenAI GPT2 output dataset, ELLIPSE corpus, NarrativeQA, Wikipedia, NLTK Brown corpus, IMDB 리뷰

### AI 생성 텍스트 소스
- **상용 LLM**: GPT-3.5, GPT-4, Claude, Cohere, Gemini, Palm
- **오픈소스 LLM**: Llama, Falcon, Mistral, Mixtral
- **기존 AI 텍스트 데이터셋**: DAIGT V2, OUTFOX, Ghostbuster data, GPT-2 output
- **파인튜닝된 오픈소스 LLM**: Mistral, Llama, Falcon, T5, BLOOM, GPT2 등

### 파인튜닝 전략 (다양성 극대화)
- **Instruction tuning**: 에세이 메타데이터(점수, 학년 등)를 지시문으로 사용
- **One topic held out**: 한 프롬프트 제외 후 학습, 제외된 프롬프트로만 생성 → 새로운 문체 유도
- **Span-wise generation**: 한 번에 한 구간씩 생성

### 생성 다양성 확보 기법
- Contrastive search, Guidance scale, High temperature, Large top-k
- Fill-in-the-blank 프롬프팅 (랜덤 마스킹 후 재구성 요청)
- 기존 에세이 재작성 프롬프팅

### 데이터 증강 (공격 방어)
- 철자 교정/삽입/삭제/스왑, 동의어 교체, 역번역, 대소문자 무작위 변환

최종 학습 데이터: 160k 샘플 (이 중 인간 작성 40k)

## 모델 구성

| 모델 전략 | 세부 내용 |
|----------|---------|
| LLM (Q)LoRA 파인튜닝 | Mistral-7B, r=64, alpha=16 |
| DeBERTa 분류 | deberta-v3-large, BCE Loss + 연속 레이블 |
| 커스텀 토크나이저 + MLM + PL | deberta-v3-small, 테스트셋 의사 레이블링 |
| Ranking | deberta-v3-large, Pairwise loss with margin |
| Ghostbuster | Llama 7B + Tiny Llama 1.1B 토큰 확률 기반 |
| Ahmet의 비지도 접근법 | 이웃 에세이 유사도 기반 앙상블 |

---

## 앙상블 방법

- 원시 예측값 대신 **순위(ranking)** 기반 앙상블
- 테스트 에세이를 점수 순으로 순위 부여 후 순위를 평균
- Mistral-7B 모델에 가장 높은 가중치 적용

---

## 핵심 교훈

- **데이터가 전부**: 모델보다 다양하고 품질 높은 데이터셋 구성이 결정적
- 공격적 예시(obfuscation, character-level attack)를 학습 데이터에 포함해야 일반화 성능 향상
- 전처리 버전(소문자화, 공백 정규화) 모델을 앙상블에 추가하면 다양성 확보 가능
