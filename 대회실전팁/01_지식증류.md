## 대회 개요

이 대회는 두 LLM의 응답을 보고 사람이 어느 쪽을 선호하는지(A 승리 / B 승리 / 무승부) 예측하는 3-클래스 분류 문제입니다.

---

## 핵심 아이디어: **지식 증류(Knowledge Distillation)**

이 솔루션의 핵심은 **대형 모델에서 소형 모델로 지식을 증류**하는 것입니다.

---

## 사용 데이터셋

- Kaggle 공식 학습 데이터
- UT 데이터 (~ut dataset)
- 33k 추가 데이터 (중복 제거 후 21k)

---

## 기반 모델(Base Models)

| 모델 | 파라미터 | 역할 |
|------|---------|------|
| Llama3 70B | 70B | 교사 모델 |
| Qwen2 72B | 72B | 교사 모델 |
| Gemma2-9B | 9B | 학생 모델 (최종 추론 모델) |

---

## 학습 설정
```
- 모델 아키텍처: AutoModelForSequenceClassification
- LoRA: r=64, alpha=128 (모든 선형 레이어 대상)
- QLoRA: Llama3 70B, Qwen2 72B에 적용
- 최대 길이: 1024 토큰
- 에폭: 2
- 글로벌 배치 크기: 64
```

---

## 학습 파이프라인

### 1단계: 사전 파인튜닝 (Post-pretrain)
- UT 데이터셋으로 3개 모델 각 1 에폭 학습 (lr=1e-5)
- 학습된 모델에서 logits 분포를 추출

### 2단계: 대형 모델 파인튜닝
- 데이터를 5-fold로 분할
- Llama3 70B + Qwen2 72B로 학습
- 학습 세트의 확률 분포(logits) 추출

### 3단계: 소형 모델로 증류 (핵심!)
- 추출한 logits 분포를 사용해 Gemma2-9B 파인튜닝
- KL Divergence Loss + Cross Entropy Loss 등 3가지 이상 손실 함수 사용 (lr=5e-5)
- **Cosine 스케줄러 + 100 스텝 워밍업**

### 4단계: 모델 앙상블
- 5-fold 모델의 LoRA 레이어를 단순 평균(linear averaging)으로 합산

### 5단계: 8비트 양자화 (GPTQ)
- 최종 제출 시 GPTQ로 8비트 양자화
- TTA (Test-Time Augmentation) 적용 (최대 길이 2000)

## 성능 결과

| 모델 | CV (5-fold 평균) |
|------|----------------|
| Qwen2 72B | ~0.876 |
| Llama3 70B | ~0.875 |
| 증류된 Gemma2-9B | ~0.867 |

- **LB (LoRA merge + 8bit 양자화)**: 0.882
- **최종 Private LB**: 0.96898 🏆

## 핵심 교훈

1. **지식 증류가 핵심**: 큰 모델(70B/72B)의 soft label을 작은 모델(9B)에게 전달하면 성능이 크게 향상됨
2. **소프트 레이블 사용**: Hard label보다 logits 분포를 직접 증류하는 것이 효과적
3. **TTA 효과**: 테스트 시 응답 A/B를 바꿔서 예측하는 TTA가 긴 토큰보다 더 효율적
4. **Kaggle 인퍼런스 제약**: 큰 모델은 Kaggle 환경에서 직접 추론 불가 → 소형 모델로 증류가 필수
