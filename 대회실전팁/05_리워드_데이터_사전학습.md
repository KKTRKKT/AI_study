# LMSYS Chatbot Arena - 5위 솔루션: Team Danube

**대회**: LMSYS - Chatbot Arena Human Preference Predictions  
**순위**: 5위 (Team Danube: Psi, ilu000, dott1718)  
**URL**: https://www.kaggle.com/competitions/lmsys-chatbot-arena/discussion/527669

---

## 핵심 전략

**리워드 데이터 사전학습(Pre-training) + Gemma2-9B 파인튜닝**

다른 팀들과 달리 Pseudo Labeling이나 지식 증류를 사용하지 않고 **공개 리워드 데이터로 사전학습**이 핵심이었습니다.

---

## 아이디어 발굴 과정

1. RewardBench 최상위 모델들이 기반 모델(base/instruct)보다 파인튜닝 후 성능이 좋음을 발견
2. `RLHFlow` 모델들이 UltraFeedback 등의 데이터로 학습된 것을 확인
3. → Gemma2-9B를 공개 리워드 데이터(주로 UltraFeedback)로 **사전학습** 결정

---

## 사전학습 방법

- UltraFeedback 데이터를 **binary win/loss 레이블**로 학습
- 데이터 양이 결정적으로 중요하지는 않음 (전체 UltraFeedback 사용으로 충분)
- **H2O LLM Studio** 사용 (코드 없이 GUI로 학습)

→ 이 사전학습 체크포인트를 시작점으로 파인튜닝하면 **CV/LB에서 최대 +20 포인트** 향상!

---

## 최종 모델 구성

- **동일한 설정으로 2번 전체 데이터 학습** (epoch=1씩, 학습 순서만 다름)
- 1번 모델: 원본 순서로 추론
- 2번 모델: 응답 A/B 스왑 순서로 추론 (TTA 효과)
- 두 모델의 예측 앙상블

---

## 추론 최적화

- 두 GPU에 데이터를 반씩 나눠 동시 처리
- 길이 기준 정렬 및 동적 배치 구성
- INT8 (bitsandbytes) + float16 계산
- 최대 컨텍스트: 4096 토큰 (최종 제출 4k 사용 가능)

---

## 성능

- 단일 모델 CV: ~0.880
- 2모델 앙상블 CV: ~0.873

---

## 교훈

- **사전학습에서 head를 버리고 새로운 head로 파인튜닝**해야 효과적
- 사전학습 설정은 크게 중요하지 않으며, 어떤 형태의 사전학습이든 없는 것보다 나음
- GPTQ 양자화는 단일 토큰 예측 시 오히려 느림 (BitsAndBytes 8bit가 더 빠름)
