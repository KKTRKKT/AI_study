## LLM - Detect AI Generated Text 대회 2등 솔루션 설명

이 글은 Kaggle의 **"LLM - Detect AI Generated Text"** 대회에서 **2등을 차지한 Guanshuo Xu(`wowfattie`)**의 풀이 설명서입니다.

---

### 대회 배경
이 대회의 목표는 **글이 사람이 쓴 것인지, AI가 생성한 것인지를 구별하는 이진 분류 문제**입니다. LLM(대형 언어 모델)의 등장으로 AI 생성 텍스트 탐지가 중요한 연구 주제가 되었고, 이 대회가 그 연구에 기여하는 것을 목표로 합니다.

---

### 풀이 단계별 설명

**1단계: 초기 시도 실패 — DeBERTa 파인튜닝**

처음에는 다른 참가자들이 공유한 데이터셋으로 **DeBERTa 모델을 파인튜닝**했습니다. 결과는 교차 검증(CV)에서는 거의 완벽한 점수가 나왔지만, 공개 리더보드(LB)에서는 훨씬 낮고 불안정한 점수가 나왔습니다. 즉, 모델이 **특정 데이터에 과적합(overfitting)**되어 범용성이 없었던 것입니다.

**2단계: 방향 전환 — 대규모·다양한 데이터 생성 (핵심!)**

다른 참가자인 James의 포스트에서 힌감을 받아, **대규모로 다양한 데이터를 만들어 범용적인 모델을 사전 훈련(pretraining)하는 것**이 핵심이라는 것을 깨달았습니다.

이를 위해 **SlimPajama 데이터셋**에서 여러 LLM을 사용해 약 **50만 쌍의 인간 글 / AI 글 데이터**를 며칠 만에 직접 생성했습니다. 이 데이터로 DeBERTa-v3-large를 훈련하자 공개/비공개 LB가 **0.916 / 0.967**로 향상되었습니다.

> 이 단계가 핵심입니다. 특정 주제에 편향되지 않은 방대한 데이터로 "AI 글인지 아닌지"를 일반적으로 구분할 수 있는 모델을 만든 것입니다.

**3단계: 도메인 적응 파인튜닝 — 학생 에세이 스타일 모방**

테스트 데이터가 **Persuade Corpus(학생 에세이 데이터)**에서 나온다는 것을 파악했습니다. 테스트 프롬프트 5개가 모두 특정 기사를 읽고 에세이를 쓰는 과제였는데, 이 기사들을 이용해 AI가 글을 생성하면 학생 글과 매우 비슷해질 수 있다고 판단했습니다.

이를 해결하기 위해 **LLM을 학생 에세이 스타일에 맞게 추가 훈련(LM 파인튜닝)**시켰습니다. 즉, AI가 학생처럼 쓰게 만들었고, 심지어 원문 인용이나 오타까지 포함하도록 했습니다. 이렇게 생성한 데이터로 분류기를 추가 파인튜닝하자 공개/비공개 LB가 **0.94 / 0.98** 수준으로 크게 올랐습니다.

> 도구로는 코드 없이 GUI만으로 LM 파인튜닝이 가능한 **llm-studio**를 사용했습니다.

**4단계: 앙상블(모델 합치기)**

최종 제출물은 아래 여러 모델의 예측을 합친 **앙상블**이었습니다.
- DeBERTa-v3-large 모델 × 2
- DeBERTa-large 모델 × 1
- 위 3개 모델을 DAIGT-V4 데이터셋으로 추가 파인튜닝한 버전 × 3

이 앙상블로 공개/비공개 LB가 **0.967 / 0.983**까지 올랐습니다.

또한 TF-IDF 기반 머신러닝 예측과 블렌딩(blending)하면 공개 LB는 더 올랐지만, 비공개 LB는 오히려 낮아져 최종 제출에서는 사용하지 않았습니다.

---

### 전체 흐름 요약

```
[SlimPajama 대규모 데이터로 범용 분류기 사전 훈련]
           ↓
[학생 에세이 스타일로 LLM 파인튜닝 → AI 글 생성]
           ↓
[해당 도메인 데이터로 분류기 추가 파인튜닝]
           ↓
[여러 DeBERTa 모델 앙상블 → 최종 제출]
```

---

### 핵심 교훈

- **데이터 품질과 다양성**이 가장 중요했습니다. 과적합된 데이터로 아무리 훈련해도 실제 테스트엔 약했습니다.
- **도메인 적응**: 테스트 데이터가 어떤 스타일인지 파악하고, 그에 맞는 데이터를 직접 생성해 추가 훈련한 것이 결정적인 성능 향상을 가져왔습니다.
- 사용 하드웨어: **NVIDIA RTX A6000 × 3대**를 사용했습니다.
