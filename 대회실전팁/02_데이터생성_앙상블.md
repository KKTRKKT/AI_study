## 핵심 전략

1. **Full Swap 증강**: 응답 A/B 순서를 바꾸는 데이터 증강
2. **Pseudo Labeling**: 대량의 외부 데이터에 소프트 레이블 적용 (3-stage 학습)
3. **패딩 없는 효율적 학습**: sequence concat 기반 학습

---

## 베이스라인 설정

- 검증 세트: StratifiedGroupKFold (프롬프트 기준), 20% 홀드아웃
- 외부 데이터: 33k 중복 제거 데이터 21k 추가
- 입력 포맷: `프롬프트-응답A-응답B` (PAB 형식)
- 최대 시퀀스 길이: Gemma2-9B 기준 약 4340 토큰

### 커스텀 분류 헤더

기본 `XXXForSequenceClassification` 대신 드롭아웃과 GELU를 포함한 2-layer 헤더 사용:
```python
model.score = torch.nn.Sequential(
    torch.nn.Dropout(0.1),
    torch.nn.Linear(hdim, hdim // 2),
    torch.nn.Dropout(0.1),
    torch.nn.GELU(),
    torch.nn.Linear(hdim // 2, 3),
)
```

## Full Swap 기법 (핵심!)

- 무작위 스왑 대신 **원본 샘플 + 스왑 샘플을 항상 쌍으로 학습**
- 동일 optimizer.step에서 그래디언트를 누적하여 오버피팅 방지
- 학습 시간은 2배 증가하지만 성능 **+0.003 향상** (random swap 대비)

---

## 3단계 학습 파이프라인

### Stage 1: 기본 모델 학습
- 파인튜닝 모델: `gemma-2-9b-it`, `gemma-2-27b-it`, `RLHFlow/ArmoRM-Llama3-8B-v0.1`
- TTA 없이 검증 log-loss: 0.891 / 0.883 / 0.899
- 앙상블 후: 0.876

### Stage 2: 의사 레이블링 (Pseudo Labeling)
- Stage 1 앙상블로 **240k 데이터에 의사 레이블** 생성
  - lmsys-1m에서 110k
  - 기타 외부 데이터셋에서 130k
- 이 데이터로 Gemma2-9B와 Llama3-8B 추가 파인튜닝

### Stage 3: 최종 모델 학습
- Stage 2 체크포인트 기반으로 55k+21k 데이터 파인튜닝
- 검증 log-loss: 0.884 / 0.890 (앙상블 평균 ~0.877)

## 효율적 학습/추론 기법

### 학습 속도 개선
- `flash-attn 2.6.2` (logit_softcapping 지원)
- 패딩 없는 sequence concat으로 불필요한 계산 제거
- RMSNorm, FusedRoPEFunc from transformer_engine 활용

### 추론 속도 개선
- T4×2로 7B-9B 모델 FP16 추론 가능
- GPU 2개에 모델 분산 + 파이프라인 병렬 처리
- Triton 연산자 활용 (lightllm, lmdeploy)

## 최종 제출

- Gemma2-9B (1개) + Llama3-8B (1개) 앙상블
- Llama3-8B는 입력 A/B 스왑 적용 (TTA 효과)
- 최대 토큰: Gemma2는 4340, Llama3는 4096
