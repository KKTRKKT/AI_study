## LLM - Detect AI Generated Text 대회 3등 솔루션 설명

이 글은 같은 대회(AI 생성 텍스트 탐지)에서 **3등을 차지한 팀(Yevhenii Maslov 등 5인 팀)**의 풀이 설명서입니다. 2등 솔루션과 비교하면 훨씬 더 **체계적이고 복잡한 파이프라인**을 사용했습니다.

---

### 최종 솔루션 구성 한눈에 보기

최종 제출물은 크게 두 가지를 **가중 평균으로 앙상블**한 결과입니다.
- **DeBERTa-v3-large 모델 12개**의 앙상블
- **TF-IDF 기반 머신러닝 파이프라인**

---

### 1부: 트랜스포머(DeBERTa) 앙상블

#### A. 데이터 전처리

- 텍스트에 숨겨진 오류(난독화)를 복원하는 **디오브퍼스케이터(deobfuscator)** 도구를 사용했습니다. 단, 오류가 **15개 이상인 텍스트만** 교정했습니다.
- 원본 훈련 세트에 없는 이상한 특수 기호를 제거하고, 인코딩을 정규화했습니다.

#### B. 11k 선별 데이터로 훈련한 4개 모델

약 11,000개의 선별된 데이터로 DeBERTa 모델 4개를 훈련했습니다. 이 데이터를 만드는 방법이 독특합니다.

**반복적 하드 샘플 마이닝 알고리즘:**
1. 먼저 기존 공개 데이터로 초기 모델을 훈련합니다.
2. 이 모델이 **틀리게 예측한 샘플**들을 찾습니다 (정답과 가장 거리가 먼 것 각 500개씩).
3. 그 어려운 샘플들을 훈련 데이터에 추가합니다.
4. 새 모델을 훈련하고, 이 과정을 반복합니다.
5. 4번 반복할 때마다 리더보드(LB)로 검증하고, LB가 더 이상 오르지 않으면 직전 최고 데이터셋을 채택합니다.

> 즉, 모델이 어려워하는 샘플만 골라서 계속 추가하는 **능동 학습(Active Learning)** 방식입니다.

이 데이터로 훈련한 단일 최고 모델: 공개 **0.927** / 비공개 **0.845**

#### C. 대규모 생성 데이터로 훈련한 3개 모델 (핵심!)

2등 솔루션처럼 **Pile, SlimPajama 데이터셋에서 AI 글을 대량 생성**했습니다.

생성 방법이 매우 정교합니다.
- 너무 짧거나 길거나, 코드/수식 포함, 비영어, 특수문자 비율이 높은 텍스트는 **필터링으로 제거**.
- **35가지 이상의 다양한 오픈소스 LLM 모델** 사용.
- 샘플링 파라미터(temperature, top_p, min_p 등)를 **온도값에 따라 3가지 시나리오로 나눠** 무작위로 조합 → 매우 다양한 스타일의 AI 글 생성.
- 이렇게 50만 / 100만 / 120만 개 규모의 데이터 3세트를 만들어 각각 모델을 훈련.
- 훈련 시 최대 길이 256(추론 시엔 1512), 배치 크기 48로 효율적으로 훈련.

이 데이터로 훈련한 단일 최고 모델(~100만 개 샘플): 공개 **0.956** / 비공개 **0.967**

#### D. 500k 이상 모델 가중치로 추가 파인튜닝한 5개 모델

위 C단계에서 훈련한 모델 가중치를 출발점으로, B단계의 11k 선별 데이터로 **추가 파인튜닝**한 모델 5개를 만들었습니다. 공개 LB는 약간 올랐지만, 비공개 LB는 ~0.005 낮아졌습니다.

---

### 2부: TF-IDF 파이프라인

공개된 초기 노트북을 기반으로 몇 가지 개선을 했습니다.
- CatBoost, LightGBM의 반복 횟수를 250 증가시켰습니다.
- 투표 분류기의 가중치를 `[0.05, 0.225, 0.225, 0.5]`로 조정했습니다.
- **의사 레이블링(Pseudo-labeling)**: 트랜스포머 앙상블이 확신하는(확률 0.01 미만 또는 0.99 초과) 테스트 샘플 1,000개를 훈련 데이터에 추가했습니다.

이 개선으로 비공개 LB가 **0.893 → 0.927**로 크게 향상되었습니다.

---

### 3부: 최종 앙상블 전략

단순히 모든 모델을 평균 낸 것이 아니라, **2단계 가중 평균**을 사용했습니다.

**1단계:** 트랜스포머 예측값이 확신이 높은 구간(0.1 미만 또는 0.9 초과)에서만 11k 모델과 TF-IDF를 혼합, 나머지 불확실한 구간은 TF-IDF만 사용.

**2단계:** 1단계 결과와 대규모 데이터로 훈련한 모델들을 조건 없이 가중 평균.

---

### 4부: 후처리 (Postprocessing)

같은 프롬프트(prompt_id)로 작성된 샘플이 1,000개 이상인 경우, 추가적인 거리 기반 조정을 했습니다.
- UMAP으로 TF-IDF 벡터를 차원 축소합니다.
- 각 샘플에서 **가장 가까운 인간 글 7개**, **가장 가까운 AI 글 7개**까지의 거리를 계산합니다.
- `인간_거리 / AI_거리` 비율로 예측값을 조정합니다 (0.9~1.1 범위로 제한).

---

### 전체 흐름 요약

```
[전처리: 디오브퍼스케이션, 정규화]
        ↓
[11k 하드 샘플 데이터 → DeBERTa 4개 훈련]
[대규모 생성 데이터(50만~120만) → DeBERTa 3개 훈련]
[대규모 모델 가중치 + 11k 데이터 → DeBERTa 5개 파인튜닝]
[TF-IDF 파이프라인 + 의사 레이블링]
        ↓
[2단계 가중 평균 앙상블]
        ↓
[UMAP 거리 기반 후처리]
```

---

### 핵심 교훈

- **데이터 다양성**이 가장 큰 성능 향상 요인이었습니다. 35개 이상의 다양한 모델과 파라미터 조합으로 매우 다양한 AI 글을 생성한 것이 핵심입니다.
- **반복적 하드 샘플 마이닝**: 모델이 틀리는 어려운 데이터만 계속 골라 훈련하는 방식이 효과적이었습니다.
- 단일 모델 앙상블 효과는 크지 않았고, 성능의 대부분은 **훈련 데이터의 품질과 구성**에서 나왔습니다.
