# LMSYS Chatbot Arena - 3위 솔루션

**대회**: LMSYS - Chatbot Arena Human Preference Predictions  
**순위**: 3위 (팀: Mark Tenenholtz + Raja Biswas)  
**URL**: https://www.kaggle.com/competitions/lmsys-chatbot-arena/discussion/527766

---

## 핵심 전략

1. **Reward Model에서 파인튜닝 시작** (일반 chat 모델 대신)
2. **대규모 Pseudo Labeling** (500k+ 샘플)
3. **TTA**: 응답 A/B를 뒤집어 예측 앙상블

---

## Mark의 접근법

### 모델
- `sfairXC/FsfairX-Gemma2-RM-v0.1` (Gemma2 리워드 모델)
- `RLHFlow/pair-preference-model-LLaMA3-8B` (Llama3 페어 선호도 모델)

### 학습 설정
- QLoRA: r=64, alpha=16, dropout=0.05, 모든 선형 레이어
- 배치 크기: 16
- 시퀀스 길이: 1800 토큰

### Softcapping 비활성화
- Gemma2의 softcapping을 `None`으로 설정하면 **CV/LB 약 +0.002** 향상

### Pseudo Labeling
- vLLM으로 500k 샘플 생성 (lmsys-1M 데이터에서 다양한 LLM이 생성한 응답쌍)
- 2회 라운드 PL 수행
- TTA 없이 PL 적용 Gemma2-RM 모델: LB 0.880 (시퀀스 길이 8k)

---

## Raja의 접근법

### 베이스라인 모델
- Llama3-8B: QLoRA (r=32, alpha=64) + label smoothing=0.03
- 입력 포맷: `[CONTEXT] {prompt}\n\n[RESPONSE A] {response_a}\n\n[RESPONSE B] {response_b}`
- 긴 대화는 첫 2턴 + 마지막 2턴만 유지

### 최종 모델
- 다양한 오픈 소스 LLM(Gemma2-9B, Llama3.1-8B 등)으로 lmsys-1M 데이터에 응답 생성
- `vllm` 사용 (temperature=0.7, top_p=0.9)
- 인기 DPO 데이터셋도 추가 활용
- 베이스라인 모델의 앙상블로 소프트 레이블 생성 후 파인튜닝

---

## 최종 제출

- Gemma2-9B × 2개 앙상블 (LB: 0.880 + 0.896)
- 최대 길이: 4096 + 3072
- 1번 모델에 응답 A/B 스왑 TTA 적용

---

## 교훈

- **리워드 모델**을 시작점으로 하면 일반 chat 모델보다 성능이 좋음
- 대규모 PL(pseudo labeling)이 CV/LB 향상에 크게 기여
- TTA(응답 순서 뒤집기)는 ~+0.003 성능 향상 효과
