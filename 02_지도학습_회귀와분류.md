# 📚 지도 학습 (Supervised Learning)

---

## 1. 지도 학습이란?

> **입력(Feature) + 정답(Label)을 함께 주고, 그 사이의 규칙을 배우는 방법**

쉽게 말하면, 학생에게 **문제(Feature)와 정답(Label)을 동시에 주면서** 공부시키는 것과 같습니다.

```
지도 학습의 흐름:
[Feature + Label] 쌍으로 구성된 훈련 데이터
       ↓
  패턴/규칙 학습
       ↓
처음 보는 데이터(Test case)에도 예측 가능
```

---

### 핵심 목표

훈련 데이터만 잘 맞추는 것이 아니라,  
**처음 보는 데이터(Test case)에서도 높은 예측 성능**을 내는 것이 진짜 목표입니다.

---

### 주요 용어 정리

| 용어 | 기호 | 설명 | 예시 |
|------|------|------|------|
| 특성 (Feature) | x | 예측에 사용되는 입력 변수 | 집값 예측의 {지역, 평수, 방수, 연식} |
| 라벨 (Label) | y | 맞춰야 하는 정답 (실제값) | 집값, 스팸/정상 |
| 예측값 | ŷ | 모델이 내놓은 결과 | y의 추정치 |
| 오류 (Error) | ŷ - y | 예측값과 실제값의 차이 | 예측: 5억, 실제: 6억 → 오류: -1억 |

> ⚠️ **오류(Error)** 와 **측정 오차(Measurement Error)** 는 다릅니다!
> - 오류: 모델의 예측값 vs 실제값의 차이 → **줄이는 것이 목표**
> - 측정 오차: 데이터를 수집하는 과정에서 발생하는 노이즈

---

## 2. 회귀 (Regression)

> **예측하고 싶은 결과값이 숫자일 때 사용**

---

### 언제 사용할까?

예측 결과가 **연속적인 수치**일 때 사용합니다.

📌 **예시:**
- 공부 시간(Feature) → 시험 점수(Label) 예측
- 평수, 위치(Feature) → 집값(Label) 예측
- 기온, 습도(Feature) → 에너지 사용량(Label) 예측

---

### 회귀 모델의 특징

- **라벨**: 연속적인 숫자 (1, 2.5, 100, -3 등 어떤 수든 가능)
- **예측 결과**: 연속적인 수치
- **핵심 질문**: "얼마나 정확하게 숫자를 예측하는가?"

---

## 3. 회귀 성능 측정 지표

### 📏 MSE (Mean Squared Error, 평균 제곱 오차)

> **모델의 예측 정확도를 숫자로 측정하는 가장 기본적인 지표**

$$MSE = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$

**풀어서 설명하면:**
1. 각 데이터마다 (실제값 - 예측값)을 계산
2. 그 값을 제곱 (음수가 양수로 변환됨)
3. 모든 데이터의 제곱 오차를 더해서 평균을 냄

---

#### 왜 제곱을 할까?

```
데이터 1: 실제 = 5, 예측 = 7  →  오차 = -2
데이터 2: 실제 = 8, 예측 = 6  →  오차 = +2
```

단순히 오차를 더하면 (-2) + (+2) = 0 → 마치 완벽한 모델처럼 보임!

→ **제곱을 하면:** 4 + 4 = 8 → 실제 오류가 존재함을 정확히 표현

또한, **큰 오류에 더 큰 패널티를 부여**합니다:
- 오차 2 → 제곱 = 4
- 오차 10 → 제곱 = 100 (5배 큰 오차인데, MSE는 25배 더 큼)

---

#### MSE 계산 예시

| 실제값 (y) | 예측값 (ŷ) | 오차 (y-ŷ) | 오차² |
|-----------|-----------|-----------|------|
| 100 | 90 | 10 | 100 |
| 200 | 210 | -10 | 100 |
| 150 | 160 | -10 | 100 |

MSE = (100 + 100 + 100) / 3 = **100**

> 💡 MSE 단위가 제곱이라 해석이 어려울 때는 **RMSE (Root MSE)** 를 사용합니다.
> - 위 예시: RMSE = √100 = **10** → "평균적으로 ±10 정도 틀린다"고 해석 가능

---

### 📐 결정 계수 (R², R-squared)

> **"평균으로 예측하는 것보다 내 모델이 얼마나 더 잘 예측하는가?"를 0~1 사이로 나타낸 지표**

$$R^2 = \frac{SSR}{SST} = 1 - \frac{SSE}{SST}$$

---

#### 구성 요소 이해하기

| 기호 | 이름 | 의미 | 공식 |
|------|------|------|------|
| **SST** | Total Sum of Squares | 전체 변동 (실제값이 평균에서 얼마나 퍼져있나) | Σ(y - ȳ)² |
| **SSR** | Sum of Squares Regression | 모델이 설명하는 변동 (예측값이 평균에서 얼마나 퍼져있나) | Σ(ŷ - ȳ)² |
| **SSE** | Sum of Squared Errors | 오차 (실제값 - 예측값의 차이) | Σ(y - ŷ)² |

```
전체 변동(SST) = 모델이 설명하는 변동(SSR) + 설명 못한 오차(SSE)
R² = SSR/SST = 내 모델이 전체 변동 중 설명할 수 있는 비율
```

---

#### R² 해석

| R² 값 | 의미 |
|--------|------|
| 1.0 | 완벽한 모델 (예측이 완전히 정확) |
| 0.8 | 전체 변동의 80%를 설명 (좋은 모델) |
| 0.5 | 전체 변동의 50%를 설명 (보통) |
| 0 | 평균으로 예측하는 것과 다를 바 없음 |
| 음수 | 평균으로 예측하는 것보다 더 못함 (모델이 매우 나쁨) |

> ❓ **R²가 음수가 될 수 있나요?**
> 
> 가능합니다! 모델의 예측값들이 평균값보다 실제값에서 더 멀리 있다면 음수가 나옵니다.
> 즉, **"이 모델은 평균으로 예측하는 것보다 더 형편없다"** 는 의미입니다.

---

### MSE vs R² 비교

| 항목 | MSE | R² |
|------|-----|-----|
| **정의** | 예측값과 실제값 차이의 제곱 평균 | 평균 예측 대비 모델의 상대적 성능 |
| **값의 범위** | 0 ~ 무한대 | 0 ~ 1 (음수도 가능) |
| **단위** | 실제 데이터 단위의 제곱 | 없음 (비율) |
| **해석** | 절대적 오차 크기 | 상대적 성능 비율 |
| **좋은 값** | 0에 가까울수록 | 1에 가까울수록 |
| **장점** | 구체적인 오차 크기를 알 수 있음 | 직관적 해석, 모델 간 비교가 쉬움 |
| **단점** | 직관적 해석이 어려움 | 구체적인 오차 크기를 모름 |

> **💡 실전 팁:** MSE는 "얼마나 많이 틀렸냐", R²는 "얼마나 잘 예측하냐"에 초점.
> 두 지표를 함께 보면 모델을 더 정확하게 평가할 수 있습니다.

---

## 4. 분류 (Classification)

> **예측하고 싶은 결과값이 범주(카테고리)일 때 사용**

---

### 언제 사용할까?

예측 결과가 **미리 정해진 카테고리 중 하나**일 때 사용합니다.

📌 **예시:**
- 이메일 내용(Feature) → 스팸/정상(Label) 분류
- 환자 데이터(Feature) → 질병 있음/없음(Label) 분류
- 이미지(Feature) → 고양이/강아지/새...(Label) 분류

---

### 분류의 종류

| 종류 | 설명 | 예시 |
|------|------|------|
| **이진 분류** | 두 가지 중 하나 | 스팸/정상, 양성/음성 |
| **다중 분류** | 세 가지 이상 중 하나 | 고양이/강아지/새/토끼 |

> ⚠️ **중요:** 분류 모델에서 Label은 반드시 **미리 정해진 범주 중 하나**입니다.
> 임의의 숫자가 나오는 회귀와 다릅니다!

---

## 5. 분류 성능 측정 지표

### ✅ 정확도 (Accuracy)

> **전체 예측 중에서 올바르게 맞춘 예측의 비율**

$$Accuracy = \frac{올바른\ 예측\ 수}{전체\ 예측\ 수} = \frac{1}{n}\sum_{i=1}^{n}\mathbb{1}(y_i = \hat{y}_i)$$

📌 **예시:** 100개를 예측했을 때 90개를 맞추면 → Accuracy = 90%

---

#### 🚨 정확도의 함정: 불균형 데이터 문제

> **Q. 정확도가 99%면 좋은 모델인가?**

반드시 그렇지 않습니다! 다음 예를 보세요:

```
데이터: 암 진단 데이터 1000개
- 정상 환자: 990명 (99%)
- 암 환자: 10명 (1%)

"모든 사람을 정상으로 예측하는 모델"의 정확도:
990 / 1000 = 99% ← 정확도는 높지만 쓸모없는 모델!
```

→ 이런 경우엔 **정확도 외의 다른 지표**도 함께 봐야 합니다.

---

### 🔲 혼동 행렬 (Confusion Matrix)

예측과 실제 값 사이의 관계를 행렬로 나타낸 것입니다.

|  | **예측: 양성(Positive)** | **예측: 음성(Negative)** |
|--|------------------------|------------------------|
| **실제: 양성** | **TP** (True Positive) | **FN** (False Negative) |
| **실제: 음성** | **FP** (False Positive) | **TN** (True Negative) |

#### 각 용어 쉽게 이해하기

| 기호 | 이름 | 설명 | 암 진단 예시 |
|------|------|------|------------|
| **TP** | True Positive | 양성이라 예측했고, 실제로 양성 | 암 환자를 암이라 진단 ✅ |
| **TN** | True Negative | 음성이라 예측했고, 실제로 음성 | 정상을 정상이라 진단 ✅ |
| **FP** | False Positive | 양성이라 예측했지만, 실제로는 음성 (오탐) | 정상을 암이라 진단 ❌ |
| **FN** | False Negative | 음성이라 예측했지만, 실제로는 양성 (누락) | 암 환자를 정상이라 진단 ❌❌ |

> ⚠️ **FN이 특히 위험한 경우:** 실제 암 환자를 "정상"으로 진단하는 것처럼, 놓치면 안 되는 케이스를 놓치는 것

---

### 📊 정밀도 (Precision)

> **"양성이라 예측한 것들 중 실제로 양성인 비율"**

$$Precision = \frac{TP}{TP + FP}$$

```
예시: "암이다"라고 100번 예측했을 때, 
실제로 암인 경우가 80번이면 → Precision = 80%

의미: "양성 판정의 신뢰도" - 허위 양성을 줄이고 싶을 때 중요
```

---

### 📡 재현율 (Recall / Sensitivity)

> **"실제 양성인 것들 중 양성으로 잡아낸 비율"**

$$Recall = \frac{TP}{TP + FN}$$

```
예시: 실제 암 환자가 100명일 때,
모델이 80명을 암이라 진단하면 → Recall = 80%

의미: "실제 양성을 얼마나 놓치지 않는가" - 누락을 줄이고 싶을 때 중요
```

---

### ⚖️ 정밀도 vs 재현율 트레이드오프

```
모델을 보수적으로 설정 (양성 기준 엄격):
→ Precision ↑ (양성 판정하면 거의 맞음)
→ Recall ↓ (실제 양성도 많이 놓침)

모델을 공격적으로 설정 (조금이라도 의심되면 양성):
→ Precision ↓ (양성 판정 중 오탐이 많음)
→ Recall ↑ (실제 양성은 거의 다 잡아냄)
```

---

### 🎯 F1-Score

> **정밀도와 재현율의 조화 평균 — 둘 다 중요할 때 사용**

$$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$

- 정밀도와 재현율이 **둘 다 높을수록** F1-Score도 높아집니다.
- 하나만 높고 하나가 낮으면 F1-Score는 낮게 나옵니다.
- 불균형 데이터에서 특히 유용한 지표입니다.

📌 **예시:**
- Precision = 0.9, Recall = 0.9 → F1 = 0.9 (균형 잡힌 좋은 모델)
- Precision = 1.0, Recall = 0.1 → F1 = 0.18 (재현율이 너무 낮음)

---

## 6. 회귀 vs 분류 한눈에 비교

| 구분 | 회귀 (Regression) | 분류 (Classification) |
|------|------------------|----------------------|
| **예측값 유형** | 연속적인 숫자 | 미리 정해진 범주 |
| **예시** | 집값, 온도, 점수 | 스팸/정상, 질병 유무 |
| **주요 지표** | MSE, RMSE, R² | Accuracy, F1-Score |
| **핵심 질문** | 얼마나 정확한 숫자? | 어떤 카테고리에 속하나? |
